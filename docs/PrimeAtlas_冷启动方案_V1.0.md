## 1. 冷启动总体策略

### 核心原则

根据市场评审报告的修正结论：

- **三画像合计 TAM 约 200-300 万人**，属利基市场，不是大众市场
- **核心交叉用户约 80 万人**，高付费意愿+高频使用约 15-25 万人
- **6 个月合理 MAU 目标：8-15 万**（非 PRD 原 50 万）
- **唯一潜在护城河**：伙伴情感连接 + 数据积累组合（需 12-18 个月建成）
- **MVP 阶段定位**："训练+伙伴"为核心，英语和读书为轻量附加

### 三阶段概览

```
阶段A: 种子用户 (0→100)      阶段B: 早期增长 (100→1000)     阶段C: 规模增长 (1000→10000)
┌────────────────────┐  ┌────────────────────┐  ┌────────────────────┐
│ 第1-4周             │  │ 第5-8周             │  │ 第9-16周            │
│                     │  │                     │  │                     │
│ 一对一邀请           │  │ 内容营销+KOL         │  │ 小程序获客→App转化    │
│ 封闭内测             │  │ 小红书/B站/知乎      │  │ 病毒传播机制          │
│ 深度用户验证         │  │ 垂直社区渗透         │  │ 社区运营              │
│                     │  │                     │  │                     │
│ 目标: 100人          │  │ 目标: 1000人         │  │ 目标: 10000人         │
│ 次日留存 ≥50%        │  │ 7日留存 ≥30%         │  │ 付费转化 ≥3%          │
└────────────────────┘  └────────────────────┘  └────────────────────┘
```

### 阶段A：种子用户（0→100）— 第1-4周

#### 目标

找到前 100 个深度使用的高质量用户，验证核心价值被感知。

#### 种子用户画像聚焦：谁最可能成为"布道者"？

根据三画像分析，种子用户按传播力排序：

| 优先级 | 画像类型 | 特征 | 种子人数 | 布道潜力 | 获取策略 |
|:---:|------|------|:---:|:---:|------|
| **P0** | 健身博主/小KOL | 自有粉丝、内容产出能力强 | 20人 | **极高** | 一对一私信邀请，深度合作 |
| **P0** | 产品经理/设计师 | 对产品敏感、愿写测评、圈内传播 | 30人 | **高** | 产品社区（即刻/少数派/PMCAFF）定向邀请 |
| **P1** | 篮球/羽毛球重度爱好者 | 目标明确、训练频率高、社区活跃 | 25人 | **中高** | 虎扑/懂球帝/中羽在线 DM 邀请 |
| **P1** | 产后妈妈 KOC | 需求真实、小红书活跃、愿意分享 | 15人 | **中高** | 妈妈社群/产后康复群定向邀请 |
| **P2** | 大学生意见领袖 | 传播力强、愿意尝鲜、时间充裕 | 10人 | **中** | 校园大使模式 |

#### 获取方式

**方式一：一对一邀请（核心渠道，60人）**

- 制作个性化邀请函（数字卡片，非群发消息）
- 邀请函包含：产品概念简述、"你的专属伙伴 [Alex/小野/Elena] 在等你"、内测专属权益
- 每位受邀者配专属 onboarding 引导（产品经理/运营 1v1 微信对接）
- 邀请话术模板（分画像）：

| 画像 | 邀请切入角度 |
|------|------------|
| 健身博主 | "我们正在做一个有 AI 训练伙伴的产品，不是模板 App。想请你作为第一批体验者，你的反馈会直接影响产品方向。" |
| 产品经理 | "一个把健身+英语+读书融合的产品，核心是 AI 伙伴陪伴。想听听你的产品判断。" |
| 篮球爱好者 | "你的弹跳训练有 AI 专属教练了。5 个伙伴可选，每个都有性格。" |
| 产后妈妈 | "我们知道产后康复最需要的是'被理解'而非'被催促'。Elena 是运动科学博士，她会陪你。" |

**方式二：封闭内测社区（30人）**

- 创建种子用户微信群（上限 100 人）
- 采用邀请码机制，每人限邀请 1-2 人
- 群内氛围营造：每日伙伴金句、用户成就分享、产品经理实时答疑

**方式三：社区定向招募（10人）**

- 即刻 App 动态："我们在做一个有灵魂的训练伙伴产品"
- 少数派文章：《一个产品经理为什么想做 AI 训练伙伴》
- PMCAFF 帖子（不直接推广，以产品讨论形式）

#### 种子用户权益设计

| 权益等级 | 条件 | 权益内容 |
|:---:|------|------|
| **先行者** | 完成注册 + 首次训练 | 永久"先行者"徽章、Pro 功能 3 个月免费 |
| **共建者** | 提交 5 条以上有效反馈 | 终身 Pro 会员（¥699 价值）、参与产品决策投票 |
| **布道者** | 邀请 3 人以上注册并完成首次训练 | 伙伴自定义外观解锁（P2 功能提前体验）、名字进入 App 感谢页 |
| **所有种子用户** | 内测期间注册 | Pro 功能 3 个月免费、专属客服通道 |

#### 种子用户运营日历（第1-4周）

| 周次 | 主题 | 关键动作 | 产出 |
|:---:|------|------|------|
| W1 | 初识伙伴 | 1v1 引导完成伙伴选择+首次训练、群内伙伴自我介绍接龙 | 首批用户激活 |
| W2 | 深度体验 | 发布"7天挑战"（每天解锁一个新功能）、收集首次反馈 | 功能体验报告 |
| W3 | 共创反馈 | 线上圆桌讨论（分画像）、投票决定 P1 功能优先级 | 优先级排序 |
| W4 | 破圈准备 | 邀请种子用户产出 UGC（伙伴对话截图、训练成就）、准备公测素材 | UGC 内容库 |

### 阶段B：早期增长（100→1000）— 第5-8周

#### 目标

验证 PMF，建立口碑，1000 用户，7 日留存 ≥30%。

#### 渠道策略

根据市场评审报告的渠道分析，聚焦 **3 个核心渠道**：

**渠道一：小红书 — 产后恢复/健身垂类（40% 预算，P0）**

- **为什么**：产后妈妈画像需求刚性、社区活跃、竞争相对较少，获客成本 ¥30-60
- **内容矩阵**：
  - "产后康复日记"系列：用户视角的真实记录（非硬广）
  - "我的 AI 教练对我说……"系列：伙伴对话截图（参考 Replika 增长模式）
  - "30 天核心重建计划"系列：专业内容+产品植入
- **KOL 合作**：3-5 个产后康复/健身 KOL，预算 ¥2-5万/条
- **投放节奏**：W5 开始自然内容铺垫 → W6 KOL 首发 → W7-W8 信息流投放

**渠道二：知乎/少数派 — 效能/产品圈（30% 预算，P0）**

- **为什么**：与"效能系统"定位契合，产品经理/效率工具爱好者聚集
- **内容矩阵**：
  - 《我做了个有 AI 训练伙伴的 App——不是工具，是陪伴》产品理念文章
  - 《当 Keep 和多邻国在同一个 App 里——PrimeAtlas 的产品逻辑》深度分析
  - 《为什么健身坚持不下去？因为没有人陪你》痛点文章
  - "产品经理的自我效能系统"系列
- **策略**：先发深度文章建立认知，再引导下载体验

**渠道三：B站 — 运动科技区（20% 预算，P1）**

- **为什么**：内容营销主阵地，但制作成本高、竞争激烈
- **内容矩阵**：
  - "我用 AI 训练了一个月，弹跳涨了 X cm"（实测向）
  - "5 个 AI 训练伙伴，哪个最适合你？"（对比向）
  - "AI 教练 vs 真人私教：30 天对比实验"（冲突向）
- **KOL 合作**：1-2 个运动科技区 UP 主，预算 ¥5-15万/条

**渠道四：垂直社区渗透（10% 预算，P2）**

- 虎扑/懂球帝：非硬广，以"篮球训练干货+AI 工具辅助"形式
- 中羽在线：羽毛球专项训练内容
- 豆瓣读书小组：AI 读书伙伴概念传播

**KOL 筛选标准**：
- 粉丝画像与目标用户重叠度 ≥60%
- 过往内容调性与产品定位一致（非纯流量号）
- 愿意深度使用产品（至少 14 天）再产出内容
- 接受"真实体验"而非"念稿推广"

### 阶段C：规模增长（1000→10000）— 第9-16周

#### 目标

建立增长飞轮，10000 用户，付费转化 ≥3%。

#### 小程序获客→App转化

根据市场评审 P0-05 建议，小程序应纳入 MVP（而非 PRD 的 P2）。具体设计：

**小程序 MVP 功能范围**：

| 模块 | 小程序端 | App 端（引导转化） |
|------|------|------|
| 伙伴系统 | 伙伴选择 + 基础对话 | 动态头像 + 完整记忆 + 语音 |
| 训练 | 预设模板训练 + 基础记录 | AI 定制计划 + 多场景编排 + 动作视频 |
| 英语 | 每日 3 次微会话 | 无限对话 + 音素纠错 + LISA 评估 |
| 读书 | 书单推荐 + 基础总结 | PDF 导入 + Agent 对话 + 伙伴共读 |
| 数据 | 本周概览 | 完整仪表盘 + 趋势分析 |

**转化漏斗设计**：

```
微信小程序
  │ 用户完成第3次训练 → 弹出"你的伙伴想和你说句话"
  │ 伙伴说："我们认识3天了。你想看看我真正的样子吗？"
  │  → "下载 App，解锁完整伙伴体验" [CTA按钮]
  ↓
App Store / 应用宝
  │ 下载完成 → 自动继承小程序数据
  │ 伙伴说："你来了。我一直在等你。"
  │ → 展示 App 独占功能（动态头像首次亮相）
  ↓
免费版使用
  │ 第7天弹窗："你的伙伴已经认识你了"
  │ → 展示 Pro 会员权益对比
  │ → 首月 ¥19 体验价（原价 ¥49）
  ↓
付费转化
```

**转化率预估**：
- 小程序→App下载：15-20%
- App下载→注册激活：80-85%
- 注册→7日留存：40-50%
- 7日留存→付费转化：8-12%
- 整体漏斗：小程序用户→付费用户 ≈ 0.5-1%

#### 病毒传播机制

根据市场评审建议，强化病毒传播（PRD 原方案仅靠成就卡片分享，传播力不足）：

**机制一：伙伴对话截图（核心病毒引擎）**

- 参考 Replika 增长模式——用户天然愿意分享 AI 说出的有趣/暖心/扎心的话
- 实现：训练结束后/对话中，一键生成"伙伴对我说"卡片
- 卡片设计：伙伴头像 + 对话内容 + 日期 + "来自 PrimeAtlas"水印
- 传播力评估：★★★★★（最强病毒机制）

**机制二：挑战邀请（社交竞争）**

- "我连续训练 7 天了，你能超过我吗？" → 生成挑战卡片
- 好友点击 → 进入小程序 → 开始 7 天挑战
- 双方可在 App 内看到对方进度（匿名或好友模式）
- 传播力评估：★★★★☆
- **注**：好友 PK 已根据市场评审 P1-02 建议提升至 P0

**机制三：PR 成就卡片（原方案保留但优化）**

- 训练 PR、英语 PR、知识 PR 三种风格可选
- 增加"伙伴的点评"增加传播力（纯数据卡片缺乏社交货币）
- 传播力评估：★★★☆☆

**机制四："我的伙伴档案"生成**

- 训练 30 次后解锁："你和 [伙伴名] 的 30 天报告"
- 包含：训练次数、PR 数量、伙伴金句 TOP5、伙伴对你的评价
- 高度个性化的内容，传播力强
- 传播力评估：★★★★☆

## 3. 产品冷启动体验

### 3.1 "90 秒试炼"免注册体验

**设计理念**：用户在 90 秒内体验核心差异化价值，无需注册即可感受到"有人陪我变强"。

#### 体验流程

```
打开 App
  │
  ├─ 闪屏（3秒）："你的 AI 训练伙伴在等你"
  │   动态伙伴剪影 + "不需要注册，先试试？"
  │
  ├─ 选择体验伙伴（10秒）
  │   ┌─────────┬─────────┬─────────┐
  │   │  Alex   │  Elena  │  小野    │
  │   │  "硬核"  │ "科学"   │ "热血"   │
  │   │  适合突破 │ 适合康复  │ 适合球类  │
  │   └─────────┴─────────┴─────────┘
  │   [跳过，随机分配]  [我要注册]
  │
  ├─ 体验训练（60秒）
  │   预设3套"体验训练计划"（30秒/套）：
  │   ├─ 弹跳入门：3个动作体验
  │   ├─ 核心激活：3个动作体验
  │   └─ 全身唤醒：3个动作体验
  │   
  │   体验中：
  │   • 3-2-1-GO 倒计时（完整动画）
  │   • 伙伴语音鼓励（1-2句）
  │   • 动作教学卡片（关键要领）
  │   • PR 庆祝模拟（展示效果）
  │
  ├─ 伙伴对话（15秒）
  │   体验伙伴说一段话：
  │   Alex: "刚才那几下不错。想知道我还能为你做什么吗？"
  │   小野: "兄弟！刚才那几下有点意思！想继续吗？"
  │   Elena: "刚才的动作质量不错。我还能帮你更多。"
  │
  └─ 转化引导（2秒）
      "创建账号，我会记住你的一切。"
      [手机号注册] [微信登录] [稍后再说]
```

#### 体验后转化路径

**转化时机**：
1. **即时转化**：体验结束后 → "创建账号，我会记住你"（预计转化率 15-20%）

- 体验伙伴自动成为正式伙伴（无需重新选择）

## 4. 数据冷启动

AI 在没有用户数据时如何工作。

### 4.1 基于画像的默认参数

当用户首次使用时（无历史数据），系统根据画像预设默认参数：

| 参数 | 张逸飞默认值 | 林雨桐默认值 | 陈知行默认值 | 通用默认值 |
|------|:---:|:---:|:---:|:---:|
| 训练经验 | 中级（12-24月） | 初级（0-6月） | 中级（6-12月） | 初级 |
| 推测 1RM | 基于体重×系数 | 基于体重×0.3系数 | 基于体重×系数 | — |
| 训练频率 | 4-5次/周 | 3-4次/周 | 5-6次/周 | 3次/周 |
| 英语水平 | B1（职场基础） | B2（留学基础衰退） | B1（应试基础） | A2 |
| 阅读偏好 | 运动科学/商业 | 育儿/自我成长 | 运动技术/学习方法 | 综合 |
| RPE 基准 | 7-8（高负荷） | 5-6（中等负荷） | 7-8（竞技向） | 6 |
| 恢复能力 | 中等 | 较低（产后） | 高（年轻） | 中等 |

**画像默认参数的来源**：
- 用户注册时选择的画像标签
- 数据采集流程中填写的身体基础数据
- "跳过"的数据使用画像默认值

### 4.2 冷启动推荐策略

**训练计划冷启动**：

```
第 1-3 次训练：使用预设模板计划（见 2.1）
  ├── 基于画像 + 目标 + 器材 匹配最接近的模板
  └── 首次训练以动作学习+建立基准为主

第 4-7 次训练：模板 + 微调
  ├── 根据前 3 次 RPE 反馈调整负荷
  └── 开始使用 1RM 推算（基于实际数据）

第 8+ 次训练：AI 个性化编排
  ├── 积累足够数据（≥7次训练记录）
  └── 启用完整 AI 训练计划生成
```

**英语对话冷启动**：

```
第 1-3 次对话：固定话题（画像匹配）
  ├── 张逸飞：职场英语基础话题
  ├── 林雨桐：日常英语恢复话题
  └── 陈知行：考研口语话题

第 4-7 次对话：LISA 初评后自适应
  └── 根据评估结果调整难度和话题

第 8+ 次对话：个性化话题推荐
  └── 结合训练数据+阅读数据生成关联话题
```

**读书推荐冷启动**：

```
首次使用：基于画像的预设书单（见 2.5）
  └── 每画像推荐 3 本入门书

第 1 本读完后：基于阅读行为推荐
  └── 阅读速度、笔记偏好、对话深度 → 推荐相似书籍
```

### 4.3 数据积累→个性化过渡路径

```
用户数据量         AI 个性化程度
──────────────────────────────────────
0 次训练          ██░░░░░░░░ 20%  画像默认参数
1-3 次训练        ████░░░░░░ 35%  模板 + RPE 反馈
4-7 次训练        ██████░░░░ 55%  1RM 推算 + 初步模式识别
8-14 次训练       ████████░░ 70%  AI 个性化编排
15-30 次训练      █████████░ 85%  多场景编排 + 模式记忆
30+ 次训练        ██████████ 95%  完整 AI 能力 + 伙伴 Lv.3
```

**过渡的关键信号**：
- 当某动作的 1RM 推算置信度 >80% → 启用该动作的个性化负荷
- 当 RPE 评分一致性 >70%（用户自评 vs AI 预测） → 启用动态调整
- 当连续训练 ≥7 次 → 启用模式识别（周几状态好/差）
- 当伙伴记忆 ≥10 条 → 伙伴开始引用用户历史

### 4.4 "借用"相似用户数据的策略（隐私安全）

在不侵犯隐私的前提下，利用相似用户的脱敏数据辅助冷启动：

| 策略 | 实现方式 | 隐私保障 | 使用场景 |
|------|------|------|------|
| **人群基准值** | 计算同类用户的统计分布（均值/标准差） | K-匿名化（K≥50），仅展示群体统计 | 1RM 初始估算、训练频率基准 |
| **动作效果预估** | 同类用户某动作的平均进步曲线 | 聚合数据，不关联个体 | "类似用户练这个动作 8 周平均提升 X" |
| **计划模板优化** | 同类用户计划完成率+满意度排序 | 仅使用聚合指标 | 冷启动时推荐"最受欢迎"的模板 |
| **伙伴对话风格** | 同类用户对伙伴风格的偏好分布 | 仅使用统计分布 | 首次伙伴推荐（性格匹配） |

**红线**：
- 绝不使用个体级别的数据（即使是脱敏的）
- 所有聚合统计必须 K≥50
- 在 App 中明确告知用户："我们参考了与您目标相似的用户群体数据来优化您的初始体验"
- 数据贡献采用 Opt-in（PRD 设计铁律 V）

| **时间错峰展示** | 将不同时段的用户内容在活跃时段集中展示，避免"发了没人看" | ✅ 内容真实，仅展示时间调整 |
| **种子用户激励** | 邀请种子用户在特定时段集中活跃（如每周三晚"社区之夜"） | ✅ 真实用户行为 |

**不做的**：
- ❌ 虚构用户、虚构内容、虚构数据
- ❌ 机器人自动回复假装真人
- ❌ 夸大用户数量或活跃度
