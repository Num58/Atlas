# 第二部分：模块详细规格

## 2.1 伙伴系统 (Companion System)

### CP-01 伙伴创建与选择

**功能描述**：首次启动的伙伴选择流程。新用户注册后进入沉浸式引导动画，展示5个初始伙伴（Alex·赤焰/小野·疾风/Elena·晨曦/K·零/凛·静流），每个伙伴展示姓名、标语、人格描述、专长、英语口音和推荐用户画像。用户可左右滑动浏览，选择伙伴后触发2秒握手动画+伙伴签名色屏幕闪烁。支持返回浏览和3题人格测试推荐匹配。

**核心交互流程**：
```
注册完成 → 沉浸式引导动画(介绍AI训练伙伴概念) 
→ 展示第一个伙伴卡片(含姓名/人格/专长/口音/推荐画像) 
→ 用户左右滑动浏览5个伙伴 
→ 点击"选择"→2秒握手动画(伙伴伸手+屏幕闪伙伴色) 
→ 确认选择→保存→进入数据采集(TR-01)
```

**关键数据结构**：
```json
{
  "partner_id": "alex_v1",
  "name": "Alex",
  "codename": "赤焰",
  "personality": "硬核老兵，退伍军人转健身教练。话少但精准。",
  "specialty": ["力量训练", "体能突破"],
  "english_accent": "美式",
  "signature_color": "#FF4444",
  "recommended_profiles": ["张逸飞型"],
  "voice_profile": {"base_pitch": 0, "speed": 0, "energy": "high"}
}
```

**与其他模块的关联点**：
- TR-01：伙伴选择完成后进入数据采集
- CP-02：选择后激活伙伴动态头像
- CP-03：开始记录伙伴记忆
- IF-01：依赖用户注册完成

### CP-02 伙伴动态头像系统

**功能描述**：伙伴头像在多个页面（Tab Bar/训练执行/英语对话/读书）持续渲染，≥30 FPS。支持8种状态（空闲/专注/满意/惊叹/庆祝/关切/对话/阅读），每种状态有不同的动画表现。低端设备自动降级为简化或静态头像。深夜模式动画幅度减半。

**核心交互流程**：
```
状态机驱动：
IDLE(空闲) → 用户开始训练 → FOCUSED(专注) 
→ 完成一组RPE≤7 → SATISFIED(满意) 
→ 完成一组RPE≥8.5 → IMPRESSED(惊叹) 
→ 突破PR → CELEBRATE(庆祝,8秒) 
→ 标记疼痛 → CONCERNED(关切) 
→ 进入英语 → CONVERSATIONAL(对话) 
→ 进入读书 → READING(阅读)
```

**关键数据结构**：状态机+动画规格（时长/缓动函数/关键属性）

**与其他模块的关联点**：
- TR-04：训练执行页右上角持续显示
- TR-08：3-2-1-GO倒计时中变化
- TR-10：PR庆祝动画
- EN-02：英语对话中对话状态
- RD-04：读书中阅读状态
- IF-05：低端设备渲染降级

### CP-03/CP-15 伙伴四层记忆系统

**功能描述**：四层记忆结构，每层有独立的数据结构和触发条件。L1里程碑记忆（PR/读完书/英语里程碑），L2声明记忆（用户表达情感/目标/偏好），L3模式记忆（3+相似事件生成的模式），L4合成记忆（跨模块关联）。存储超过10MB时压缩30天前记忆。

**核心交互流程**：
```
事件发生 → 判断记忆层级 → 写入对应层级
→ 伙伴响应时查询相关记忆(时效性加权) → 融入响应上下文
```

### TR-13~TR-20 训练编排核心算法

**功能描述**：
- **NVU**：标准化容量单位，五因子修正（MF/CF/NF/EF/GF），日容量上限基于训练水平
- **ACWR**：7d急性/28d慢性滚动窗口，按训练水平分级阈值，超限自动干预
- **五维疲劳**：局部(12肌群30%)+CNS(20%)+累积(25%)+感知(15%)+恢复(10%)，指数0-100
- **疲劳传递**：动作间疲劳系数矩阵，CNS累积，组间休息动态计算
- **连续高RPE**：同肌群连续高RPE触发累积惩罚，含回升规则
- **疼痛熔断**：≥4级立即暂停，展示停止/跳过/忽略选项
- **12肌群热力图**：绿/黄/橙/红四色编码，含自然语言恢复建议

**核心交互流程**：
```
训练日启动 → 疲劳评估(12肌群+CNS+ACWR+RPE+恢复) 
→ 综合疲劳指数 → 容量分配(日容量上限×疲劳修正×场景分配) 
→ 动作编排(根据恢复热力图选择+多级拓扑排序+拮抗配对+疲劳传递调整) 
→ 实时监控(每组后更新局部疲劳+RPE动态调整+心率约束) 
→ 训练后处理(更新疲劳追踪+记录偏差+调整次日计划+恢复建议)
```

**与其他模块的关联点**：
- TR-03：训练计划生成的基础算法

- DA-03：数据仪表盘的可视化

**功能描述**：追踪连续训练天数，火焰动画随天数增长而进化。8级火焰：小火苗(1-3天)→橙色(4-6)→橙红(7-13)→红焰(14-20)→蓝焰核心(21-29)→金色(30-59)→紫焰(60-99)→彩虹焰(100+)。里程碑(7/14/21/30/60/100天)触发进化动画+伙伴祝贺。休息日火焰进入休眠状态(灰色微光，不灭)。任何模块(训练/英语/读书)活动都计入。

**核心交互流程**：
```
用户完成任意模块活动 → 检查是否为当日首次
→ 是: 连续天数+1 → 更新火焰等级和动画
→ 达到里程碑→触发进化动画+伙伴祝贺
→ 休息日: 火焰休眠(灰色微光)
→ 中断: 火焰熄灭(可重新点燃)
```

**与其他模块的关联点**：

## 2.7 Agent生态 (Agent Ecosystem)

### AG-01~AG-06 多Agent评审体系

**功能描述**：11个专业Agent组成的评审委员会，对所有AI生成的训练/学习计划进行交叉验证。训练评审团5个Agent（运动科学/安全/个性化/资料一致性/用户体验），英语评审团4个Agent（语言学/难度适配/内容一致性/学习体验），通用2个Agent（跨模块协调/首席评审）。三阶段协作（冷启动）：提案→并行评审→裁决确认。五阶段协作（完整）：提案→并行评审→交叉借鉴→分歧裁决→终稿确认。安全审核员拥有一票否决权。防敷衍四机制保障评审质量。

**核心交互流程（三阶段冷启动版）**：
```
计划生成请求 → 提案Agent产出初稿
→ 并行评审: 所有Agent同时独立评审(30秒超时)
→ 裁决确认: 首席评审官汇总
  → 安全VETO? → 直接驳回
  → 2+ FAIL? → 驳回重做
  → 1 FAIL? → 标注争议，有条件通过
  → CONDITIONAL_PASS → 自动应用修改
  → 全部PASS → 直接通过
→ 最终计划 + 评审摘要 → 推送给用户
```

**关键数据结构**：
```json
{
  "review_id": "rev_xxx",
  "plan_id": "plan_xxx",
  "phase": "3_stage_cold_start",
  "reviews": [
    {
      "agent": "运动科学审核员",
      "result": "CONDITIONAL_PASS",
      "findings": [{"dimension": "ACWR", "score": 0.85, "suggestion": "深蹲5x5→4x5"}],
      "reasoning": "..."
    }
  ],
  "verdict": "CONDITIONAL_PASS",
  "auto_modifications": [{"field": "深蹲.sets", "from": 5, "to": 4}]
}
```

**与其他模块的关联点**：
- TR-03：训练计划生成后触发评审
