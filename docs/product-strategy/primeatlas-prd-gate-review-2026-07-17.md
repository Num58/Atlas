## 4. 已识别且已在 PRD 修正的冲突

- 最多 3 目标 → 建议统一为最多 3 活跃域，每域多目标；
- 标签选择 → 用户自由表达，标签仅内部推理；
- PR 自动改画像 → 自动产生候选，用户确认后建版本；
- 冲突绝不阻断 → 普通冲突不阻断，危险方案必须阻断；
- Pulse 解锁训练 → Pulse 为可选仪式，不锁核心计划；
- 默认热血 → 默认专业；
- 跳过 RPE 用上次值 → 改为本次缺失；
- 四套调性 UI → 单套语义 Token；
- 全量 E2E 加密 + 云端计算 → 改为可验证的数据分级与服务可见范围；
- 用户任意换模型 → 模型偏好不得改变专业和安全规则；
- 原始 Agent 推理链 → 只展示结构化证据、规则和决策日志；
- SwiftUI/Compose + Go/Rust 技术描述 → 不作为产品需求，后续架构阶段重新裁决。

## 5. 技术前提审查追加结论

架构侧产品技术审查进一步确认，以下不是研发实现细节，而是必须由 PRD/Spec 明确定义的产品语义：

- Consent 不能只有布尔值，必须保存目的、范围、接收方、有效期和撤回；同意记录必须可版本化、可审计，撤回后立即停止后续非必要处理；
- 在生成训练方案前，若用户属于高风险或当前不支持的身份/健康场景，必须执行一次独立、显式的主动确认；该确认不得通过默认勾选、预选项、静默接受或与其他同意捆绑完成；
- 高风险/不支持身份确认必须记录确认版本、确认时间、适用范围与当时展示内容，用于审计；确认只表达用户已知悉产品边界，不免除系统在计划生成、执行前和执行中的持续安全检查；
- 命中未成年人、孕期、急性疼痛、明确疾病或术后康复任一场景时，阻止生成普通训练方案，进入保守提示和专业转介；不得以用户确认、普通冲突仲裁、模型超时或数据缺失为由默认放行；
- 安全否决是独立状态与权限边界：普通冲突可由用户仲裁，安全阻止不可由用户覆盖；安全暂停立即生效，完整理由可异步返回，但超时不能自动放行；
- 真实执行采用追加事实，不能被云端新计划覆盖；多端画像冲突不能自动合并成正式画像，`system_auto` 只生成画像候选；
- 离线执行必须分配稳定幂等键并保存本地事实；联网同步以追加事实、版本比较和显式冲突解决为准，不得重复执行、重复 PR 或覆盖用户手改；
- 状态机必须区分候选、待确认、已生效、普通冲突、安全阻止、已撤回与同步冲突；禁止从待确认/冲突状态静默跃迁到正式生效，禁止安全阻止因超时进入可执行；
- 调性、身份引导、日内调整、安全、完整计划和知识评估使用不同 P95 SLO；安全判定超时采用 fail-closed，非安全生成超时采用可解释降级；
- 数据可见范围必须分为仅设备、PrimeAtlas 服务端、第三方模型三类；按目的最小化传递，第三方模型不得默认接收健康原文、知识原文或完整画像，具体字段和保留期在 Spec 数据契约中冻结；
- 普通冲突 S0 `blocked_user=false` 不能外推到安全事件；
- “永不断档、无条件信赖、精准预测”等措辞须改成可验收范围和失败降级。

因此，技术前提已满足进入 Spec 的产品语义门槛。后续仍须在 Spec 中锁定状态转换表、Consent/审计 Schema、数据可见字段矩阵、各场景 P95 SLO、离线幂等与冲突算法、错误码及 Given/When/Then 验收；在这些内容锁定前，不允许进入架构实现和开发。

### QA 最终复核

QA 对已汇编内容进行最终复核后确认：**无新增 P0 产品决策缺口**。现有门禁已经覆盖安全否决、专业计划、训练方案前高风险/不支持身份主动确认、画像显式确认、离线/同步不丢数据、外部依赖降级、Consent 与审计、指标可测性、调性禁区和普通冲突用户主权。

QA 维持分层门禁：产品 PRD 已 Accepted 并允许进入 Spec；Spec 中的公式、状态、数据契约、SLO 与降级验收未锁定前，不进入架构实现和开发；除此之外不再扩写需求范围。

### PRD Ready 最终判定规则

以下任一未在 Spec 中冻结，相关需求保持 `Blocked`，且不得进入架构实现和开发：

1. 不可验收的愿景承诺尚未改写为行为、边界和降级；
2. 预计时间、恢复率、进度、融合等数字缺公式版本、样本和误差/置信表达；
3. 场景自动检测缺真实授权数据源、用户声明和拒权降级；
4. 完成质量仅依赖滑动完成/RPE，缺客观或可说明证据和不确定状态；
5. 隐藏进步/觉醒PR缺规则、样本、证据、去重、反馈、撤销和重算；
6. 任意场景允许跳过安全/康复复评；
7. 用户仲裁可覆盖安全否决，或高风险/不支持身份确认被实现为默认勾选、捆绑同意、不可审计的一次性布尔值；
8. 未成年人、孕期、急性疼痛、明确疾病、术后康复可进入普通训练方案，或用户确认被解释为免除持续安全检查；
9. S0 普通冲突、画像确认、调性 cooldown/block 仍存在双重口径；
10. 依赖云端模型、可穿戴、日历、位置或第三方内容的需求，未同时覆盖权限拒绝/撤回、数据不足/异常/过期、服务超时/限流/不可用、内容失效/下架/撤权/来源不可达四类 Given/When/Then 降级。

外部依赖每类降级必须定义：仍可用能力、禁止输出、用户提示、缓存与新鲜度、重试或安全替代、审计证据。安全场景不得因依赖失败默认放行。

这 10 条属于 Spec Ready 判定标准，不是新的功能范围。
