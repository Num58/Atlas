## 1. 使用说明

本矩阵确保每项终局需求都能追溯到蓝图来源、v6 原型证据、PRD 规则、数据/事件、异常与验收。覆盖状态：

- **已覆盖**：v6 已提供可交互的主要体验证据，但不代表生产能力已实现；
- **部分覆盖**：仅有页面、静态样例或单域演示，缺少状态、规则、数据或异常；
- **未覆盖**：v6 没有足够体验证据，需要新增原型；
- **冲突**：v6 或蓝图中的现有表达与评审原则矛盾，必须纠偏。

评审状态：`Accepted / Spec Blocked / Approved / Changed`。12 项 P0 产品决策均已由用户确认，因此产品 PRD 状态为 **Accepted**，可进入 Spec；矩阵中保留的 `Blocked` 均表示需要在 Spec 冻结公式、状态、接口、数据或异常验收，不能解释为允许直接开发。任何受影响 Requirement 在 Spec 未完成且门禁未通过前均为 **Spec Blocked**。

## 2. 追踪字段

| 字段 | 含义 |
|---|---|
| Requirement_ID | 稳定需求编号 |
| 模块/子模块 | A–K、Agent 或全局能力 |
| 产品目标/铁律 | 对应价值和不可违背原则 |
| 蓝图章节 | 权威需求来源 |
| v6 证据 | 页面、函数、交互或缺失点 |
| 原型覆盖 | 已覆盖/部分/未覆盖/冲突 |
| PRD 章节 | 产品规则归属 |
| 正常流程 | 主成功路径 |
| 异常/边界 | 空、错、权限、低数据、安全等 |
| 验收摘要 | 可验证结果 |
| 数据/事件 | 关键对象和因果事件 |
| 离线 | 离线产品语义 |
| 隐私分类 | 敏感性与 Consent |
| 无障碍/性能 | 等效路径和 SLO |
| 依赖/风险 | 前置能力和主要风险 |
| 决策/状态 | 已确认裁决及 Spec 状态 |

## 9. 原型补齐门禁

在 Spec 完成并进入开发前，原型层至少需要补齐以下体验证据，不能只靠文字 PRD：

1. 身份草案确认、版本历史和过渡候选；
2. 身份→多个活跃域→多目标→统一计划；
3. 普通冲突双轨选择与安全否决完整流程；
4. 场景变化的调整前后对比、确认、撤销和失败；
5. Pulse 三语义拆分、可绕过和无障碍替代；
6. 离线执行、待同步、冲突解决和账号合并；
7. Consent 拒绝/撤回、导出/删除和第三方模型说明；
8. 数据不足、模型超时、内容失效和专业计划降级；
9. PR 证据、误报纠错、撤销、周期回顾和画像确认；
10. 单域用户、休息日、低能量、高风险和跨时区旅程；
11. 训练方案生成前适用范围确认：首发仅支持 18 岁以上普通健康成年人，默认均未选择、逐项主动确认；命中、拒绝或无法确认后的普通精细训练拦截、保守提示/专业转介、回到非训练能力、持续安全审查提示和审计记录；
12. 六 Tab 候选 IA 在身份主轴、三活跃域、单域用户和新增成长域下的可达性验证。

| 拒绝或无法确认 | 用户主动拒绝确认，或因理解、辅助技术、网络/设备故障等原因无法完成确认 | 用户请求生成训练方案 | 不把拒绝/无法确认解释为符合范围，不生成普通精细训练方案；提供可访问的重试、保守提示、专业转介及非训练能力入口；分别记录 `eligibility_rejected` 或 `eligibility_unavailable` 及失败原因，不捆绑诱导 Consent |
| 命中未成年人 | 用户选择或声明属于未成年人 | 提交确认或后续资料发现冲突 | 安全否决；不输出成人普通精细训练方案；说明当前不支持/需监护与地区规则（以 Spec 为准）；提供返回目标管理或安全信息入口；记录命中规则与版本，不允许用户覆盖原危险路径 |
| 命中孕期 | 用户选择孕期或执行期状态更新为孕期 | 请求生成、重排或继续相关训练 | 安全否决当前通用训练方案；不诊断；提供停止生成、寻求合格专业人员及 Spec 定义的安全替代入口；已缓存危险计划失效；记录声明、规则命中、替代和复评条件 |
| 命中急性疼痛 | 用户在准入或执行反馈中报告急性疼痛 | 请求生成/继续训练 | 立即本地暂停相关危险动作；不得因模型超时、离线或用户坚持而放行；展示原因、保守替代/暂缓和重新评估条件；记录 `safety_blocked` 与证据来源 |
| 命中明确疾病 | 用户声明明确疾病，或授权数据提示需进入不支持范围 | 请求生成普通精细训练方案 | 不生成针对疾病的诊断/治疗/康复方案；安全否决并转入 Spec 定义的专业咨询/非训练能力；第三方模型不接收疾病原文，除非字段级清单和独立授权允许；完整审计 |
| 命中术后康复 | 用户声明处于术后康复 | 请求生成、恢复或升级计划 | `REHAB-SPECIALIST`/安全规则否决通用方案；不得承诺康复效果；仅提供 Spec 允许的暂缓/转介/已审核替代；恢复须满足重新评估条件并生成新安全版本 |
| 历史确认过期或状态变化 | 过去存在通过回执，但有效期届满、规则版本变化、用户修改身份/健康状态或出现新疼痛 | 用户再次生成/重排/继续高风险训练 | 旧回执不可自动续作新确认；要求重新主动确认并重新安全评审；在完成前保持更严格状态；保留新旧版本因果链 |
| 第三方模型最小可见 | 适用范围确认已完成，任务需要云端第三方模型参与，且用户已给予该目的的独立 Consent | 系统组装模型输入 | 仅发送 Spec 字段清单中完成当前任务所需的最小、默认去标识化数据；默认不发送健康原文、知识/笔记原文、完整身份叙事；不得用于训练通用模型；接收方、目的或范围变化须重新授权；记录字段集合、接收方、用途和模型/规则版本 |
| 离线或依赖失败 | 确认页面离线，或安全/康复规则、第三方模型、同步服务超时/不可用 | 用户提交确认、请求生成、重排或继续训练 | 本地可记录待处理动作但不得伪造确认或审核通过；未完成准入时不生成普通精细训练方案；安全复评失败时保持暂停/更严格状态；展示可重试、保守替代、缓存新鲜度和影响；恢复后幂等处理并保留失败与重试审计 |
| 持续安全审查 | 用户已通过适用范围确认 | 后续出现疼痛、恢复异常、负荷异常、场景变化、数据过期或计划重大调整 | 仍执行安全/康复复评；准入回执不得当作安全通过证据；安全超时或依赖失败保持暂停/保守方案，不得默认放行；所有否决、替代、恢复均可审计 |

验收红线：上述任一命中、拒绝、无法确认、离线或依赖失败场景若仍能生成普通精细训练方案或继续原危险方案，均为 **P0 缺陷**；确认控件出现默认勾选、历史自动代填、用通用协议替代、缺少 Consent/审计，或把一次确认当作持续安全通过，也均为 **P0 缺陷**。

## 13. 当前结论

- **产品 PRD：Accepted。** 用户已确认 12 项 P0 产品决策；蓝图终局范围已映射到 PRD A–K、12 Agent、数据、非功能和核心旅程。
- **进入 Spec：允许。** 可据本矩阵编写页面、状态、数据对象、API/事件、权限、SLO 和 Given/When/Then 契约；v6 仍是高质量体能纵向体验证据，不是完整终局原型，六 Tab 仅是 IA 候选。
- **进入开发：禁止。** 第 10–12 节要求的公式、状态机、支持人群、主动高风险身份确认、Consent/审计、第三方模型字段级最小可见、同步合并、Agent 权限和异常验收未在 Spec 冻结并通过审查前，相关 Requirement 一律 `Spec Blocked`。
- **P0 安全红线：** 首发仅支持 18 岁以上普通健康成年人；训练方案生成前必须由用户以默认未选的主动动作完成适用范围确认；命中未成年人、孕期、急性疼痛、明确疾病、术后康复，或拒绝/无法确认时，不生成普通精细训练方案并转保守提示/专业转介；安全否决不可覆盖；一次确认绝不替代生成前、调整时和执行期的持续安全/康复审查。
- **QA 验收结论：产品 PRD Acceptance 门禁通过，允许进入 Spec；开发门禁不通过。** 12 项 P0 产品裁决已确认；仍存在必须在 Spec 归零的 P0 可测试性与实现契约缺口，Spec 未冻结并通过门禁前禁止开发。
