## 2. 已确认的 12 项 P0 产品决策

| ID | 产品决策 | 状态 |
|---|---|---|
| P0-01 | 用户目标、方向、行动、约束与证据是最高层主轴；只引导用户生成目标，不定义角色、身份或人格标签 | Changed by 2026-07-21 V6 locked verdict |
| P0-02 | 早期采用“体能单域深扎 + 融合可见”，体能是首发锚点但不是产品终局身份 | Accepted |
| P0-03 | 最多同时主动编排 3 个活跃成长域，每域可有多个目标；超限只建议聚焦，不静默删除 | Accepted |
| P0-04 | v6 六 Tab 仅为信息架构候选，不冻结为最终导航 | Accepted |
| P0-05 | 安全否决后不能继续原危险方案，且不能被用户、调性、编排器或其他 Agent 覆盖 | Accepted |
| P0-06 | 首发仅支持 18 岁以上普通健康成年人；训练方案前须由用户主动、默认未勾选确认不属于未成年人、孕期、急性疼痛、明确疾病、术后康复；命中、拒绝或无法确认时不生成普通精细训练方案，转保守提示/专业转介；确认不替代持续安全审查 | Accepted |
| P0-07 | Atlas 负责计划生成和普通自动调整，不设置逐项审批；必须解释原因、依据、差异与影响；安全规则独立且不可覆盖 | Changed by 2026-07-21 V6 locked verdict |
| P0-08 | 用户控制是否发起知识注入；发起后由 Atlas 决定知识是否采纳以及如何编排，解释理由但不要求逐条批准 | Changed by 2026-07-21 V6 locked verdict |
| P0-09 | Prime Pulse 可绕过、不是训练硬门槛；拆分长期目标势能、今日准备度、启动仪式；长按产品建议 500ms，提供按钮/辅助技术等效入口 | Accepted |
| P0-10 | 指标默认展示阶段、趋势、证据和数据充分度；只有条件充分时才显示精确百分比 | Accepted |
| P0-11 | 第三方模型仅获取当前任务所需的最小必要、默认去标识化数据，需独立 Consent，用户数据默认不用于训练通用模型 | Accepted |
| P0-12 | 12 Agent 是 12 类产品职责、权限和审查角色，不等于 12 个独立模型、聊天机器人或真人专家 | Accepted |

## 3. 信息架构候选

设计阶段已形成三套候选：

| 候选 | 一级导航 | 当前定位 |
|---|---|---|
| A 生命周期四 Tab | 今日 / 旅程 / 成长域 / 我的 | 推荐进入 Spec 与低保真验证 |
| B 身份主轴四 Tab | 今日 / 身份 / 计划 / 我的 | 身份显著度对照组 |
| C v6 收敛五 Tab | 今日 / 身体 / 训练 / 成长 / 我的 | 低迁移成本对照组 |

当前只推荐方案 A 进入 Spec 验证，**不视为最终路由合同**。冻结前必须完成单域/双域/三域、多目标、二级 IA、安全、权限、离线、返回路径和可用性验证。

## 4. Spec 必须归零的门禁

在 Spec 中，以下内容必须形成唯一、可测试、可追溯的合同：

1. 产品范围、版本边界和 Won't Have；
2. 页面、路由、一级/二级 IA 和全部状态；
3. 用户原话与约束、目标、计划、执行、成果、Consent、安全、同步等状态机；
4. API、领域事件、请求/响应、错误码和幂等键；
5. 数据对象、SQLite Schema、索引、生命周期、版本和删除语义；
6. 训练适用范围确认、持续安全审查、替代和复评；
7. 第三方模型字段级最小可见范围、接收方、目的、保留和删除规则；
8. 离线能力矩阵、同步合并、追加事实、冲突解决和时区语义；
9. 指标公式、样本、窗口、误差/置信表达、金标和禁止显示条件；
10. Agent 权限矩阵、SLO、超时、失败接管和 fail-closed；
11. 外部依赖的权限、数据质量、服务可用性和内容有效性四类降级；
12. Design Token、无障碍、性能和 Given/When/Then 验收。

任一相关项未冻结，该 Requirement 保持 `Spec Blocked`，不得进入开发。
