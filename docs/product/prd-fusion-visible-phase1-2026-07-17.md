## 1. 产品目标（3 个清晰、正交的目标）

- **G1 · 认知锚定（扭转归类）**：让用户进入首屏即感知"成长操作系统"而非"健身 App / 背单词工具"。通过首屏锚定"身份/目标"、三项融合机制作为一等公民可见，重构心智归类。
- **G2 · 机制可见（落地双轨）**：将派生语言流、专家门控💉、量化回流刻度在 Phase1 完整落地，并以"首屏即见、长在训练场景"的方式可见，验证"体能·语言·阅读在同一'想成为的自己'之下交叉融合"的产品主张。
- **G3 · 度量可信（杜绝伪量化）**：以代理指标族（M1–M4）替代"交叉次数/点击数/专家通过率"等伪量化口径，建立可验证、可审计的融合深度基线；所有指标仅作反映与护栏，不进入身份进度指数考核。

> 三轴正交：G1=感知层（用户怎么看）、G2=机制层（产品做了什么）、G3=度量层（怎么证明有效），互不替代。

## 2. 用户故事（3-5 个场景）

- **US-1 · 导入书后主动询问能否加入（💉触发，知识 Tab）**：作为读《运动解剖学》的体能用户，在知识 Tab 导入书后看到常驻💉入口与书卡「💉问专家」；点💉询问"能否加入计划"，专家给"能/不能"及理由（能则确认加入，不能也理解原因）。始终掌握决策权，无任何东西自动加入。
- **US-2 · 训练时顺带派生英语并可溯源（派生语言回流）**：作为想练扣篮又想聊 NBA 的用户，在弹跳训练卡看到动作名旁的领域英文术语与发音按钮（源：今日弹跳训练）；点「派生源」芯片可回溯父训练节点，确认英语是"训练长出来的"而非独立背单词。
- **US-3 · 查看自己的融合深度（量化回流查看）**：作为跨域用户，在首页看到"融合深度"刻度，理解这只是如实反映、不是考核，也不存在新的连续天数或待办压力。
- **US-4 · 笔记页就一条笔记问专家（💉触发，笔记页）**：在读书笔记里记了一句关键机制，长按该条笔记点「💉问这条」，专家判断它是否适用于当前训练目标并给理由；不适用则温和搁置，无惩罚。
- **US-5 · 训练后温柔接上同域书（💉软提示，训练回顾）**：完成训练后，回顾页 AI 识别到同域书，软提示"接上吗"；选"接上"则进入💉评估，选"不接"则提示消失，零后果。

## 3. 用户研究洞察（来自瑞思）

### 3.1 💉触点地图（零操作 = 零后果）

| 页面 / 情境 | 入口位置 | 触发路径 | 手势 | 反馈 |
|---|---|---|---|---|
| 知识 Tab（书籍列表/详情）主阵地 | ①右上常驻💉悬浮钮 ②书卡右下「💉问专家」微按钮 | 手动（主）；AI 自动识别（次） | 点💉→底部弹层确认；书封**长按**→「快问专家」 | 评估中（脉冲）→结果卡（能/不能/部分+理由） |
| 目标 Tab（目标详情页） | 「资料」区「＋关联知识」入口 | 手动 | 点「＋关联知识」→选书→自动对该目标💉评估 | 结果卡锚定目标下 |
| 训练 Tab（训练完成回顾页） | 回顾卡底部「💉关联你读过的书」软提示 | AI 识别（主）；手动（次） | AI 检测同域书→淡入软提示→点「接上」 | 接入后并入回顾与笔记；不接则消失，无惩罚 |
| 读书笔记场景 | ①单条笔记「💉问这条」②页底「💉整本评估」 | 手动（主） | **长按单条笔记**→「这条能否加入计划」 | 单条级判定，可单条采纳 |

> 铁律嵌入：💉从不自动加入计划、无红点/角标计数、决策权 100% 在用户。

### 3.2 「派生源」溯源芯片（R7 验收权威定义）
- 每条派生英语卡带芯片「📌 源：今日弹跳训练 · 落地缓冲」或「💉 源：财经书《XXX》· 第3章」。
- 点击芯片→跳转父节点（训练回放 / 书摘）；父节点类型：training_replay / book_note / note / goal。
- 训练 Tab 回顾页「今天顺带练了这些英语」卡（上滑展开、点术语发音）；知识 Tab 术语云；今日 Tab「今日融合」模块。
- 约束：英语无打卡/连续天数；无"今日英语未完成"警告。

### 3.3 专家理由卡统一组件（R6 验收权威定义）
- 容器：圆角卡，左 4px 色条（绿=能加入 / 琥珀=不能 / 双色=部分）。
- 头部：专家徽标（头像 + 领域标签）+ 加粗结论；正文 1–2 句陈述式非评判理由；可展开「依据 ▾」。
- 按钮：能加入→「加入计划」；不能加入→「稍后提醒我」「我知道了」；**绝不"放弃/失败/你不行"**。

### 3.4 铁律对齐与禁做清单
| 铁律 | 禁做清单 |
|---|---|
| 成果 > 压力 | 禁：英语打卡/连更、"今日英语未完成"警告、新待办计数、💉红点、指标显分数/排名 |
| 目标 > 打卡 | 禁：💉包装成打卡任务、知识注入以完成度考核 |
| 身份 > 意志力 | 禁："不能加入"措辞为"你不行/失败"、不采纳记失败 |
| 自适应 > 固定 | 禁：AI 提示强制加入、固定入口暗示"必须点" |
| 专业可靠 | 禁：理由空泛无依据、无徽标/领域标签、理由不可展开 |

### 3.5 今日融合轻模块 v2（基于数析指标契约）
- 数据契约：`{metric, domain, ratio, n, ci, signal, evidence[]}`；前端仅消费 `signal` + `evidence[]`。
- evidence = `{metric, edge_id, anchorA, anchorB, ts, strength?}`（一条 evidence = 一条实例边，直接渲染不聚合）。
- anchor = `{type, id, domain, ts, label?, ref?}`；type 枚举 7 类：derived_term / training_replay / note / goal / review / book_note / plan。
- 呈现：零数字/零排名/零连更；顶部信号行 + 派生链路卡（点击按 type 路由跳转）。信号色 emerging/active/sustained（低显著度，加图例区分💉卡语义）。
- 冷启动：n<5 或 CI 过宽 → signal=none → 模块显示"样本还太少，暂无融合回流"或整体不出现，**绝无"0 分"**。
- 单域安全：仅活跃域有链路；未用维度不呈现、不占位、不惩罚（权重=0）。

**差异化结论**：现有产品均在单域闭环内，或仅提供无训练闭环的知识容器。PrimeAtlas 的不可归类性来自三点——① 派生语言流长在训练卡（非独立 Tab，说服力最强）；② 💉专家门控是护城河（竞品无"能否加入计划"的专家判断）；③ 融合深度以代理指标可见刻度呈现（首日偏弱但必须可见）。

**反归类 5 建议（已落入 PRD 范围 R1–R4 + R16/R13）**：① 首屏锚定身份/目标；② 派生语言流默认开+长训练卡；③ 💉常驻+预置示例；④ 首页融合深度刻度；⑤ 示例目标强制跨域。

**R16 证据链修订（竞析反馈，主理人认可）**：R16 单项不足以构成"可见的反归类证据"，须与 R13 绑定为 M2「反归类证据链」——R16 首启播种跨域 + R13 每日回流融合证据 + R3/R1 互证，构成"首启即见、每日重温"的连续信号。

## 5. 数据依据（来自数析）

代理指标族作为验收口径（均含下游验证，防伪量化）；**Phase1 不进入身份进度指数**。

| 指标 | 定义 | 健康区间 | 埋点优先级 | 窗口 |
|------|------|---------|-----------|------|
| M1 派生语言回流率 | 派生条目 14 天内于下游场景被复用比例 | 15%–30% | **P0** | 派生后 14 天 |
| M2 知识注入采纳率 | 💉判定"可加入"后 7 天内实际纳入比例 | 40%–60% | **P0** | 判定后 7 天 |
| M3 读书→训练应用率 | 已加入书关联动作/术语在训练记录出现频率 | 20%–40% | **P1** | 加入后 30 天 |
| M4 跨域融合留存率 | 融合动作素材 30 天后仍被调阅比例 | ≥35% | **P2** | D0→D30 |

**伪量化黑名单（严禁作为成功口径）**：不数交叉次数、不数点击、不数生成卡/导入书数、不把专家通过率当成功、不把"活跃域数"当融合。

**北极星衔接**：M1–M4 为过程型/护栏探针，静默汇入；若未来晋升，仅 M3 可候选且权重 ≤10% + "融合探针"标签；首页刻度仅作反映非考核。

**呈现层契约（v2，与瑞思对齐）**：前端仅消费 `signal` + `evidence[]`；直接画 evidence 实例边、不聚合；`signal=none → evidence=[]` 冷启动分支；`strength` 阈值（后端用）：sim≥0.82 strong / 0.70–0.82 weak / <0.70 不计。

### 验收红线正式版（4 条，可判）

- **红线 1 — 派生语言流默认开启且长在训练卡**：合格=新用户首进训练卡即见派生内容、无关闭开关、不可跳过；不合格=存在关闭开关/仅"实验室"可选/默认关闭。
- **红线 2 — 💉常驻一等公民入口**：合格=主框架所有主路径页均可见💉、不可关闭/不可进"更多"；不合格=藏在二级页/提供"不再提示"/需主动开启。
- **红线 3 — 专家理由卡必现**：合格=每次💉必返结构化理由卡、无理由流程不允许结束；不合格=存在"询问即采纳无理由"通道。
- **红线 4 — 不引入新打卡/红点/压力（含 R13 约束）**：合格=无新打卡/连更/红点/催促、融合刻度不排名、R13 叙事化被动可见；不合格=出现红点/催促/排名字样、R13 为每日重温引入连续机制。

```
[AI 后台] 识别是否适用当前训练环境/状态
   ↓
[专家理由卡] 绿(能加入)+理由 → 用户「加入计划」；琥珀(不能)+保护型理由 → 搁置无惩罚；双色(部分) → 勾选段落加入
   ↓
内容 = 用户读书笔记 + AI 总结，挂到对应目标/域
```
红线：任何一步不允许"跳过💉静默加入"或"一键关闭💉"。

**流程 B：训练 → 派生英语 → 溯源**
```
[场景化训练] 弹跳训练（活跃域：体能/康复）
   ↓
[训练卡] 动作名旁自动渲染领域英文术语 + 发音（默认开，长卡内；派生源芯片标"源=今日弹跳训练"）
   ↓
[训练完成回顾] "今天顺带练了这些英语"卡 + 同域书软提示"接上吗"（不接则消失）
   ↓
点派生源芯片 → 回溯父节点；下游 14 天内复用 → 计入 M1
```

**关键 UI 组件规格**：训练卡（动作图示+中文名+领域英文+🔊，英文不喧宾夺主）；💉浮动入口（右下常驻圆钮，无红点无计数）；专家理由卡（4px 色条+徽标+结论+理由+可展开依据）；融合深度刻度（首页可视化趋势，仅反映无考核压力）；派生源芯片（小标签角标，点开抽屉式回溯）。

## 8. Non-goals（Phase1 明确不做什么）

- ❌ 不做独立语言域（英语仅从活跃领域派生，无独立 Tab/课程/词汇量体系）。
- ❌ 不做语言打卡/连续天数/词汇考核（严格对齐铁律）。
- ❌ 不做社交分享/排行榜/好友系统。
- ❌ 不做跨书批量自动加入（💉从不自动加入，每次须用户主动确认）。
- ❌ 不做融合深度复合指数（M1–M4 不进身份进度指数，仅 M3 未来可候选且 ≤10% + "融合探针"标签）。
- ❌ 不引入红点/待办/新连续天数（零操作=零后果）。
- ❌ 不做专家实时在线问答（专家理由卡为规则化/预生成组件，非真人对话）。
- ❌ 不做 Phase2 域扩展（音乐/烹饪等），Phase1 仅覆盖体能/康复/财经等已定义活跃域。
