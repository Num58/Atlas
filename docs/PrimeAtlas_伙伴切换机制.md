# PrimeAtlas 伙伴切换机制

## 核心原则

**数据是用户的，伙伴是服务者。切换伙伴 = 换个教练，你的训练记录、身体数据、读书笔记全部保留。**

## 切换流程

### 1. 触发入口
- "我的" → "伙伴设置" → "切换伙伴"
- 可选：在伙伴面板底部增加"换个伙伴"入口

### 2. 切换确认

```
┌──────────────────────────────────────────────┐
│  确认切换伙伴                                 │
│                                              │
│  当前：🔥 Alex                               │
│  切换至：⚡ 小野                              │
│                                              │
│  ✅ 你的所有数据都会保留：                     │
│     · 训练记录（186次）                       │
│     · 身体数据（体脂/1RM/PR）                 │
│     · 英语学习进度（LISA B1）                 │
│     · 读书笔记（42条）                        │
│     · 目标与里程碑                            │
│                                              │
│  🔄 小野会阅读你的数据，了解你的训练习惯        │
│                                              │
│  ❌ 不会迁移：Alex 和你之间的私密对话           │
│     （那些只属于你们的聊天记录）               │
│                                              │
│  [取消]              [确认切换]               │
└──────────────────────────────────────────────┘
```

### 3. "自来熟"机制

新伙伴接管后，通过以下方式快速建立连接：

**立即了解：**
- 读取用户的训练历史摘要（总次数、PR记录、训练偏好）
- 读取用户的身体数据（当前1RM、体脂率、损伤史）
- 读取用户的目标（篮球弹跳+15cm）
- 读取用户最近的训练状态（上周RPE趋势、完成率）

**首次见面表现：**
- 新伙伴不会说"你好，我是新来的"——那太生疏了
- 而是说"186次训练，深蹲PR 95kg，最近在冲弹跳。我看了你的数据，是个狠人。"
- 让用户感觉"这个伙伴好像认识我很久了"

**渐变过渡：**
- 切换后前3天：新伙伴更多参考数据说话（"你的数据显示..."）
- 3天后：逐渐发展出自己的风格（小野会说"兄弟"，Elena会说"研究表明"）
- 7天后：完全进入新伙伴的人格模式

## 数据所有权

| 数据类型 | 切换后保留 | 说明 |
|---------|:---:|------|
| 训练记录 | ✅ | 全部历史训练数据 |
| 身体数据 | ✅ | 1RM/体脂/PR/损伤史 |
| 英语学习进度 | ✅ | LISA评估/对话记录 |
| 读书笔记 | ✅ | 划线/批注/语音笔记 |
| 目标与里程碑 | ✅ | 目标进度/知识PR |
| 旧伙伴对话历史 | ❌ | 私密对话不迁移（隐私保护） |
| 旧伙伴的记忆 | ❌ | 如"你上次说膝盖不舒服"（这是伙伴个人记忆，不是用户数据） |
| 火焰连续天数 | ✅ | 连续训练天数不受影响 |

## 技术约束

- 用户数据与伙伴数据完全分离存储
- 伙伴只能读取用户数据摘要（不能读取旧伙伴的私密对话）
- 切换时生成一份"用户画像简报"供新伙伴初始化上下文
