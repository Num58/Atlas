## 1. Release 边界与技术裁决

### 1.1 V0.2 唯一范围

V0.2 是纯本地“方向—目标”闭环；`IdentityDraft`、`PortraitVersion` 等名称只为既有存储/API 兼容，不定义用户角色、身份或人格：

1. 游客在设备内创建、编辑、提交、确认或丢弃目标方向与现实约束草案；
2. 用户确认目标边界版本，选择 1–3 个活跃成长域；
3. 用户创建/确认/编辑/暂停/重校准/归档目标与里程碑；
4. 用户查看旅程总览、目标详情、目标边界版本历史和本机保存状态；
5. 所有写操作有 `owner_id`、版本、`operation_id`、事件、审计和本地幂等结果；物理表已裁决为本地 `operation_ledger`；
6. 正式云同步、HTTP、认证、第三方模型及系统数据连接均不进入 V0.2。

V0.2 一级入口冻结为两项：

| 入口 | V0.2 责任 | 允许对象 | 明确不显示 |
|---|---|---|---|
| 旅程 | 身份发现、画像确认、域选择、目标/里程碑、旅程总览 | `IdentityDraft`、`PortraitVersion`、`ActiveDomain`、`Goal`、`Milestone`、`EvidenceLink` | 今日行动、训练、Pulse、准备度、知识、PR |
| 我的 | 画像版本历史、本机数据状态、调性与基础治理说明 | `PortraitVersion`、本地保存/待处理状态、主体信息 | 登录、云同步、第三方授权、健康/日历/位置入口 |

终局一级入口冻结目标仍为四项：`今日 / 旅程 / 成长域 / 我的`。V0.2 的两入口是 Release 子集，不另造第二套对象真源；后续扩为四入口时，`旅程` 与 `我的` 的路由 ID、对象 ID 和返回语义保持兼容。

V6 只作为交互品质基线：继承层级、反馈、状态完整度、触控与动效品质；不继承六 Tab、演示数据、硬编码指标、训练范围、功能图标或任何 Release 外入口。

### 1.2 明确不做

- 不提供 HTTP Server、REST endpoint、OpenAPI、WebSocket、GraphQL 或远程 Job。
- 不提供云 pull/push、登录/注册、账号合并、第三方 AI、健康平台、日历、位置、通知、媒体、第三方内容。
- 不提供 Today、Plan、PlanAction、Execution、Feedback、Eligibility、Safety、Knowledge、PR、正式 Sync Conflict UI。
- 不以“未来会用”为理由创建可点击入口、网络 client、远端配置或半状态页面。
- 不复制 V6 六 Tab；不使用 emoji 作为功能图标；不混用图标库；不使用紫色到粉色渐变。

### 1.3 HTTP/OpenAPI N/A 的证明合同

`openapi.yaml` 对 V0.2 明确为 N/A，而不是缺失交付物。理由是本 Release 没有 HTTP 边界，契约形态是 Dart Use-case + Repository + SQLite Schema。出现任何 HTTP endpoint 即构成范围扩大，必须重新过 Release Freeze。

必须提供以下三类证据：

1. **静态零业务网络能力**：业务 `dependencies` 不直接声明 HTTP client/server 包；`lib/` 不 import `dart:io` 的 `HttpClient/Socket`、`package:http`、Dio、gRPC 或 WebSocket，不调用 `SvgPicture.network` 等网络 constructor。允许经依赖树与用途审计、且业务只走纯本地 API 的传递依赖；其存在不豁免 ThrowOnCall、HAR 与运行时业务请求为 0。
2. **运行时 throw-on-call**：对 `CloudSyncPort`、`RemoteAssistedDiscoveryPort`、`AuthPort`、`HealthPort`、`CalendarPort`、`LocationPort`、`ContentPort` 使用测试专用 `ThrowOnCall*`，任一方法调用立即抛出 `UnexpectedExternalCall` 并记录 adapter/method。
3. **E2E 零调用**：新建到完成身份—目标闭环，所有 ThrowOnCall 计数为 0；Android/iOS 测试代理抓包为 0 个应用业务请求。系统字体/系统组件流量不得作为应用请求豁免，测试设备应断网复跑同一闭环。

生产 `NoopSyncAdapter` 可以返回“禁用”，但测试必须注入 ThrowOnCall，不能用静默 no-op 证明没有调用。

### 1.4 技术选型矩阵

评分：5 为最符合 V0.2；权重为团队熟悉度 30%、学习成本 25%、生态成熟度 20%、部署成本 20%、远期扩展 5%。

| 方案 | 组成 | 熟悉度 | 学习成本 | 生态 | 部署 | 扩展 | 加权分 | 裁决 |
|---|---|---:|---:|---:|---:|---:|---:|---|
| A | Flutter + Riverpod + GoRouter + `sqlite3` + 手写 Repository/UoW | 5 | 5 | 4 | 5 | 3 | 4.65 | 选择；与锁文件和 S0 一致，改动最小 |
| B | Flutter + Riverpod + GoRouter + Drift | 3 | 3 | 5 | 4 | 4 | 3.65 | 不选；迁移/生成层会扩大 Freeze 缺口 |
| C | Flutter + Riverpod + GoRouter + Isar/Hive | 2 | 3 | 4 | 4 | 3 | 3.15 | 不选；关系、复合所有权和事务合同不如 SQLite 直接 |

架构分层与依赖方向：

```text
lib/app/                     表现层：页面、路由、语义、Riverpod 装配
    | 只能向下
    v
lib/application/             用例层：命令/查询、事务编排、错误映射
    | 只能向下
    v
```

执行期目录硬规则：入口只装配；页面不得直接 SQL；Repository 不含业务决策；每文件一个主职责；源文件不超过 300 行；跨功能调用通过用例/领域 Port，不直连其他 Repository。

## 3. V0.2 SQLite 物理 Schema（baseline 已固化，运行时集成待实现）

### 3.1 全局物理规则

- SQLite：`PRAGMA foreign_keys=ON; PRAGMA journal_mode=WAL; PRAGMA synchronous=FULL; PRAGMA busy_timeout=1000;`。
- **已关闭 `DATA-ID-01`**：采用 RFC 9562 UUIDv7，规范小写带连字符 TEXT；精确锁定 `uuid 4.6.0`，固定时钟、同毫秒批量、时钟回退和 SQLite 排序 fixture 已通过。`owner_id`、`operation_id` 与事件/聚合 ID 使用同一格式。
- 时间统一 UTC epoch 微秒 `INTEGER`，涉及用户发生时间另存 IANA `timezone_id` 与 `timezone_offset_min`。
- 布尔值使用 `INTEGER NOT NULL CHECK(value IN (0,1))`；枚举使用 `TEXT CHECK(...)`。
- JSON 存 `TEXT`，写前 canonical JSON；另存 `payload_hash`（SHA-256 小写十六进制）。
- 每个用户数据表均含 `owner_id`；每个父表声明 `UNIQUE(owner_id,id)`；子表使用复合外键 `(owner_id,parent_id)`，从物理层阻断跨 owner 关联。
- 所有 Repository 方法首参必须是 `OwnerId ownerId`；禁止“先按 id 查询再在 Dart 比 owner”。
- V0.2 不提供删除 UI；归档是业务状态，不等于删除。**已裁决 `DATA-TS-01=不纳入`**：baseline 不建立 tombstone 表、Repository 不增加 tombstone 过滤、防复活错误或删除传播语义。**`DATA-RET-01=N/A`**，不得加入保留期或清理任务；后续 Release 若引入删除/备份恢复，必须新 ADR 与迁移。
- 每表均有 `created_at_us`、`updated_at_us`；追加表的 `updated_at_us=created_at_us`，不得 UPDATE 覆盖事实。

字段为 Frozen Envelope：`id TEXT PRIMARY KEY, owner_id TEXT NOT NULL, event_type TEXT NOT NULL, schema_version INTEGER NOT NULL, aggregate_type TEXT NOT NULL, aggregate_id TEXT NOT NULL, aggregate_version INTEGER NOT NULL, operation_id TEXT NOT NULL, causation_id TEXT, correlation_id TEXT, source_type TEXT NOT NULL, actor_type TEXT NOT NULL, actor_ref_id TEXT, device_id TEXT NOT NULL, device_seq INTEGER NOT NULL, occurred_at_us INTEGER NOT NULL, timezone_id TEXT NOT NULL, timezone_offset_min INTEGER NOT NULL CHECK(timezone_offset_min BETWEEN -840 AND 840), recorded_at_us INTEGER NOT NULL, payload_json TEXT NOT NULL, payload_hash TEXT NOT NULL, created_at_us INTEGER NOT NULL, updated_at_us INTEGER NOT NULL, UNIQUE(owner_id,id), UNIQUE(owner_id,device_id,device_seq), FOREIGN KEY(owner_id,device_id) REFERENCES devices(owner_id,id), FOREIGN KEY(owner_id) REFERENCES subjects(id) ON DELETE RESTRICT`。

索引：`idx_events_owner_aggregate(owner_id,aggregate_type,aggregate_id,aggregate_version)`；`idx_events_owner_type_time(owner_id,event_type,occurred_at_us DESC,id DESC)`；`idx_events_owner_operation(owner_id,operation_id)`。

#### `audit_records`

`id TEXT PRIMARY KEY, owner_id TEXT NOT NULL, operation_id TEXT NOT NULL, event_type TEXT NOT NULL, object_type TEXT NOT NULL, object_id TEXT NOT NULL, actor_type TEXT NOT NULL, before_version INTEGER, after_version INTEGER, reason_code TEXT, metadata_json TEXT NOT NULL DEFAULT '{}', occurred_at_us INTEGER NOT NULL, created_at_us INTEGER NOT NULL, updated_at_us INTEGER NOT NULL, UNIQUE(owner_id,id), FOREIGN KEY(owner_id) REFERENCES subjects(id) ON DELETE RESTRICT`。

索引：`idx_audit_owner_object(owner_id,object_type,object_id,occurred_at_us DESC,id DESC)`；`idx_audit_owner_operation(owner_id,operation_id)`。

#### `operation_ledger`（已裁决纳入；`DATA-OUTBOX-01=纳入`）

纯本地 V0.2 以独立物理账本处理崩溃重试去重，原子记录 `operation_id/payload_hash/result_json`。它不代表云队列，不允许网络 worker、pull/push、cursor 或 `syncing/synced` 语义。字段：`id TEXT PRIMARY KEY, owner_id TEXT NOT NULL, operation_id TEXT NOT NULL, device_id TEXT NOT NULL, device_seq INTEGER NOT NULL, operation_type TEXT NOT NULL CHECK(operation_type IN ('create','update','archive','restore')), entity_type TEXT NOT NULL, entity_id TEXT NOT NULL, base_version INTEGER, payload_json TEXT NOT NULL, payload_hash TEXT NOT NULL, result_json TEXT, state TEXT NOT NULL CHECK(state IN ('committed','rejected')), error_code TEXT, created_at_us INTEGER NOT NULL, updated_at_us INTEGER NOT NULL, UNIQUE(owner_id,id), UNIQUE(owner_id,operation_id), UNIQUE(owner_id,device_id,device_seq), FOREIGN KEY(owner_id,device_id) REFERENCES devices(owner_id,id), FOREIGN KEY(owner_id) REFERENCES subjects(id) ON DELETE RESTRICT`。`operation_type` 不含 tombstone。

索引：`idx_operation_ledger_owner_state_seq(owner_id,state,device_seq)`；`idx_operation_ledger_owner_entity(owner_id,entity_type,entity_id,created_at_us DESC)`。不得出现 `local_pending/syncing/synced` 云队列状态或用户文案；正式云状态不属于 V0.2。

#### `tombstones`（V0.2 明确排除）

`DATA-TS-01=不纳入`：baseline 无表、无索引、无 Repository 过滤、无防复活错误、无删除/恢复入口。`DATA-RET-01=N/A`：不定义 `retain_until_us`、默认天数或清理任务。

### 3.4 Tombstone 排除、迁移与回滚

- V0.2 只支持归档和画像历史恢复，不提供用户删除；operation ledger 不允许 `tombstone` operation。
- 后续 Release 若引入删除或旧备份恢复，必须先提交新 ADR、forward migration、签字数据保留政策和防复活 fixture，不得回写 V0.2 baseline。
- 迁移文件采用不可变编号：`0001_baseline.sql`、`0002_*.sql`；V0.2 baseline 已落盘于 `lib/infrastructure/storage/sqlite/migrations/0001_baseline.sql`，schema snapshot 与可重复机械测试已落盘。`schema_migrations.checksum` 防止已发布迁移被改写；checksum 写入与校验仍由待实现 migration runner 承担。
- 启动迁移流程：完整性检查 -> 复制 `primeatlas.db.pre-migration-{version}` -> `BEGIN IMMEDIATE` -> 逐迁移 -> `PRAGMA foreign_key_check` -> `COMMIT` -> 再开放写路径。
- 任一步失败 `ROLLBACK`，原 DB 与备份均保留，应用进入只读恢复页，错误 `dependency_unavailable/local_migration_failed`；禁止删库重建。
- 数据降级回滚不承诺自动执行 down migration。应用回滚只能打开其支持的 schema 版本；超出则只读并提示使用兼容版本。修复采用 forward-only 新迁移。

## 4. Dart Use-case、事务、错误、幂等与兼容

### 4.1 统一签名

```dart
Future<Result<T, DomainFailure>> execute<T>(
  OwnerId ownerId,
  OperationId operationId,
  Command<T> command,
);
```

所有变更 Command 必含 `baseVersion`（创建除外）、`deviceId`、`occurredAtUs`、`timezoneId`、`timezoneOffsetMin`。Query 必含 `ownerId`；列表使用 `(updated_at_us,id)` keyset cursor，默认 `limit=20`、最大 `100`。

### 4.3 错误码补齐

V0.2 Release 允许的确定性错误码：

- 通用：`validation_failed`、`not_found`、`invalid_state_transition`、`version_conflict`、`idempotency_key_reused`、`account_ownership_mismatch`、`data_insufficient`、`dependency_unavailable`。
- 本地存储子码：`local_store_readonly`、`local_store_full`、`local_store_busy`、`local_store_corrupt`、`local_migration_failed`、`local_transaction_failed`。
- 业务：`domain_focus_required`、`impact_confirmation_required`、`change_classification_required`、`offline_pending`、`unexpected_external_call`。V0.2 不定义 tombstone 或防复活错误码。

`retryable=true` 仅用于 `local_store_busy`（内部最多3次、总计不超过1秒）及用户释放空间后的 `local_store_full` 手动重试。`local_store_corrupt/local_store_readonly/local_migration_failed/local_transaction_failed` 不得自动重试；校验、owner、幂等键复用和状态错误均不可盲重试。所有读取与写入的 SQLite 故障只能映射上述六个本地存储子码。

### 4.4 幂等与版本兼容

- `(owner_id,operation_id)` 永久保留到本地对象生命周期结束；同 payload hash 返回 `result_json`，不再发事件；不同 hash 返回 `idempotency_key_reused`。
- 业务聚合 UPDATE 使用 `WHERE owner_id=? AND id=? AND version=?`；影响行数不是 1 即回滚并返回 `version_conflict`。
- `schema_version=1` 的事件读写器必须忽略未知可选字段，拒绝未知必需字段；payload 只允许追加可选字段的兼容变更。
- enum 新值属于不兼容变更，必须新增 reader 分支和 migration；不得静默落入默认成功状态。
- V0.2 数据库支持窗口：当前 schema 与前一 schema 可升级；更高 schema 只读。事件 writer 只写 v1，reader 支持 v1。
- 后续 HTTP 若进入 Release，必须另起 `/api/v1/` 和 `openapi.yaml`；不得把本地 Dart 签名直接宣称为 REST 合同。

## 5. Flutter Lucide SVG 唯一方案决策门

项目总监已裁决方向：选择 I-B“官方 Lucide SVG upstream 固定 tag/commit + 仓库 allowlist 资产 + 单一 `flutter_svg` 渲染器”。当前 `pubspec.lock` 无 Lucide/SVG 依赖，故 ADR-009 状态为 `Direction Accepted / Pending Verification`，不是 Frozen；`ICON-DEC-01` 仅关闭方向选择，包版本、license、兼容性实测未通过前 UI 实现仍为 `B0`。

以下保留候选对比作为决策证据，不允许团队成员改选或混用：

| 候选 | 方案 | 优点 | 主要风险 |
|---|---|---|---|
| I-A | 锁定一个维护中的 Flutter Lucide package，使用其矢量 Widget | 名称映射直接、无资产生成 | 是否为 SVG path、版本/许可证/Tree shaking 需实测 |
| I-B | 官方 Lucide SVG 源 + 仓库内 allowlist 资产 + `flutter_svg` 单一渲染器 | 真 SVG、资产可审计、版本可固定 | 需要生成/校验脚本与许可证归档 |
| I-C | 自建 Dart Path 转换 | 依赖少 | 自研维护和可访问性成本高，不适合 MVP |

I-B 已是唯一方向；只有以下验证全部通过并写入 ADR-009 后才能从 Pending Verification 升为 Frozen：

1. 记录并校验 Lucide upstream tag/commit、`flutter_svg` exact lock、实际 SPDX/license 文件与图标 allowlist；不得在核验前预写许可证结论；
2. 只允许 `AppIcon` 组件访问 SVG，业务页面禁止直接 `SvgPicture`、`Icon(Icons.*)` 或第二图标包；
3. 尺寸只允许 16/20/24，颜色只从语义 Token 获取；功能图标必须有语义标签，装饰图标从语义树排除；
4. golden 覆盖 Android/iOS、1.0/1.3/2.0 字体倍率、浅/深色、高对比；像素差阈值由 QA 在 ADR 附件固定；
5. 静态扫描功能图标中 emoji 数为 0、Material/Cupertino 功能图标引用数为 0、非 allowlist SVG 数为 0；
6. 在 Flutter `3.44.2` / Dart `3.12.2` 上完成 resolve、build、tree shaking、Android/iOS 渲染和未知 icon key fail-fast 测试，不显示空白或默认问号。

### 8.2 测试数据隔离

- 每个 unit test 使用新的 in-memory Repository，不共享 singleton。
- 每个 SQLite integration test 使用唯一临时目录/DB 文件，test teardown 关闭连接并删除目录；禁止访问开发者真实 DB。
- `owner_a`、`owner_b` 固定为不同的稳定 ID fixture（具体算法随 `DATA-ID-01`）；每个 Repository 至少验证 A 不能读/写/关联 B。
- 时间、ID generator、设备序列、磁盘/VFS、network Port 全部依赖注入；禁止依赖测试执行顺序和真实当前时间。
- golden、E2E 使用脱敏合成数据，不包含真实身份/健康文本；失败日志不得打印 payload 原文。
- 并发/重试测试用 barrier 控制顺序，不用不稳定 sleep。

### 8.3 必须自动化的故障注入

| 注入 | 预期机械断言 |
|---|---|
| `SQLITE_FULL` | 身份确认四件套 0 条增量；输入仍在内存；返回 `local_store_full` |
| `SQLITE_BUSY/LOCKED >1s` | 最多3次后回滚；版本不前进；返回 `local_store_busy` |
| 迁移第 N 条异常 | schema/data checksum 与迁移前一致；备份存在；只读恢复 |
| `foreign_key_check` 失败 | 不 commit，不开放写路径 |
| 同 operation+同 payload | 原结果相等；事件/audit/已批准的幂等 ledger 各只有1份 |
| 同 operation+不同 payload | 第二次 `idempotency_key_reused`；无副作用 |
| stale `baseVersion` | `version_conflict`；不自动最后写赢 |
| cross-owner id/FK | `account_ownership_mismatch`；两方数据 checksum 不变 |
| tombstone/旧备份防复活 | V0.2 N/A；断言 baseline 无 tombstone 表且无删除/备份恢复入口 |
| N/A adapter 任意调用 | 立即 `UnexpectedExternalCall`；测试失败并给 method 名 |
| 时钟倒退/时区变化 | device_seq 不回退；保留 occurred/recorded/timezone |
| event subscriber 抛错 | 已 commit 事实保持；其他 subscriber 可继续；错误可审计 |
