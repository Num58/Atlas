# PrimeAtlas V0.2 可执行测试追踪合同 v1.0

> 文档角色：V0.2「身份—目标闭环」Requirement 级测试追踪与 P0 故障注入合同。  
> 依据：`primeatlas-spec-v1.0.md`、`primeatlas-spec-architecture-input-v1.0.md`、`primeatlas-spec-design-input-v1.0.md`、`primeatlas-spec-qa-input-v1.0.md`。  
> 范围：仅 V0.2；不修改主 Spec，不引入计划、训练执行、Pulse、完整云同步或第三方模型敏感数据发送。  
> 状态：**Test Traceability Draft / NOT SPEC READY / NOT DEV READY**。  
> 质量规则：P0 未决项或 P0 缺陷不为 0，不得开发；尚需真实低保真验证的条目只标 `EVIDENCE_REQUIRED`，不得伪造通过。

### 1.2 状态词

| 状态 | 含义 |
|---|---|
| `CONTRACT_DEFINED` | 合同已能静态审查，但不代表实现或测试已经通过 |
| `AUTOMATABLE` | 可由单元/集成/E2E 自动验证，仍需开发实现 |
| `EVIDENCE_REQUIRED` | 必须由真实低保真/辅助技术任务产生证据，不得填写模拟通过率 |
| `SPEC_BLOCKED` | 缺少唯一字段、接口、错误恢复、页面或规则合同，禁止开发对应 Requirement |
| `WONT_HAVE_V02` | V0.2 明确不提供，不得出现半成品入口 |

### 1.3 ID 与双向追踪规则

- Requirement：`V02-Rxx`；状态转换：`ST-xxx`；事件：`EV-xxx`；Use-case/API：`UC-xxx`；Repository：`RP-xxx`；页面状态：`PS-xxx`；夹具：`FX-xxx`；故障注入：`FI-xxx`；测试：`T-V02-xxx`；低保真证据：`LF-V02-xxx`。
- 每个 `V02-Rxx` 必须至少映射 1 个正向测试和 1 个异常/非法测试。
- 每个合法/非法状态转换、事件、Use-case、Repository、页面状态必须被至少 1 个 Test ID 引用。
- 每个 Test ID 必须反向引用至少 1 个 Requirement；孤立测试、孤立合同 ID 和重复 ID 均为门禁失败。
- 本文的 API 名称锁定领域语义，不擅自冻结尚未裁决的 Dart/HTTP 签名、事务或网络协议。
