### 1.2 状态词

| 状态 | 含义 |
|---|---|
| `CONTRACT_DEFINED` | 合同已能静态审查，但不代表实现或测试已经通过 |
| `AUTOMATABLE` | 可由单元/集成/E2E 自动验证，仍需开发实现 |
| `EVIDENCE_REQUIRED` | 必须由真实低保真/辅助技术任务产生证据，不得填写模拟通过率 |
| `SPEC_BLOCKED` | 缺少唯一字段、接口、错误恢复、页面或规则合同，禁止开发对应 Requirement |
| `WONT_HAVE_V02` | V0.2 明确不提供，不得出现半成品入口 |
