# PrimeAtlas V0.2 身份渐进追问数据合同

> 版本：v1.0 Frozen Candidate for V0.2 Readiness  
> 日期：2026-07-20  
> 范围：V0.2 的 A1 身份自由表达、渐进追问、草案生成与失败接管  
> 上位合同：`primeatlas-spec-v1.0.md`、`primeatlas-spec-architecture-input-v1.0.md`  
> 不包含：正式云同步、第三方模型、目标/计划生成、健康数据、知识内容、开放聊天  
> 核心红线：**第三方模型字段白名单未 Frozen 前，任何身份原文、身份摘要或身份推断均不得发送给第三方模型。**

---

## 0. 裁决摘要

1. V0.2 身份发现的**状态编排、问题阶段、草案落库、停止/跳过、版本和确认全部在设备端执行**。
2. 设备端的本地模板/规则路径是完整最低体验，不依赖网络或模型，能从自由表达走到可编辑 `IdentityDraft`。
3. 可选的语义辅助只允许调用 **PrimeAtlas 自有受控服务**；该服务只返回候选追问或候选摘要，无 Repository 写权限、无画像确认权、无目标/计划写权限。
4. PrimeAtlas 服务只可接收本合同白名单中的当前轮文本和有界候选摘要；不得接收完整历史原文、账号直接标识或无关数据。
5. V0.2 禁止 PrimeAtlas 服务把请求继续转发给第三方模型。用户同意不能覆盖此产品级禁用边界。
6. 可选服务失败、超时、拒权、撤回、限流或返回无效结果时，设备端立即接管为本地模板/手动编辑；用户输入不得丢失，未完成推断不得写正式画像。
7. 所有远程辅助结果必须绑定 `draft_id + base_version + operation_id`；草案版本已变化的迟到结果必须丢弃并审计，不能覆盖用户后续编辑。

### 0.1 合同状态

| 合同项 | V0.2 状态 |
|---|---|
| 设备端流程编排与本地落库 | `Frozen` |
| 本地模板/规则最低体验 | `Frozen` |
| PrimeAtlas 自有服务允许字段 | `Frozen` |
| PrimeAtlas 自有服务具体部署与模型实现 | `Implementation Deferred` |
| 第三方模型调用 | `Won't Have / Hard Disabled` |
| 第三方模型身份字段白名单 | `Spec Blocked` |
| 正式云同步与跨端身份发现 | `Won't Have for V0.2` |

## 1. 执行位置与权限边界

### 1.1 职责矩阵

| 职责 | 设备端 | PrimeAtlas 自有服务 | 第三方模型 |
|---|---:|---:|---:|
| 创建/更新 `IdentityDraft` | 是，唯一写入方 | 否 | 否 |
| 选择发现阶段 | 是，唯一状态机所有者 | 可返回建议，不可推进状态 | 否 |
| 显示本地追问模板 | 是 | 否 | 否 |
| 生成候选下一问 | 可使用本地模板 | 可选，只返回候选 | 禁止 |
| 生成候选身份摘要 | 可由用户手工编辑/本地规则产生 | 可选，只返回候选 | 禁止 |
| 读取完整历史原文 | 是 | 否 | 否 |
| 读取当前轮自由文本 | 是 | 仅满足 Consent 与 Manifest 时 | 否 |
| 写 `PortraitVersion` | 仅用户显式确认的设备端事务 | 否 | 否 |
| 写 `PortraitCandidate` | 设备端可将服务输出落为候选 | 否 | 否 |
| 确认正式画像 | 仅设备端处理 `user_confirm` | 否 | 否 |
| 生成目标/计划 | 不属于本合同 | 否 | 否 |
| 继续向其他接收方转发 | 不适用 | 禁止 | 不适用 |

### 1.2 设备端最小流程

```text
用户自由输入
  → 设备端创建/更新 IdentityDraft
  → 设备端选择：本地模板 或 已授权的 PrimeAtlas 辅助
  → 候选下一问/候选摘要
  → 用户继续、跳过、停止或编辑
  → pending_confirmation
  → 仅 user_confirm 创建 PortraitVersion
```

硬规则：

- `system_auto`、服务完成、追问轮数或数据充分度均不能触发 `confirmed`。
- 建议 3–6 轮不是门槛；用户可在任意轮次停止并编辑现有草案。
- PrimeAtlas 服务返回失败不得使 `IdentityDraft` 进入错误终态；状态保持 `draft`。
- 本地模板不是“错误提示页”，而是功能完整的最低辅助体验。

## 3. 明确禁止发送的数据

### 3.1 PrimeAtlas 服务也不可见

- `owner_id`、账号 ID、姓名、手机号、邮箱、推送 token；
- `device_id`、广告标识、精确 IP 派生位置、精确坐标；
- 完整身份发现历史问答、被跳过问题的原始答案；
- 健康、伤病、疾病、孕期、疼痛、睡眠、HRV 等原文或序列；
- 日历标题、参与人、地点原文；
- 笔记、书摘、文件、相册、麦克风或摄像头内容；
- 其他目标、计划、执行、反馈、画像历史全集；
- Consent 回执全文、Audit 日志、模型私有推理链。

### 3.2 第三方模型硬禁用

V0.2 中，下列条件即使存在也**不能**授权第三方调用：

- 用户勾选了通用 AI 同意；
- PrimeAtlas 服务不可用；
- 用户希望“更智能”；
- 本地模板问题不足；
- 第三方模型声称零保留或不训练；
- 只发送当前轮文本或身份摘要。

只有未来同时满足以下条件，才能从 `Won't Have` 转为候选：

1. 第三方接收方与模型版本 Frozen；
2. 身份字段级白名单、转换、保留与删除时限 Frozen；
3. 独立第三方模型 Consent Frozen；
4. 第三方不用于通用模型训练的可验证合同 Frozen；
5. 日志脱敏、故障注入与 `third_party_payload_blocked` 验收通过。

## 4. Consent 与 Manifest

### 4.1 Consent 目的

| 处理路径 | Consent 要求 | 拒绝后的最低体验 |
|---|---|---|
| 纯设备端自由表达/本地模板 | 不要求远程处理 Consent | 完整可用 |
| PrimeAtlas 自有服务语义辅助 | 独立目的 `identity_assisted_discovery_first_party` | 回退本地模板/手动编辑，无惩罚 |
| 第三方模型 | V0.2 不提供可授权入口 | 不发送；本地/第一方最低体验继续 |

第一方辅助 Consent 最少记录：

- `purpose_code=identity_assisted_discovery_first_party`；
- `data_categories=[current_turn_identity_text, bounded_identity_summary]`；
- `scope=identity_discovery_current_draft`；
- `recipient_type=primeatlas_service` 与固定 `recipient_id`；
- `policy_version`、`retention_policy_id`、`granted_at_us/valid_until_at_us/revoked_at_us`；
- 用户显式动作证据。

撤回后：

- 立即停止新请求；
- 已在途响应不得应用；
- 本地原始草案保留，不因撤回丢失；
- 生成撤回传播审计；
- 服务端既有请求数据按已 Frozen 保留策略删除。具体绝对删除 SLA 未 Frozen 时，第一方辅助不得进入 Dev Ready。

### 4.2 每次远程请求的 `input_manifest`

| 字段 | 必填 | 约束 |
|---|---:|---|
| `purpose_code` | 是 | 固定第一方身份辅助目的 |
| `recipient_type` | 是 | 固定 `primeatlas_service` |
| `recipient_id` | 是 | 不允许运行时改为第三方 |
| `service_id/service_version` | 是 | 可审计执行者 |
| `visibility_policy_version` | 是 | V0.2 第一方字段白名单版本 |
| `consent_receipt_id` | 是 | 当前有效 |
| `field_paths_sent[]` | 是 | 必须是 2.2/2.3 的子集 |
| `transform_applied[]` | 是 | `omit/pseudonymize/summarize/categorize/redact` |
| `source_record_ids[]` | 是 | 内部引用，不含原文 |
| `retention_class` | 是 | 当前第一方辅助保留等级 |
| `onward_transfer_allowed` | 是 | MUST 为 `false` |
| `general_model_training_allowed` | 是 | MUST 为 `false` |
| `payload_hash` | 是 | 与请求一致 |
| `created_at_us` | 是 | UTC 微秒 |

Manifest 校验顺序：有效 Consent → recipient 第一方校验 → 字段白名单 → 转换校验 → payload hash → 发送。

任何失败都不得发送。字段越界或第三方路由返回 `third_party_payload_blocked`；Consent 问题返回 `consent_required/consent_revoked`。

## 5. 服务结果应用与并发规则

1. 响应必须匹配 `request_id + operation_id + base_version`。
2. 当前 `IdentityDraft.version != response.base_version` 时，结果标记 stale，记录 `assisted_discovery_result_discarded`，不得自动合并。
3. 响应 `valid_until_at_us` 已过期时不得应用；可以由用户重新发起或直接使用本地模板。
4. 服务响应只能生成 `IdentityDraft` 新版本或 `PortraitCandidate`；不能生成 `PortraitVersion`。
5. 用户在等待期间的编辑优先于服务结果；不得用时间戳 last-write-wins。
6. 相同 `operation_id + payload_hash` 重放返回原结果；相同 key 不同 payload 返回 `idempotency_key_reused`。
7. 服务返回未知字段、无效 enum 或缺少 `service_policy_version` 时整体拒绝，不做部分静默应用。

## 6. 模型/服务失败接管

| 场景 | 错误码 / 原因码 | 是否自动重试 | 接管行为 | 用户可见结果 | 审计事件 |
|---|---|---:|---|---|---|
| 未授权第一方辅助 | `consent_required` | 否 | 本地模板 | 可继续，不弹强迫授权 | `assisted_discovery_degraded` |
| Consent 已撤回/过期 | `consent_revoked` | 否 | 停止在途结果，本地模板 | 提示已切换本地模式 | `consent_revoked`、`assisted_discovery_degraded` |
| 离线 | `offline_pending` | 否 | 不排队发送身份原文；直接本地模板 | 输入已保存在本机 | `assisted_discovery_degraded` |
| P95 10 秒未返回 | `dependency_unavailable/assisted_discovery_timeout` | 最多一次，且不阻塞 UI | 立即本地模板；迟到结果按版本校验 | 可继续或稍后重试 | `assisted_discovery_timed_out` |
| 429 限流 | `dependency_unavailable/assisted_discovery_rate_limited` | 仅尊重 `retry_after` 的一次用户可选重试 | 本地模板 | 不显示失败羞辱 | `dependency_failure_observed` |
| 5xx/连接重置 | `dependency_unavailable/assisted_discovery_service_unavailable` | 最多一次，同 operation_id | 本地模板 | 草案不丢失 | `dependency_failure_observed` |
| 服务响应 Schema 非法 | `validation_failed/assisted_discovery_response_invalid` | 否 | 整体丢弃，本地模板 | 可继续编辑 | `assisted_discovery_result_rejected` |
| 结果已过期/版本不匹配 | `version_conflict/assisted_discovery_result_stale` | 否 | 丢弃，不覆盖用户编辑 | 不强制打断；可重新请求 | `assisted_discovery_result_discarded` |
| 字段超白名单/第三方路由 | `third_party_payload_blocked` | 否 | 请求发送前阻止，本地模板 | 告知远程辅助不可用，不暴露安全细节 | `visibility_payload_blocked` |
| 本地持久化失败 | `dependency_unavailable/local_store_write_failed` | 否自动循环 | 保留当前内存输入，禁称“已保存”；提供复制/重试 | 明确未保存风险 | 恢复后补记 `local_persistence_failed` |

### 6.1 重试规则

- 自动重试不得超过一次，不得延长用户进入本地模式的时间。
- 重试复用相同 `operation_id`；网络层可使用新 `request_id`，但必须通过 `causation_id` 关联原请求。
- `validation_failed`、Consent、字段越界、第三方禁用、版本冲突不重试。
- 用户继续编辑后，旧请求的任何重试与迟到响应均失效。

## 7. 领域事件、API 与 Repository

### 7.1 V0.2 事件

- `identity_draft_created`
- `identity_draft_updated`
- `identity_draft_submitted`
- `identity_draft_confirmed`
- `identity_draft_discarded`
- `identity_discovery_answer_recorded`
- `assisted_discovery_requested`
- `assisted_discovery_completed`
- `assisted_discovery_degraded`
- `assisted_discovery_timed_out`
- `assisted_discovery_result_rejected`
- `assisted_discovery_result_discarded`
- `visibility_payload_blocked`
- `portrait_candidate_created`
- `portrait_version_activated`

事件使用主 Spec §7.1 Envelope；payload 不记录身份原文，只记录字段路径、hash、版本、错误/降级码和结果引用。

### 7.2 Use-case 语义

- `create_identity_draft(input, operation_id)`
- `update_identity_draft(draft_id, patch, base_version, operation_id)`
- `record_identity_discovery_answer(draft_id, answer, base_version, operation_id)`
- `request_assisted_discovery(draft_id, requested_output, base_version, operation_id)`
- `apply_assisted_discovery_result(draft_id, result_id, base_version, operation_id)`
- `continue_with_local_discovery(draft_id, operation_id)`
- `submit_identity_draft(draft_id, base_version, operation_id)`
- `confirm_identity_draft(draft_id, base_version, user_confirmation, operation_id)`
- `discard_identity_draft(draft_id, base_version, operation_id)`

最终 Dart/HTTP 签名与事务实现由 Release Architecture 冻结；上述状态、权限、数据边界和失败语义不得改变。

### 7.3 Repository 权限

- `IdentityRepository` 是 `IdentityDraft/PortraitCandidate/PortraitVersion` 的唯一持久化入口。
- 远程服务不得直接访问 Repository。
- `ConsentRepository` 提供第一方辅助目的的有效回执。
- `VisibilityPolicyRepository` 在发送前校验字段集合。
- `AuditRepository` 只保存脱敏 manifest、事件和结果引用。
- `SyncRepository` 在 V0.2 仅保存本地 outbox；不得把身份原文发送到未 Frozen 的云同步端点。

| AD-03 | 第三方白名单未 Frozen | 任何路径尝试路由第三方 | 发送前返回 `third_party_payload_blocked`，身份原文离不开允许边界 |
| AD-04 | 服务超时/限流/不可用 | 用户等待下一问 | P95 10 秒后可继续本地流程；输入不丢；不写正式画像 |
| AD-05 | 服务响应对应旧 `base_version` | 用户已编辑草案 | 丢弃迟到结果并审计，不覆盖用户编辑 |
| AD-06 | 服务返回候选摘要 | 用户未确认草案 | 字段标 `model_generated/unconfirmed`，只能作为草案/候选 |
| AD-07 | 用户撤回 Consent | 存在在途请求 | 停止新请求，响应不得应用，切回本地模板 |
| AD-08 | 本地数据库写失败 | 用户继续输入 | 不显示“已保存”；保留内存输入并提供复制/重试；恢复后可补记故障 |
| AD-09 | 用户跳过或停止追问 | 系统有未完成候选 | 仍可编辑并提交草案；轮数和充分度不构成门槛 |
| AD-10 | 用户显式确认 | 创建正式画像 | 设备端事务创建 `PortraitVersion`；服务和模型无确认权限 |

## 9. V0.2 Readiness 门禁

以下全部满足后，A1 辅助发现才可标为 V0.2 `Spec Ready`：

- [ ] 第一方 `recipient_id`、`service_id/version` 已冻结；
- [ ] `identity_assisted_discovery_first_party` Consent 文案、有效期和保留/删除 SLA 已冻结；
- [ ] 2.2/2.3 字段白名单实现前测试可枚举；

- [ ] 超时、429、5xx、Schema 非法、撤回、迟到结果、版本冲突和本地写失败均有故障注入；
