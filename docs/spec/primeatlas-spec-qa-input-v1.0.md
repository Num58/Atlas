# PrimeAtlas Spec QA 验收输入 v1.0

> 文档角色：Phase 1.5 Spec 验收输入；只定义 Spec 必须可测试化的合同，不测试代码、不裁决实现技术。  
> 依据：已接受的全量 PRD、蓝图—原型—PRD 追踪矩阵、PRD 门禁审查、产品决策确认单。  
> 当前结论：**产品 PRD Accepted；允许编写 Spec；在本文门禁归零并完成独立 Spec 审查前，开发准入不通过。**  
> 范围约束：不新增 A–K、12 类 Agent、全局能力以外的功能；六 Tab 仍为 Candidate，不作为已冻结路由合同。

---

## 1. QA 裁决规则

1. `P0 缺陷 = 0` 是开发准入和后续上线的硬条件，无例外。
2. 产品已确认原则不等于工程合同已冻结。字段、状态、阈值、错误码、SLO、降级或 Given/When/Then 任一缺失，相关 Requirement 标记 `Spec Blocked`。
3. 安全正确性优先于速度和可用性：安全/康复审查失败、超时、离线或数据不足时采用 fail-closed，不得默认放行。
4. 用户主权只覆盖普通目标、时间和偏好冲突；不得覆盖安全否决。
5. 数据不足必须表达未知或保守降级；不得用默认值、历史值、模型推断或静态演示值伪装实测事实。
6. 所有外部依赖必须覆盖权限、数据质量、服务可用性、内容有效性四类降级；仅有 Happy Path 不得标记 Spec Ready。
7. 每个小版本必须是可体验、可验证的完整子集，并逐条声明纳入的 Requirement ID；未纳入项保持 Won't Have/后续范围，不得留下半状态入口。

| 非功能 | 每个版本至少运行受影响的性能、可靠性、无障碍、隐私扫描 | 冷启动/写入/同步/Agent SLO；WCAG；日志敏感字段；崩溃恢复 | 所有冻结门槛达标；无障碍关键路径 100%；敏感原文泄漏 0 |

### 6.2 固定 P0 回归包（每小版本必跑）

1. 适用范围默认未选、主动确认、未完成/拒绝/无法确认和五类命中拦截。
2. 安全否决不可覆盖；安全超时/离线 fail-closed；目标和安全替代仍可访问。
3. 普通冲突“一键采纳/我自己来”，不删目标、不惩罚。
4. 未激活域不显示、不存储、不计权、不进 Agent 上下文。
5. 画像变化只产生候选，用户确认后才生效；多端冲突不自动正式合并。
6. Pulse 可绕过，500ms 长按与按钮/辅助技术入口等价，不改变准备度。
7. RPE 跳过为 missing；滑动完成不等于客观质量；低样本不显示精确百分比。
8. 离线执行追加事实；重连幂等；不重复执行、PR；旧缓存不覆盖新安全/撤回。
9. 第三方模型无独立 Consent 不发送；字段超白名单不发送；日志无健康/知识原文和完整身份叙事。
10. 严厉型无羞辱、威胁、罪感；调性切换不改事实、权限和安全。
11. 融合采集/生成可撤回，关闭无惩罚、红点、降权或核心阻断；安全门禁/既有审计不可关闭。
12. 读屏、动态字体、键盘、减少动效和等效按钮可完成版本核心旅程。

### 6.3 缺陷准入策略

| 级别 | Spec/开发阶段定义 | 准入策略 |
|---|---|---|
| P0 | 违反安全、数据真实性、主权、Consent、隔离/不丢数据、最小可见、核心无障碍或阻塞主旅程 | 必须 0；存在即 Spec/开发门禁失败 |
| P1 | 严重影响体验、可靠性或解释性但不阻塞核心安全旅程 | 开发/发布前至少修复 80%；未修须有 Owner、期限和不可升级为 P0 的证据 |
| P2 | 一般视觉、文案或低频非阻塞问题 | 可入 Backlog，但不得掩盖 P0/P1 或破坏已冻结 Token/无障碍 |

7. 离线读写、待联网、同步/冲突/时区；
8. 第三方与三层字段级可见范围；
9. 指标公式、样本、窗口、误差/置信、至少 3 组金标、禁止显示；

## 8. 开发准入标准

只有同时满足以下条件，版本才可由 `Spec Frozen` 转为 `Dev Ready`：

1. 第 7 节 Spec 包级门禁全部达标，且独立 QA 审查结论为通过。
2. 纳入版本范围、Won't Have、页面/路由、状态机、数据 Schema、API/事件、错误码、幂等和迁移均已版本化冻结。
3. P0 安全、真实性、主权、Consent、离线同步、第三方模型、无障碍、性能门禁均有可自动化的测试设计和明确数据夹具。
4. 每个验收场景已映射测试层级与测试 ID；P0 场景无“仅人工判断”的不可复现口径。
5. 外部依赖四类故障均有可注入/可模拟的错误码、缓存时间和审计证据。
6. 关键公式具备金标；状态机具备合法/非法转换表；同步具备冲突夹具；安全规则具备版本和回滚方案。
7. IA 已从 Candidate 中完成本版本冻结，并通过单域/双域/三域、多目标、安全、权限、离线、返回路径和无障碍可达性验证；不得把 v6 六 Tab 默认当终局。
8. 开发自检命令、CI 门禁、覆盖率收集和测试数据隔离方案已定义。
9. 仍缺参数的 Requirement 全部排除出开发范围并标 `Spec Blocked`；禁止边开发边猜测补齐。
10. P0 缺陷和 P0 未决项均为 0。

**当前判定：`NOT DEV READY`。** 原因见第 9 节；这些是已接受文档明确要求在 Spec 冻结的参数，不是新增范围。
