## 0. 范围裁决

### 0.1 V0.2 边界

V0.2 是身份—目标闭环，`今日`、训练计划生成、训练执行、Prime Pulse、持续安全审查与 Eligibility 均为 `Won't Have for V0.2`。因此：

- V0.2 不展示 Eligibility 首次确认、过期、被替代或重确认页面；
- V0.2 不创建 `EligibilityDeclaration` 业务记录；
- V0.2 不因缺少 Eligibility 阻断身份发现、成长域、目标、里程碑、旅程总览和画像版本；
- 本合同服务于未来首个包含普通精细训练方案生成、重排、恢复或继续执行能力的 Release；
- 本文中的页面路径、页面数量和承载形式仍为 `Candidate`，不得据此提前冻结 IA。

### 0.2 本合同冻结什么

本合同补齐两种业务状态的页面行为：

- `eligibility_expired`：旧确认超过有效期，不能继续作为当前准入依据；
- `eligibility_superseded`：旧确认因政策/声明版本或用户相关状态发生变化而被新条件取代。

冻结内容包括：触发、页面表达、允许操作、禁止操作、离线行为、读屏行为和重确认流程。具体有效期、政策版本、地区文案、转介内容仍为 `Spec Blocked`。

## 1. 状态定义与优先级

| 状态 | 产品含义 | 是否仍可作为普通精细训练准入依据 | 与旧记录关系 |
|---|---|---:|---|
| `eligible_confirmed` | 用户完成当次主动确认，且当前未命中排除项 | 是，但仍须持续安全审查 | 当前有效记录 |
| `expired` | 已达到 Spec 冻结的有效期终点 | 否 | 旧记录只读保留，供审计 |
| `superseded` | 政策/声明版本或相关身份、健康、安全上下文变化，旧确认不再代表当前条件 | 否 | 旧记录只读保留，并关联替代原因/新请求 |
| `incomplete` | 本次重确认未完成 | 否 | 不改变旧记录的审计事实，但旧记录仍不可恢复有效 |
| `rejected` | 用户主动拒绝本次确认 | 否 | 记录拒绝，不推断符合范围 |
| `unavailable` | 因辅助技术、理解、设备、网络或系统故障无法完成 | 否 | 记录失败原因，不推断同意/拒绝 |
| `ineligible` | 用户命中不支持身份/场景 | 否 | 转安全否决、保守提示/专业转介 |

### 1.1 状态优先级

当多个条件同时存在时，页面按更明确、更严格且更可行动的状态表达：

1. 已命中当前安全事件或不支持场景：进入安全决定/不支持范围表达，不只显示“已过期”；
2. 用户相关状态或政策变化导致失效：显示 `superseded`；
3. 仅有效期届满：显示 `expired`；
4. 重确认过程中出现离线或技术失败：保持原失效状态，同时将本次尝试记录为 `incomplete` 或 `unavailable`；
5. 不得以旧 `eligible_confirmed` 覆盖更新的 `expired`、`superseded`、疼痛或安全限制。

## 2. `eligibility_expired` 页面合同

### 2.1 触发条件

满足以下任一条件，且没有更高优先级的 `superseded` 或安全事件时，旧确认进入 `expired`：

- 当前时间达到该 `EligibilityDeclaration.valid_until_at_us`；
- Release Spec 冻结的有效期规则判定回执到期；
- 后台或本地规则在用户请求生成、重排、恢复或继续高风险训练时发现回执已到期；
- 多端同步收到同一声明的更新、更严格 `expired` 状态；
- 设备时钟异常时，不直接根据不可信本地时间恢复有效；按时间合同使用可信时间/同步时间，无法判定时保持更严格的不可准入状态。

### 2.2 页面标题与核心表达

**推荐标题：**“需要重新确认训练适用范围”  
**状态标签：**“上次确认已过期”  
**核心说明必须包含：**

1. 上次确认的日期和适用声明版本；
2. 它为什么不能继续使用：有效期已结束，而不是用户做错了什么；
3. 重新确认用于判断是否进入普通精细训练方案流程；
4. 重新确认不构成医疗判断，也不替代后续持续安全审查；
5. 身份、目标、里程碑等无关能力仍可使用。

**禁止表达：**

- “资格失效”“你的健康状态不合格”等把到期写成用户属性的文案；
- “立即确认，否则目标将丢失”等胁迫；
- 暗示勾选后一定获得训练计划；
- 把旧答案预选为当前答案。

### 2.3 页面信息结构

```text
页面标题
├─ 状态：上次确认已过期
├─ 简要原因：有效期已结束
├─ 上次确认时间 / 声明版本（只读，可展开）
├─ 本次确认的用途和边界
├─ 主要操作：重新确认
├─ 次要操作：稍后再说
├─ 等效返回：继续管理身份与目标
└─ “为什么需要重新确认”展开说明
```

页面可作为独立业务状态页、计划请求前的阻断页或可恢复 Sheet；最终承载形式由低保真验证冻结，但不得仅用 Toast。

### 2.4 用户可做

- 选择“重新确认”，进入一份**全新、默认全部未回答/未选**的确认流程；
- 选择“稍后再说”，返回不需要训练准入的最近安全上下文；
- 选择“继续管理身份与目标”；
- 查看上次确认时间、版本和过期原因；
- 使用读屏、键盘或其他辅助技术完成相同操作；
- 在在线状态下查看当前适用范围说明和地区/版本信息。

### 2.5 禁止操作与系统禁止行为

- 不提供“沿用上次答案”“一键继续”“稍后默认同意”；
- 不将旧答案预填为已选；如为理解提供历史摘要，必须只读、与新回答区分，并默认不展开；
- 不生成、重排、恢复或继续普通精细训练方案；
- 不调用依赖有效 Eligibility 的第三方模型；
- 不清空或删除用户身份、目标、里程碑和历史执行事实；
- 不以红点、倒计时、羞辱或降权强迫重确认；
- 不把一次新的 Eligibility 确认当作持续安全审查通过。

### 2.6 离线行为

当页面离线打开：

- 可以读取已缓存的过期状态、上次确认时间、声明版本和本地说明；
- 原普通精细训练准入保持关闭，已缓存危险/过期计划不得恢复有效；
- 若当前 Release 允许离线填写，可将回答保存为本地 `pending/incomplete` 草稿，但**不得创建 `eligible_confirmed`**；
- 若政策版本或完整声明内容无法验证为最新，主要操作文案为“联网后重新确认”，并说明影响；
- 提供“稍后再说”和返回非训练能力；
- 重连后重新获取当前声明版本，再由用户主动提交；不得把离线草稿静默提交；
- 记录本地查看、尝试和失败原因，恢复后幂等同步。

### 2.7 读屏与键盘行为

页面初始焦点顺序：

1. 页面标题；
2. 状态“上次确认已过期”；
3. 简要原因；
4. 主要操作“重新确认”；
5. 次要操作“稍后再说”；
6. “继续管理身份与目标”；
7. 可展开说明和只读审计信息。

要求：

- 页面进入时播报：“需要重新确认训练适用范围。上次确认已过期。普通精细训练暂不可生成，身份和目标功能仍可使用。”
- 不自动将焦点放在“同意”控件，不自动展开全部法律说明；
- 展开说明后，焦点进入展开区域；关闭后回到触发控件；
- 错误与字段关联，并在页首提供错误摘要；
- 200% 动态字体下不截断主要/次要操作；
- 键盘 Tab 顺序与视觉顺序一致，Enter/Space 可激活按钮；
- 状态不只依赖警告色，须有文本和 Lucide 状态图标。

## 3. `eligibility_superseded` 页面合同

### 3.1 触发条件

满足以下任一条件，旧确认进入 `superseded`：

- Eligibility 政策、声明内容、地区适用规则或 `policy_version` 变化；
- 用户修改年龄/成年状态相关资料，导致旧回答不能代表当前身份；
- 用户主动更新为孕期、急性疼痛、明确疾病、术后康复或其他未来 Spec 冻结的排除场景；
- 新疼痛、恢复异常、负荷异常或其他持续安全信号表明需重新筛查/复评；
- 用户更正了旧 Eligibility 的源数据；
- 多端同步出现更新、更严格的声明/安全状态；
- 旧记录与当前用户主体、政策版本或所有权不匹配；
- 合规/产品裁决要求旧版本不再可接受。

说明：安全信号命中时，安全状态优先。页面可以从“需要更新确认”进入安全决定，但不得用温和的 `superseded` 页面弱化即时阻断。

### 3.2 页面标题与核心表达

**推荐标题：**“情况或规则已变化，需要重新确认”  
**状态标签：**“上次确认已被新条件取代”  
**核心说明必须包含：**

1. 变化类型：规则更新、用户资料变化或安全相关变化；
2. 可披露的变化来源和时间；
3. 为什么旧确认不能代表当前条件；
4. 当前不能生成/继续普通精细训练方案；
5. 可用的非训练能力，以及安全替代/暂缓/复评（如适用）；
6. 重新确认不替代持续安全审查。

如变化来自敏感数据，只显示完成决策所需的最小摘要，不在页面或日志暴露完整健康原文。

### 3.3 页面信息结构

```text
页面标题
├─ 状态：上次确认已被新条件取代
├─ 变化摘要：规则更新 / 资料变化 / 安全变化
├─ 来源与时间（最小必要，可展开）
├─ 当前影响：哪些训练能力暂不可用
├─ 非训练能力仍可用说明
├─ 条件分支操作
│  ├─ 重新确认
│  ├─ 查看安全替代或复评（如安全相关）
│  ├─ 更正资料（如用户认为来源错误）
│  └─ 稍后处理 / 返回身份与目标
└─ 上次确认的只读审计信息
```

### 3.4 用户可做

- 在非即时危险且当前规则允许时发起全新重确认；
- 查看变化摘要、来源、时间和影响；
- 若来源数据错误，进入更正流程；更正只产生新事实和重算，不直接恢复旧确认；
- 若安全相关，选择安全替代、暂缓或查看复评条件；
- 返回身份、目标、里程碑和其他无关能力；
- 稍后处理，不丢失目标或历史；
- 对政策变更查看当前声明版本的简明差异说明；
- 使用辅助技术完成所有操作。

### 3.5 禁止操作与系统禁止行为

- 不提供继续原危险方案或恢复旧回执有效的入口；
- 不把“重新确认”作为绕过安全否决的方法；
- 不允许普通模型、调性或时间编排器覆盖 `superseded`/安全状态；
- 不因用户更正资料立即假定安全，仍须按规则重评；
- 不展示完整疾病原文、笔记原文或无关身份叙事；
- 不自动复制旧回答到新确认；
- 不用“资料异常”“你不符合”等评判性文案；
- 不删除旧记录，旧记录必须只读保留并与新记录建立 `supersedes`/原因关联。

### 3.6 离线行为

当 `superseded` 状态已在本地可知：

- 本地立即执行更严格状态，阻止普通精细训练方案生成/继续；
- 新疼痛等安全信号离线也必须立即进入本地安全暂停；
- 可以展示缓存的变化摘要和安全替代；缓存需标注版本与时间；
- 需要最新政策、远端专业/安全复评或第三方模型时，只能排队或显示待处理；
- 可记录用户更正请求、本地重确认草稿或复评请求，但不得离线伪造通过；
- 重连后按 `operation_id` 幂等提交，并重新核对当前政策和更严格安全状态；
- 旧设备不得用旧 `eligible_confirmed` 覆盖新 `superseded`。

### 3.7 读屏与键盘行为

页面进入时根据原因播报，示例：

- 政策变化：“训练适用范围说明已更新，需要重新确认。普通精细训练暂不可生成。”
- 状态变化：“你的情况发生变化，之前的确认不再适用。相关训练已暂停。你可以查看安全替代或复评条件。”

焦点优先级：

1. 标题与当前阻断状态；
2. 变化摘要；
3. 若安全相关，先到“查看安全替代/复评”；
4. 非即时危险时到“重新确认”；
5. “更正资料”；
6. “稍后处理/返回身份与目标”；
7. 展开说明和审计信息。

要求：

- 安全相关操作的视觉顺序与读屏顺序一致；
- 不可用的“重新确认”应从焦点顺序移除或说明禁用原因，不能让读屏用户进入死路；
- 更新来源不只靠图标或颜色；
- 页面动态更新为安全阻断时使用礼貌但立即的 live region，避免重复播报；
- 关闭 Sheet/对话框后焦点回到原触发项；
- 所有按钮拥有结果导向的名称，如“查看安全替代”，不使用泛化“确定”。

## 4. 重确认流程合同

### 4.1 流程

```text
过期/被替代状态页
→ 用户主动选择重新确认
→ 获取并展示当前政策/声明版本
→ 所有问题默认 unanswered，所有选择默认未选
→ 用户逐项回答
→ 显式提交
→ 系统校验完整性、成年状态与排除项
├─ 符合范围：创建新的 EligibilityDeclaration，进入持续安全评审
├─ 命中排除项：ineligible → 安全否决/保守提示/专业转介
├─ 用户拒绝：rejected
├─ 无法完成：unavailable
├─ 离开/部分填写：incomplete
└─ 版本在填写中更新：本次请求 superseded，展示最新版本并重新主动确认
```

### 4.2 新旧记录

- 新确认创建新记录，不原地修改旧 `expired/superseded` 记录；
- 新记录包含目的、展示版本、用户动作、回答状态、时间、设备、主体、政策版本和结果；
- 旧记录保留审计关系，但不得恢复为当前有效；
- 新 `eligible_confirmed` 只开放进入后续持续安全评审，不直接把计划标记为 `ready`；
- 用户中止不会删除身份/目标，也不产生惩罚。

### 4.3 重确认错误状态

| 错误/状态 | 页面行为 | 可做 | 禁止 |
|---|---|---|---|
| 当前政策加载失败 | 保持失效状态，说明无法取得最新声明 | 重试、稍后、返回非训练能力 | 用缓存旧版本让用户确认 |
| 提交超时 | 保留回答草稿，状态不变 | 幂等重试、稍后 | 重复创建回执 |
| 版本冲突 | 明确声明在填写中更新 | 查看变化并重新主动确认 | 静默套用新版本 |
| 账号不匹配 | 拒绝提交并保护本地数据 | 切回正确账号、保存本地草稿（按账号合同） | 跨账号复用回执 |
| 辅助技术失败 | 标记 `unavailable` 或保持未完成 | 使用等效入口、稍后、反馈问题 | 记录为同意/拒绝 |
| 命中不支持场景 | 转安全/不支持页面 | 安全替代、暂缓、转介、非训练能力 | 继续原危险方案 |

## 5. 页面状态与组件要求

### 5.1 必须状态

该页面/组件至少覆盖：Default、Focus-visible、Active、Disabled、Loading、Error、Offline、Data stale、Version conflict、Success（成功提交后进入下一安全评审步骤），以及 `expired`、`superseded` 两个业务状态。

### 5.2 设计 Token 与图标

- 仅使用项目语义 Token；不硬编码组件色；
- 状态使用 Lucide 图标，候选：`ClockAlert`（过期）、`RefreshCw` 或 `FileWarning`（被替代）、`ShieldAlert`（安全阻断）、`CloudOff`（离线）；
- 图标必须配文本，不作为唯一信息；
- 不使用紫色渐变、发光、玻璃拟态、emoji 或倒计时压迫；
- 安全状态色不可被调性覆盖；
- 动画不超过 400ms，reduced motion 下无位移/缩放。

## 6. 验收场景

| ID | Given | When | Then |
|---|---|---|---|
| EL-PAGE-01 | Eligibility 有效期已结束 | 用户请求生成/重排/继续普通精细训练 | 显示 expired 页面；无继续入口；可重新确认、稍后或返回非训练能力 |
| EL-PAGE-02 | Eligibility 政策版本变化 | 用户进入相关训练流程 | 旧记录 superseded；显示变化摘要；新问题默认未回答；不得沿用旧答案 |
| EL-PAGE-03 | 用户身份/健康资料变化 | 旧确认不再代表当前条件 | 显示 superseded；若安全相关，优先安全替代/复评；不可只重新勾选绕过 |
| EL-PAGE-04 | 用户处于离线且旧确认 expired | 打开页面或尝试重确认 | 保持不可准入；可读缓存说明和非训练能力；不能离线伪造 eligible_confirmed |
| EL-PAGE-05 | 用户离线报告新疼痛 | 旧回执在本地仍显示曾有效 | 本地立即更严格暂停并 supersede；旧有效状态不得覆盖新安全信号 |
| EL-PAGE-06 | 用户使用读屏/键盘 | 完成查看、稍后、重确认或安全替代 | 所有操作等效；状态和影响被播报；无手势/颜色唯一入口 |
| EL-PAGE-07 | 用户部分填写后离开 | 再次请求训练 | 本次为 incomplete；不自动提交；旧 expired/superseded 不恢复有效 |
| EL-PAGE-08 | 用户提交时政策更新 | 服务返回版本冲突 | 保留输入为草稿，说明变化，要求在最新声明上重新主动确认 |
| EL-PAGE-09 | 用户主动拒绝重确认 | 请求训练 | 记录 rejected；无关身份/目标能力继续；无惩罚和诱导 |
| EL-PAGE-10 | V0.2 用户使用身份—目标闭环 | 无训练能力存在 | 不出现 Eligibility 页面、状态或隐藏入口，本合同不扩大 V0.2 范围 |

## 7. 仍需 Release Spec 冻结

以下未由本文擅自定义：

1. 有效期具体时长与起算/截止口径；
2. 地区、年龄和政策版本；
3. 哪些身份/健康/安全变化必然 supersede，以及证据优先级；

9. 平台读屏/键盘测试矩阵与测试数据。

上述任一与相关 Release 有关的项目未冻结时，Eligibility 页面保持 `Spec Blocked`，不得进入开发。
