## 2. ID 方案比较与唯一推荐

权重：标准与互操作 30%、离线唯一性 25%、SQLite 索引/排序 20%、Dart 生态 15%、可读与存储 10%；5 分最好。

| 候选 | 标准/互操作 | 离线唯一 | 索引/排序 | Dart 生态 | 字符串 | 加权分 | 结论 |
|---|---:|---:|---:|---:|---:|---:|---|
| UUIDv4 | 5 | 5 | 2 | 5 | 2 | 4.15 | 不选；122 随机位可靠，但随机前缀无时间局部性 |
| UUIDv7 | 5 | 5 | 5 | 4 | 2 | 4.70 | 唯一推荐；RFC 9562 标准、毫秒时间前缀、74 可用随机位 |
| ULID | 3 | 5 | 5 | 3 | 5 | 4.10 | 不选；26 字符较短，但 canonical spec 非 IETF RFC，Dart 候选更新较慢 |

裁决：所有稳定全局 ID（含 `owner_id`、`operation_id`、事件与聚合 ID）采用规范小写、带连字符的 UUIDv7 文本；SQLite 类型仍为 `TEXT`。禁止依赖 ID 解码出的时间作业务事实，真实时间只读 `*_at_us`。

证据：

- RFC 9562 UUIDv4：除版本/变体外有 122 随机位；不提供生成时间排序。<https://www.rfc-editor.org/rfc/rfc9562.html#name-uuid-version-4>
- RFC 9562 UUIDv7：48 位 Unix epoch 毫秒时间戳 + 74 位随机/可选单调数据，面向按原始字节排序。<https://www.rfc-editor.org/rfc/rfc9562.html#name-uuid-version-7>
- ULID canonical spec：48 位毫秒时间 + 80 位随机，26 字符 Crockford Base32；默认同毫秒顺序不保证，需 monotonic factory。<https://github.com/ulid/spec>
- Dart `uuid` 4.6.0 页面声明支持 RFC 9562 v7、全平台 CSPRNG，最低 Dart 3.0；仓库尚未安装或运行 fixture。<https://pub.dev/packages/uuid>
- Dart `ulid` 2.0.1 页面为 22 个月前发布；仅作生态候选证据。<https://pub.dev/packages/ulid>

状态：ADR-003 `Accepted`。`uuid 4.6.0` 已精确进入 lockfile；`test/architecture/uuid_v7_verification_test.dart` 已在 Flutter `3.44.2`/Dart `3.12.2` 通过格式、同毫秒 4096 次唯一、时钟回退与 SQLite 排序 fixture。

## 3. Operation ledger 验证

### 3.1 为什么纳入

V0.2 的变更用例已经要求 `(owner_id, operation_id)` 重放返回原结果、不同 payload 返回 `idempotency_key_reused`。只靠事件表无法稳定保存拒绝结果和原响应，也无法在崩溃重试时原子判定。故纳入独立 `operation_ledger`。

### 3.2 边界

- 与聚合写、`domain_events`、`audit_records` 在同一 SQLite transaction 写入。
- 唯一键至少包含 `(owner_id, operation_id)`；记录 `payload_hash`、`result_json`、`state` 与错误码。
- 状态仅 `committed/rejected`；禁止 `pending/syncing/synced`、网络 worker、cursor、push/pull。
- `operation_type` 不含 `tombstone`。
- 同 operation + 同 hash 返回已存结果且不再发事件；不同 hash 无副作用并返回确定性错误。

状态：Accepted；见 ADR-004。实现验收仍需崩溃点、重放、不同 payload、`SQLITE_FULL` 原子回滚 fixture。

## 4. Tombstone 与 retention 验证

V0.2 没有删除 UI、备份导入/恢复或远端删除传播；归档与画像版本恢复均不等于删除。因此：

- baseline 不创建 `tombstones`；Repository 不写 tombstone 排除查询；无清理 job。
- 不定义防复活错误码、`retain_until_us` 或 365 天默认值。
- `DATA-RET-01=N/A`，不是“以后补一个保留期”。
- 后续 Release 若引入删除或旧备份恢复，必须先新 ADR、迁移与数据治理审批。

状态：Accepted；见 ADR-011。

## 5. Lucide SVG 与 `flutter_svg` 验证

### 5.1 已证实

- Lucide 官方许可证页与官方仓库 LICENSE 均给出 ISC License；源自 Feather 的列举图标同时附 MIT License。分发 allowlist 资产时必须保留完整 LICENSE 与 copyright notice。  
  <https://lucide.dev/license>  
  <https://github.com/lucide-icons/lucide/blob/main/LICENSE>
- 官方 Lucide release 候选：tag `1.25.0`，commit `5136572c10214634858fcf5f726b2a9d26683918`。该值是候选 pin，不表示已导入。  
  <https://github.com/lucide-icons/lucide/releases/tag/1.25.0>
- `flutter_svg` 官方 pub.dev 页面当前候选为 `2.3.0`，verified publisher `flutter.dev`，声明渲染 SVG 1.1、支持 `SvgPicture.asset`、`semanticsLabel` 与 `ColorFilter`；最低 Dart 3.9 与 Dart 3.12.2 范围相容。  
  <https://pub.dev/packages/flutter_svg>

### 5.2 尚未证实，保持 Pending

已在本仓库精确锁定 `flutter_svg 2.3.0`，并在 Flutter `3.44.2` 通过 allowlist asset widget 渲染、语义标签、尺寸约束和静态分析。Windows/Android 平台探测确认正式仓库没有 `lib/main.dart`、`android/` 或 Gradle Wrapper；debug/release APK 构建均以 `Target file "lib\\main.dart" not found` 退出 1，未生成 APK。独立实验室临时 harness 的 16/20/24 semantics 测试通过，但它不是正式 runner。隔离 AVD `PrimeAtlas_API36_1` 已另行通过冷启动、ADB 与 `sys.boot_completed=1` 健康探测，但没有正式 APK 可安装，因而没有应用级设备证据。尚未执行正式 Android/iOS release build、golden 与产物级 SVG 资产检查；SVG 不是字体 glyph，不得以字体 tree-shaking 日志替代资产机械对照。因此只能确认依赖/渲染链，不能宣称 Release 图标门 Frozen。

唯一允许的实现边界：

```text
业务页面 -> AppIcon(key,size,colorToken,semanticLabel)
              -> 固定 allowlist 映射
              -> SvgPicture.asset
              -> 仓库内 Lucide SVG 资产
```

禁止 emoji 功能图标、`Icon(Icons.*)`/Cupertino 功能图标、第二图标包、业务页面直接 `SvgPicture`。尺寸仅 16/20/24；颜色只用语义 token；不使用紫色到粉色渐变。

状态：ADR-009 `Accepted for dependency and rendering chain / platform golden pending`。已完成 exact lock、许可证归档、资产 hash/allowlist、widget semantics 与唯一引用扫描；正式仓库缺 Android/iOS runner 与入口，Android debug/release APK 均未生成，设备/产物级 SVG 证据、Android/iOS release build 与 golden/像素阈值仍由 QA/platform runner 关闭。

## 6. 架构合同

```text
lib/app/              页面、路由、Riverpod 装配
    -> lib/application/  用例、事务编排、错误映射
        -> lib/core/     实体、值对象、Port；纯 Dart
            <- lib/infrastructure/  SQLite、UoW、ID/Clock、禁用适配器
```

硬规则：依赖只向下并通过 Port 反转；入口只装配；页面不写 SQL；Repository 不含业务规则；按功能/资源分包；每个源文件一个主职责且不超过 300 行。
