# PrimeAtlas 交互模型 — 基于真实场景约束的重设计

## 场景约束分析

| 场景 | 噪音环境 | 社交压力 | 可用交互 | 核心约束 |
|------|:---:|:---:|------|------|
| 🏋️ 健身房 | 嘈杂 | 周围有人 | 屏幕看+触控点+**耳机听** | **不能外放，不能语音指令** |
| 💼 工位午休 | 安静 | 同事在休息 | 屏幕看+触控点+耳机(小声) | **不能说话，不能外放** |
| 🏠 早晨家 | 安静 | 家人睡觉 | 屏幕看+触控点 | **不能说话，不能外放** |
| 🚇 通勤地铁 | 嘈杂 | 人群密集 | 屏幕看+触控点+耳机听 | **不能语音指令** |
| 🏀 球场 | 嘈杂 | 队友在旁边 | 屏幕看+触控点+耳机听 | **不能外放，语音很尴尬** |

## 核心洞察

**唯一通用的交互方式 = 屏幕显示 + 最简触控 + 耳机(可选)**

语音指令在所有场景都有社交压力，只能是"可选的辅助功能"，绝不能是主要交互。

## 训练中的交互设计（正确版本）

### 交互分层

```
第一层：屏幕自动显示（零操作）
  → 大字显示当前动作、目标重量、计时器
  → 用户只需"看一眼"就知道要做什么
  → 手机放在地上/架子上，1-2米外可读

第二层：最简触控（最小操作）
  → 组间休息时：1-2次点击录入数据
  → 默认值预填，用户只需点"确认"
  → 异常情况：点"跳过"/"失败"/"痛"

第三层：耳机语音（可选，私有）
  → 蓝牙耳机听伙伴指导（只有用户自己听到）
  → 绝不外放

第四层：语音指令（可选，辅助）
  → 仅在家中有隐私时可用
  → 默认关闭，需主动开启
```

### 训练中的具体交互

**热身阶段：**
- 屏幕大字显示动作名 + 倒计时
- 用户看一眼 → 跟着做 → 不需要碰手机
- 如果连了耳机：伙伴在耳机里轻声引导
- 如果没连耳机：屏幕底部有文字提示

**正式训练阶段：**
- 屏幕大字：动作名 + 目标(80kg × 5次) + 计时器
- 用户专注训练，不碰手机
- 每组完成 → 自动进入组间休息

**组间休息阶段（唯一需要操作的时刻）：**
- 自动弹出数据录入界面
- 重量预填（默认=上组重量）→ 1次点击确认
- 次数预填（默认=目标次数）→ 1次点击确认
- RPE大按钮（5个等级，不是滑块）
- 最佳情况：2次点击完成一组录入
- 可选的微交互：30秒英语短句（文字形式，无声）

**冷却阶段：**
- 同热身，屏幕大字显示，自动推进

## 训练执行页核心设计

不是"需要操作的App"，而是"可以看的仪表盘"：

```
┌─────────────────────────────┐
│  🔥 Alex                    │  ← 伙伴小头像（耳机有语音，没耳机只有文字气泡）
│  "最后一组，稳住。"          │
├─────────────────────────────┤
│                             │
│     杠铃深蹲                 │  ← 大字（28px+，1米外可读）
│     80kg × 5次              │  ← 目标数据
│                             │
│         ⏱ 1:42              │  ← 超大计时器（56px+）
│                             │
│   ●●●●○  第4/5组            │  ← 组进度点
│                             │
├─────────────────────────────┤
│  上一组：80kg × 5  RPE 7.5  │  ← 自动显示上一组数据
│  历史PR：90kg × 5           │  ← 上下文信息
├─────────────────────────────┤
│  [完成一组]  [跳过]  [疼痛]  │  ← 三个操作按钮（仅在组间休息时激活）
└─────────────────────────────┘
```

## 声音策略（重要修正）

| 场景 | 声音输出 | 声音输入 |
|------|---------|---------|
| 默认（无耳机） | **静默**，屏幕显示文字 | 无 |
| 蓝牙耳机连接 | 伙伴语音指导（仅耳机） | 可选语音指令（需开启） |
| 有线耳机连接 | 伙伴语音指导（仅耳机） | 可选语音指令（需开启） |

**绝对不做的：**
- ❌ 永远不外放语音
- ❌ 默认不开启语音指令
- ❌ 不在公共场所要求用户说话
