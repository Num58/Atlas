# 第一部分：全局设计规范

## 1.1 导航架构

### 底部Tab Bar（四Tab结构）

```
┌──────────────────────────────────────────────────────┐
│                   页面内容区域                         │
│                                                      │
│                                                      │
├──────────┬──────────┬──────────┬────────────────────┤
│  🏠 首页  │  🏋️ 训练  │  🗣️ 英语  │  👤 我的           │
│ (效能总览) │ (训练中心) │ (对话中心) │ (设置/数据)        │
└──────────┴──────────┴──────────┴────────────────────┘
```

**Tab交互规则：**
- 点击已激活Tab → 滚动到顶部 + 刷新（如同Twitter）
- 首页Tab为App冷启动默认Tab
- Tab间切换触发Mode Shift动画（1.5s首次/0.5s后续）
- 训练执行中切换Tab → 训练保持后台运行，返回时恢复

**三端导航差异：**

| 维度 | Android | iOS | 微信小程序 |
|------|---------|-----|-----------|
| 底部Tab高度 | 56pt | 50pt(非刘海)/83pt(刘海机型) | 50pt |
| 返回方式 | 底部Tab + 系统返回键 + 手势返回 | 底部Tab + 左滑返回 + 导航栏返回 | 底部Tab + 左上角返回按钮 |
| Tab切换动画 | 标准过渡 | 标准过渡 + Haptics轻反馈 | 简化过渡 |
| 首页入口 | 默认Tab | 默认Tab | 默认Tab |
| 原生导航栈 | 独立Activity栈 | 独立UINavigationController | 页面栈(最多10层) |

### 1.2.4 全局状态定义

| 状态类型 | 视觉表现 | 用户可操作 | 适用场景 |
|---------|---------|-----------|------|
| **空状态(Empty)** | 居中插图+引导文案+CTA按钮 | 点击CTA开始 | 无数据/无记录 |
| **加载中(Loading)** | 骨架屏/旋转指示器/进度条 | 可取消(部分场景) | 数据请求/计算 |
| **错误状态(Error)** | 错误图标+说明文案+重试按钮 | 重试/返回/离线继续 | 网络错误/服务异常 |
| **部分数据(Partial)** | 正常布局+缺失位置占位 | 点击补充/忽略 | 数据采集中途 |
| **离线状态(Offline)** | 顶部离线横幅+缓存数据+功能限制 | 查看缓存/等待恢复 | 无网络 |
| **低电量(Low Power)** | 动画降帧+功能提示 | 正常操作 | 电量<20% |
| **训练中(Active)** | 全局计时器+防误触+状态栏隐藏 | 训练相关操作 | 训练执行中 |
| **深夜模式(Night)** | 暖色滤镜+动画减半+轻声 | 正常操作 | 22:00-06:00 |

#### 交互流程

**进入条件：**
- 注册完成后首次进入
- 设置页选择"更换伙伴"(需确认)
- 90秒试炼中选择体验伙伴

**页面内交互：**
1. **浏览伙伴**：左右滑动卡片(支持手势+按钮)，带弹性回弹和卡片切换动效
2. **查看详情**：长按伙伴头像→弹出详细能力雷达图(300ms展开)
3. **选择伙伴**：点击"选择"→2秒握手动画序列
   - 0-500ms：伙伴伸手动作
   - 500-1000ms：屏幕闪伙伴签名色(全屏覆盖,opacity 0.3)
   - 1000-1500ms：握手确认，粒子汇聚
   - 1500-2000ms：签名色渐隐，确认状态
4. **人格测试**：点击→底部Sheet弹出3题测试
   - 每题选择后立即过渡到下一题
   - 完成→推荐伙伴卡片放大高亮+"推荐匹配"标签
5. **快速轮播**：长按左右切换按钮→400ms间隔自动轮播，松手停止

**跳转关系：**
- 选择确认 → TR-01-01 数据采集向导(6步)
- 人格测试完成 → 回到本页(推荐伙伴高亮)
- 跳过/试炼入口 → CS-01-01 90秒试炼页

**返回/退出：**
- 首次进入：不可返回(无上一页)
- 设置页进入：返回→设置页
- Android返回键：拦截(首次)，弹出"确定跳过选择？"确认框

#### 手势定义

| 手势 | 作用 | 视觉反馈 |
|------|------|---------|
| 左滑 | 切换到下一个伙伴 | 卡片向左滑出+弹性回弹，新卡片从右滑入 |
| 右滑 | 切换到上一个伙伴 | 卡片向右滑出+弹性回弹，新卡片从左滑入 |
| 点击选择按钮 | 确认选择 | 按钮按下缩放→握手动画→签名色闪烁 |

| 长按切换按钮 | 快速轮播 | 按钮高亮，卡片自动轮播 |

#### 动效规格

- **页面进入**：伙伴头像从中心放大浮现(400ms spring)
- **卡片切换**：250ms ease-out 水平滑动+0.95→1缩放回弹
- **握手动画**：2秒三阶段(伸手500ms+闪光500ms+确认1000ms)
- **选择确认**：全屏签名色闪烁(opacity 0→0.3→0, 1500ms)
- **能力雷达图**：从中心展开(300ms spring)，虚线变实线

# 第四部分：小程序专项适配方案

## 4.1 后台运行限制 → 训练计时器替代方案

### 问题
微信小程序后台运行仅5分钟，无法维持长时间的训练计时器。

### 解决方案

**方案A：前端计时器 + 时间戳补偿(推荐)**

```
训练开始 → 记录 startTimestamp
组间休息开始 → 记录 restStartTimestamp
小程序切后台 → 记录 backgroundTimestamp
小程序回前台 → 计算时间差 Δt = now - backgroundTimestamp
               倒计时 = max(0, 原倒计时 - Δt)
```

- 优点：实现简单，无需服务端
- 缺点：切后台期间无法推送提醒
- 实现：使用 `wx.onAppShow` / `wx.onAppHide` 生命周期

**方案B：服务端计时(辅助)**

```
训练开始 → 上报服务端 startTime
服务端定时推送(订阅消息)：组间休息结束提醒
```

- 优点：可推送提醒
- 缺点：依赖订阅消息(1次/周/模板)
- 实现：配合订阅消息模板

**方案C：本地通知(Android/iOS App备选)**

```
小程序不支持本地通知，但App端可使用：
- Android: Notification + Foreground Service
- iOS: UNNotificationRequest
```

**最终方案：方案A + B 混合**
- 前台：前端计时器(正常)
- 切后台<5分钟：时间戳补偿(静默)
- 切后台>5分钟：放弃计时，回前台显示"训练已暂停X分钟"
- 组间休息结束：可选订阅消息提醒(需用户授权)

## 4.2 持续语音限制 → 英语对话降级方案

### 问题
小程序录音最长60秒，不支持流式转写，无离线语音识别。

### 解决方案

**语音对话降级为"分段对话"模式：**

```
原App体验：
  按住录音 → 流式转写 → 实时纠错 → 连续对话

小程序体验：
  按住录音(≤60秒) → 松开 → 整段转写(2-3秒) → 纠错 → 继续
```

**具体调整：**

| 维度 | App体验 | 小程序体验 |
|------|---------|-----------|
| 录音方式 | 持续录音(无限制) | 分段录音(≤60秒/段) |
| 转写方式 | 流式转写(实时) | 整段转写(2-3秒延迟) |
| 纠错方式 | 实时标注 | 转写后标注 |
| 对话连续性 | 无缝连续 | 每60秒一个自然断点 |
| PR检测 | 10分钟连续对话 | 调整为"累计对话时长" |
| 离线对话 | 支持(Whisper.cpp) | 不支持(需网络) |
| 通勤模式 | 纯语音+极简UI | 不支持(降级为分段) |

**UI调整：**
- 增加"录音时长倒计时"显示(60→0)
- 每段录音完成后显示"处理中..."(2-3秒)
- 对话连续性通过快速切换录音按钮维持
