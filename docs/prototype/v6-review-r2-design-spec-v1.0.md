# PrimeAtlas V6 Review R2 设计细化规格 v1.0

> 日期：2026-07-21 | 角色：颜好看 | 状态：Implementation Ready for Review Prototype
> 定位：V6 增量演进用户评审版；不是 V0.2 Release 实现，不替代正式 Spec，不证明终局能力已上线。
> 真源优先级：Accepted Product Rule > `primeatlas-spec-v1.0.md` > 本规格 > V6 视觉资产。
> 设计寄存器：Product；平台轴：Adaptive；刻度：Variance 4 / Motion 3 / Density 6。
> 行业说明：专家包无 `fitness.md`，本轮采用最接近的 `ai-native.md` 中“结果可信、状态透明、失败可降级”基线，并以正式 Spec 的健康安全合同覆盖通用 AI 模式。

## 1. 评审目标与切片边界

R2 让胜总现在裁决产品方向：身份迁移是否能成为主轴，四入口能否承载 V6 深度，多域与统一时间是否可理解，知识双入口是否分工清楚。研究控制台位于产品画布外；画布内只出现真实产品文案。所有未来状态标注 `Research only / 本地演示数据`，不得写成已实现。

可点击模式必须相互隔离并可独立重置：

| 模式 | 起点 | 一级导航 | 允许内容 |
|---|---|---|---|
| R0 身份确认前 | 价值预览 | 无 Tab | 身份表达、追问、草案编辑、显式确认 |
| R1 V0.2 确认后 | 旅程总览 | 旅程 / 我的 | 身份、1至3域、域内多目标、里程碑、画像版本、本机保存 |
| R2 终局预览 | 今日 | 今日 / 旅程 / 成长域 / 我的 | V6 增量结构与研究夹具，不扩大 V0.2 |

跨模式入口先说明范围差异；切换不继承产品状态或评审答案。V0.2 中完全隐藏今日、成长域、Pulse、训练、知识、Eligibility、远程模型、云同步和未来空壳。

## 3. 六区到四入口映射

| V6 六区 | 终局主归属 | 二级归属与交叉入口 |
|---|---|---|
| 今日 | 今日 | 统一时间轴、安排理由、直接行动、可选 Pulse、反馈证据 |
| 身体 | 成长域 | 体能 / 状态、洞察、恢复、安全；今日只引用当日影响 |
| 训练 | 成长域 | 体能 / 计划、行动、历史、动作学习；今日可直达行动 |
| 目标 | 旅程 | 身份迁移、域、域内多目标、里程碑、证据、重校准 |
| 知识 | 成长域 + 我的 | 业务消费/评估/注入在域或目标；来源/导入/Consent/保留删除在我的 |
| 我的 | 我的 | 画像版本、设备、本机状态、数据权利、调性与治理 |

终局固定顺序：今日 / 旅程 / 成长域 / 我的。手机保留文字标签；Lucide 语义锁定为 `Sun` / `Route` / `Layers3` / `UserRound`。AppIcon 仅用于应用品牌位。全部功能图标来自同一 Lucide SVG 系统，行内16、按钮20、独立24，不混用字符图标。

## 5. Design Token 差异

本规格只引用正式 Spec 已冻结 Token，不在页面内声明颜色值。

| 层 | 沿用 / 新增引用 | 差异合同 |
|---|---|---|
| A1 Identity | `color-bg-app/surface`、`color-text-primary/secondary/muted`、`color-action-primary`、`font-display/body/mono` | V6 暗底橙色主操作迁移为浅色优先 Atlas 语义链；标题不用强调色 |
| A1 Structure | `space-*`、`radius-*`、`text-*`、`leading-*`、`container-*` | 4px网格；圆角4/8/12/16；正文16/24；设备壳不定义产品组件圆角 |
| A2 Semantic | `color-status-success/warning/danger/info`、`color-focus-ring`、`motion-fast/base/slow`、`ease-standard` | 调性不能覆盖安全、Consent、充分度、焦点；每屏主强调最多2处 |
| B Slot | `nav-*`、`button-*`、`input-*`、`list-row-*`、`state-banner-*`、`dialog-*` | 组件只消费 Slot；默认 hairline 与表面差，不用大阴影 |
| C Extension | `identity-transition-*`、`evidence-*`、`data-sufficiency-*`、`relation-fusion-*`、`safety-decision-*` | 身份迁移、证据、充分度、融合、安全分别有文本与图标，不借用 success |

字体锁定 Inter + Noto Sans SC；版本、规则、时间、对象 ID 用 JetBrains Mono。字重400/510/590。禁止紫粉渐变、渐变文字、发光、装饰毛玻璃和组件裸色。

## 6. 交互与动效

- 导航：模式内保留独立返回栈；从旅程目标进入体能计划或知识，返回到原目标与滚动位置；再次点击当前入口回到根页。
- 主操作：每屏一个主按钮，最多两个次操作，其余收入菜单；按钮用具体动词和对象。
- Pulse：500ms 长按、显式“完成启动仪式”按钮、Enter/Space 等效；“直接开始行动”始终同屏可达。
- 时间轴：拖动只作增强；必须提供提前、延后、移到时段菜单与键盘移动；保存前说明受影响目标。
- 状态反馈：短反馈150ms，内容切换250ms，跨栏重排不超过400ms；仅用 `ease-standard`，不使用 bounce、elastic、持续呼吸或随机粒子。
- reduced motion：取消位移、缩放、环形推进和自动庆祝，保留文本状态与即时样式切换。

## 7. 手机与平板响应式

| 形态 | 断点行为 | 关键规则 |
|---|---|---|
| Compact 手机 `<600` | 单列；R1两项或R2四项底部导航；Sheet | 内容驱动高度；安全区；触控至少44pt/48dp；无横滚 |
| Medium 平板 portrait `600至839` | NavigationRail + 主内容；列表可保持单栏 | 不放大手机壳；表单阅读列限制行宽；200%字体回退单列 |
| Expanded 平板 landscape `>=840` | Rail + 列表/详情双栏；左栏对象、右栏详情 | 旅程、成长域、知识治理采用双栏；选择与滚动位置保持 |

必测：360x800、390x844、430x932、600x960、768x1024、834x1194、960x600、1024x768、1194x834；手机 landscape 高风险页可滚动完成。平板双栏的详情为空时显示带任务说明的空态，不复制卡片填满空间。

## 8. 无障碍门禁

- WCAG 2.2 AA；正文对比度至少4.5:1，非文本组件至少3:1；状态使用 Lucide、文本与结构，不仅靠颜色。
- 全部交互用语义控件；可见 `focus-visible`；动态换页聚焦页面标题；Dialog/Sheet 焦点陷阱、Esc/返回关闭、关闭后回焦。
- 支持200%字体、读屏、外接键盘、系统返回和 reduced motion；标题/按钮/状态不可截断，超长身份文本可换行并折叠。
- 图表同时提供结构化列表，含值、单位、阶段、窗口、来源、更新时间和充分度。
- 动作视频提供字幕、转录、文本步骤和关键帧名称；滑动、长按、拖动、触觉均有按钮与键盘等效入口。
- Toast 只做短反馈；安全、保存失败、Consent 撤回、冲突等持久状态写回页面并由 live region 播报。

## 9. 用户评审脚本

主持人先说明：“你看到的是产品方向评审，不是已上线功能。请按自己的理解操作，我不会告诉你入口名称。”每项记录独立/提示/失败、路径、错误入口、返回迷失、关键原话、设备与原型版本。

1. 从干净状态描述现在的自己和想成为的人，建立体能域的两个目标及一个里程碑。
2. 在尚未确认身份时寻找可用主流程；预期是不出现底部 Tab。
3. 确认身份后查看长期方向，再找到画像历史与本机保存位置。
4. 在三个成长域、每域多目标中找到“右膝稳定”，打开其体能计划并返回。
5. 不做任何启动仪式，直接开始07:30行动。
6. 处理19:30双域冲突，同时保留两个目标。
7. 在急性疼痛夹具中尝试继续原动作，随后选择安全替代。
8. 从目标判断一条知识是否进入计划，再到“我的”撤回该来源 Consent。
9. 在手机与平板横屏各完成一次旅程到体能的往返；辅助技术参与者再完成 Pulse 绕过与安全替代。

通过阈值不在本规格预填；无真实 D3/D4 证据时结论写 `Insufficient evidence`。评审只裁决“继续 / 修订 / 阻塞”，不得写“IA Frozen”或“可上线”。
