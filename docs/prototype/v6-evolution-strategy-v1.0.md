# PrimeAtlas V6 增量演进策略 v1.0

> 日期：2026-07-21  
> 状态：Accepted Strategy / 待按三阶段原型任务执行  
> 适用对象：V6 主原型、V0.2 评审切片、终局四入口结构验证  
> RoleVerdict：**ACCEPT — 以 V6 为主要视觉与体验基线做增量演进，禁止推倒重做；产品真源与已接受决策始终高于 V6。**

---

## 1. 文档目的与裁决边界

本文只回答一个问题：如何在不推倒 V6 的前提下，把已确认的产品真源逐步补进原型，并把 V0.2 真实开发范围与终局结构验证严格隔离。

本文不是新 PRD、不是技术架构、不是路由或数据 Schema 冻结稿，也不授权修改正式 Flutter 工程、HTML 或其他代码。出现冲突时按以下优先级裁决：

1. 已接受的产品真源、产品决策与冻结范围；
2. 对应 Release Spec 与安全/康复规则；
3. 本增量演进策略；
4. V6 的现有视觉、页面与交互资产。

V6 是主要视觉与体验基线，但不是可以覆盖产品真源的事实源。所谓“增量演进”是保留成熟资产、调整信息归属、补齐缺失主轴和状态，不是把六个旧入口原样改名，也不是另起一套通用 AI 产品视觉。

### 行业基线

本策略已读取 AI 原生产品行业 reference。采用其中“能力可感知、结果可信、状态透明、错误可降级”的基线，同时服从项目 P0：不使用 emoji 作为功能图标，不使用紫色到粉色渐变，不出现空洞 AI 模板文案。图标统一使用 SVG 图标库，具体库由架构师按项目选型锁定，本文不预设。

## 2. Accepted Decisions

| ID | 已接受决策 | 对原型的直接约束 |
|---|---|---|
| AD-01 | V6 是主要视觉与体验基线，不能推倒重做 | 先复用版式、密度、专业动作体验与状态表达，再做结构重排；不得重建无关视觉体系 |
| AD-02 | 产品真源高于 V6 | V6 中与真源冲突的六 Tab、登录门槛、Pulse 解锁、伪精确指标等必须纠正 |
| AD-03 | 身份迁移是唯一最高层主轴 | 页面必须能说明“当前身份到目标身份”及当前功能在迁移中的作用 |
| AD-04 | 终局一级导航已确认 | 终局为“今日 / 旅程 / 成长域 / 我的”，不再把它写成待选 A/B/C |
| AD-05 | V0.2 只显示两个一级入口 | 身份确认前使用无 Tab 线性流程；确认完成后只显示“旅程 / 我的” |
| AD-06 | V0.2 不展示未来空壳 | “今日 / 成长域 / 训练 / Pulse / 知识 / Eligibility / 同步”等若不属于本版闭环，不出现入口、禁用卡片或“敬请期待” |
| AD-07 | V0.2 是纯本地闭环 | 不发起远程或 HTTP 请求，不接第三方模型，不展示模型切换、云同步或远程能力半入口 |
| AD-08 | 体能是首发锚点，不是终局产品身份 | V6 的身体、训练以及专业知识的消费/评估/注入下沉到“成长域 → 体能”二级空间；知识全局管理按 AD-15 归入“我的” |
| AD-09 | 最多同时激活三个成长域，每域可有多个目标 | 第四域只提示聚焦；不得把“三个域”误写成“三个目标”，不得静默删除既有目标 |
| AD-10 | 安全否决不可覆盖 | 安全/康复规则裁决危险具体方案；用户、调性、普通编排或其他职责均不能继续原危险方案 |
| AD-11 | Prime Pulse 可绕过 | Pulse 是可选启动仪式；主行动始终有直接入口，长按之外必须有按钮、键盘和辅助技术等效入口 |
| AD-12 | 精确指标有条件开放 | 仅在公式已定义、样本充分且置信表达成立时显示；否则使用阶段、趋势、证据与数据充分度 |
| AD-13 | 普通冲突与安全否决分离 | 普通时间冲突允许“一键采纳 / 我自己来”并保留目标；安全否决没有继续原危险方案 |
| AD-14 | 同一时间段只执行一个领域 | 统一时间编排服务多域目标，但不把“融合”表达成同时多任务；休息不被强塞为另一域任务 |
| AD-15 | 知识采用业务上下文入口与全局管理入口双入口 | 内容消费、适用性评估与计划注入位于成长域和目标上下文；来源库、导入、授权/Consent、保留删除与全局治理位于“我的”；V0.2 未纳入的知识能力全部隐藏 |

## 3. V6 保留资产

### 3.1 直接保留为主基线

| V6 资产 | 保留理由 | 增量处理 |
|---|---|---|
| 390px 左右的移动端单列画布与拇指可达操作密度 | 已形成真实移动产品感 | 延续间距、内容宽度和底部操作节奏；补窄屏、大字体与平板重排验证 |
| 专业、克制、高信息密度的体能表达 | 是早期体能单域深扎的核心资产 | 作为“成长域 → 体能”的专业工作空间，不降级为一张通用卡片 |
| 动作详情、教学层级、组次执行与反馈流程 | 已具备完整专业任务感 | 保留主流程；纠正 RPE 缺失值、安全阻断和返回上下文 |
| 调整前后差异与安排理由 | 能建立系统决策信任 | 增补目标来源、证据充分度、规则版本和可回退边界 |
| 知识来源回溯与注入概念 | 支撑知识进入实践 | 下沉至对应成长域/目标；改为先评估、给理由、用户确认，禁止静默加入计划 |
| Prime Pulse 的 500ms 长按仪式感 | 可作为品牌化的可选触觉资产 | 降为可绕过入口，并提供显式按钮及辅助技术等效入口，不再解锁训练 |
| 本地可点击、即时反馈的演示方式 | 适合离线评审与 V0.2 纯本地范围 | 保留；所有演示数据明确为 Mock / Research only，不伪装真实算法 |
| 调性预览、状态差异和渐进展开 | 支撑个性化且不牺牲事实 | 调性只能改变表达，不改变安全结论、证据或指标事实 |

### 3.2 保留外观语言但纠正语义

- 保留页面的内容节奏、排版层级和专业信息密度，不保留六入口的工具分类心智。
- 保留可解释的状态卡和详情层，不保留完成百分比、连续记录或无依据预计日期作为成长主视觉。
- 保留安全提示的高可见性，不把安全否决降格为普通建议或可关闭提醒。
- 保留品牌色与语义色分工，移除紫色到粉色渐变、发光 AI 光球、功能 emoji 和装饰性庆祝依赖。
- 保留高质量微交互，但任何长按、滑动、颜色、动画或触觉都不能成为唯一入口或唯一信息载体。

### 3.3 明确不做

- 不重新设计一套与 V6 无关的首页、卡片、字体和动效体系。
- 不为证明“四入口”而复制四套重复页面。
- 不把 V6 的身体、训练、目标、知识机械改名后继续并列。
- 不用终局能力倒灌 V0.2，也不用 V0.2 的两入口限制否定终局四入口。

## 4. 六区到四入口映射

| V6 一级区 | 终局一级归属 | 终局二级位置与职责 | 迁移原则 |
|---|---|---|---|
| 今日 | 今日 | 安全/Eligibility 优先状态、身份目标关联、安排理由、主行动、可选 Pulse、反馈与当日证据 | 保留高频落点，重排为“为什么—做什么—形成什么证据”，避免仪表盘堆叠 |
| 身体 | 成长域 | 成长域 → 体能 → 状态 / 洞察 / 恢复 / 安全 | 保留专业深度，与体能目标和计划同上下文；不再占终局一级入口 |
| 训练 | 成长域 | 成长域 → 体能 → 计划 / 行动 / 历史 / 动作学习 | 保留执行闭环；从旅程目标和今日行动可交叉进入，并能返回原上下文 |
| 目标 | 旅程 | 当前身份 → 目标身份、活跃域、域内多个目标、里程碑、证据、回顾与重校准 | 目标正式编辑真源在旅程；成长域只提供专业计划的上下文入口 |
| 知识 | 成长域 + 我的 | 业务入口：成长域 / 目标上下文 → 内容消费 / 适用性评估 / 计划注入；管理入口：我的 → 来源库 / 导入 / 授权与 Consent / 保留删除 / 全局治理 | 双入口不等于重复编辑：知识如何服务当前域或目标在业务上下文完成；来源和数据权利在“我的”统一管理；未确认不进入计划，安全否决独立裁决；V0.2 未纳入则全部隐藏 |
| 我的 | 我的 | 画像版本、数据与权限、调性、设备、本地状态、知识来源库与导入、授权/Consent、保留删除、未来同步与账号治理 | 保留全局治理能力；知识业务消费不搬入设置；身份迁移主叙事留在旅程，不藏回个人设置 |

终局固定顺序为“今日 / 旅程 / 成长域 / 我的”。映射只改变信息归属和跨层入口，不要求销毁 V6 的成熟页面；优先把原页面作为二级页面复用，再按统一页面骨架补齐标题、来源、状态和返回上下文。

## 5. 三条增量改造主线

### 5.1 身份迁移：从“目标卡”增量为持续主叙事

**保留：**V6 的目标摘要、进度反馈、计划理由和成果表达。  
**新增：**“当前身份 → 目标身份”迁移头部、身份差距、目标来源、证据充分度、画像候选与版本确认。  
**纠正：**系统推断只能生成草案或候选；用户显式确认后才成为正式画像。恢复历史版本时创建新的 restored active 版本，不改写历史。

渐进顺序：

1. 先在 V0.2 的身份表达、草案确认与旅程总览形成纯本地闭环；
2. 再让 V6 的目标、计划理由和成果逐步携带身份来源；
3. 最后在终局今日和周期回顾中形成“行动证据 → 候选变化 → 用户确认 → 新版本”的回流。

### 5.2 多域：从体能纵切增量为最多三个专业空间

**保留：**V6 体能的身体、训练、知识和安全专业深度。  
**新增：**成长域容器、1–3 个活跃域、域切换、每域多个目标、暂停/恢复和第四域聚焦提示。  
**纠正：**未激活域不展示、不占位、不进入权重；单域隐藏域切换器；跨域融合通过来源和证据流动表达，不新增独立打卡中心。

体能页面先整体下沉为第一个可用域，不先把每个专业模块抽象成空洞通用模板。第二、第三域只有在其真实专业内容和状态合同具备时才进入终局结构验证。

### 5.3 统一时间：在既有计划之上增加共享预算，不重造任务系统

**保留：**V6 今日安排、计划详情、调整前后差异和用户反馈。  
**新增：**多域共享的可用时间预算、单域时间槽、普通冲突解释、调整影响和返回目标上下文。  
**纠正：**同一时间段只执行一个领域；普通冲突不删除目标且允许用户自决；安全/康复否决优先于时间编排并不可覆盖；休息日不自动变成另一域学习日。

版本边界：

- V0.2 只可记录完成身份—目标闭环所需的本地约束，不展示自动时间编排、日历授权、云冲突或远程建议。
- 终局结构验证可使用明确标注的本地测试夹具展示多域时间冲突，但不得声称真实引擎、日历或同步已接入。
- 只有公式、样本窗口、置信边界和禁止显示条件齐备时，时间/进度相关精确指标才允许出现；否则展示区间、阶段、趋势或数据不足。

## 6. V0.2 与终局结构隔离

### 6.1 V0.2 Release 切片

V0.2 是纯本地的身份—目标闭环，不是终局四入口的缩小空壳。

```text
价值预览
→ 身份自由表达
→ 渐进追问（可跳过、可提前结束）
→ 身份草案编辑
→ 身份显式确认
→ 选择 1–3 个活跃成长域
→ 编辑域内目标
→ 确认里程碑
→ 旅程总览 / 目标详情 / 画像版本与本地状态
```

导航规则：

1. 身份确认前：流程位于一级导航外，不显示任何底部 Tab；不得用“今日 / 旅程 / 成长域 / 我的”空壳暗示能力已具备。
2. 身份确认后：只显示“旅程 / 我的”两个入口。
3. “旅程”承载身份、域、目标、里程碑和本地证据；“我的”只承载画像版本与本机数据状态等本版已有治理能力。
4. 若某能力不在 V0.2 闭环内，则完全不展示；不使用锁定态、灰态、角标或“即将上线”吸引点击。

### 6.2 终局四入口结构

终局结构是“今日 / 旅程 / 成长域 / 我的”。它用于验证长期承载力和 V6 资产归属，不自动扩大 V0.2 Release 范围。终局中的今日执行、体能专业空间、Pulse、安全/康复、知识评估、Consent、远期同步等必须由后续版本分别获得范围与 Spec 准入。

### 6.3 隔离识别要求

| 维度 | V0.2 | 终局结构验证 |
|---|---|---|
| 可见一级入口 | 身份确认前 0 个；确认后“旅程 / 我的” | “今日 / 旅程 / 成长域 / 我的” |
| 数据 | 本机真实可保存对象 | Mock / Research only 场景数据 |
| 网络 | 无远程、无 HTTP、无第三方模型 | 即使演示未来能力，也不得伪装真实连接 |
| 目标 | 验证可交付的本地身份—目标闭环 | 验证长期 IA 与 V6 资产承载力 |
| 结果措辞 | 可进入 V0.2 实施门禁 | 结构候选通过或需修订，不等于功能承诺 |

任何跨层跳转都必须在进入前明确提示“未来结构验证，不进入 V0.2”，且两层拥有独立起点、重置、状态与评审记录。

## 7. Review Prototype 定位

`primeatlas-review-prototype-v1.0.html` 的定位是**研究与裁决工作台**，不是业务实现真源、不是 V0.2 Release、不是替代 V6 的新产品，也不是终局 IA 已上线证明。

它只承担四项职责：

1. 并列查看 V6 基线、V0.2 本地切片与终局四入口结构验证；
2. 启动明确的用户任务和状态夹具；
3. 记录页面、操作、任务结果与原型版本，不记录身份自由文本或健康原文；
4. 支持评审者裁决“保留 / 增量修订 / 阻塞”，而不是用偏好投票决定产品真源。

研究控制台、模式选择、任务重置和证据记录只存在于产品画布外。产品画布内不得出现“候选 A/B/C”“评审任务”“切换 V0.2/终局”等研究术语。内部 D0–D2 走查不能冒充真实目标用户 D3 或辅助技术用户 D4 证据。

## 8. 三阶段原型改造

### 阶段一：V6 基线锁定与冲突清单

**目标：**证明团队知道哪些资产必须保留、哪些语义必须纠正，避免改造过程中发生视觉漂移或无意重做。

**产出要求：**

- 锁定 V6 代表性页面与状态：今日、身体、训练、目标、知识、我的、Pulse、动作执行、调整理由、来源回溯和调性预览；
- 对每项标注“原样保留 / 保留视觉并改语义 / 后续下沉 / P0 移除”；
- 建立六区到四入口追踪，不新增平行功能树；
- 清除评审素材中的功能 emoji、紫色到粉色渐变、Pulse 硬门槛、无依据精确指标和登录后解锁本地闭环等错误表达。

**退出条件：**V6 主要视觉语言可辨识；成熟专业页面没有被通用模板替换；所有真源冲突都有唯一去向。

### 阶段二：V0.2 纯本地闭环

**目标：**只验证“身份表达 → 确认 → 域 → 目标 → 里程碑 → 旅程”的可交付本地闭环。

**产出要求：**

- 身份确认前无底部 Tab，确认后只显示“旅程 / 我的”；
- 断网启动、编辑、保存、刷新恢复均可完成，不产生任何远程或 HTTP 请求；
- 支持 1–3 域、每域多个目标、第四域聚焦建议、候选拒绝和画像版本；
- 草案、候选、正式事实和历史版本有持久文本区分；
- 不出现今日、训练、Pulse、知识、Eligibility、第三方模型、云同步或通知半入口。

**退出条件：**从干净状态可独立完成闭环；刷新后本地事实一致；Won't Have 扫描为 0；V6 视觉连续性通过评审。

### 阶段三：终局四入口增量结构验证

**目标：**在 V6 资产基础上验证“今日 / 旅程 / 成长域 / 我的”的长期承载力，而非制作第二套完整产品。

**产出要求：**

- 今日复用 V6 高频体验，并按安全状态、身份关联、安排理由、直接主行动、可选 Pulse、反馈证据排序；
- 旅程承载身份、目标、里程碑、证据和重校准唯一主叙事；
- 身体、训练及知识的内容消费、适用性评估与计划注入下沉“成长域 → 体能”并绑定目标上下文，可从旅程和今日交叉进入并返回原上下文；
- 知识来源库、导入、授权/Consent、保留删除和全局治理进入“我的”，与成长域/目标上下文的业务入口使用同一来源对象但不形成重复编辑真源；V0.2 未纳入则不展示任一知识入口；
- 使用本地研究夹具验证单域、双域、三域多目标及统一时间冲突；
- 安全否决、普通冲突、Pulse 绕过、数据不足和精确指标条件分别可验证；
- 所有未来能力持续标注 Research only，不反向进入 V0.2。

**退出条件：**用户能完成规定任务；V6 高频能力可发现性无 P0 退化；安全与范围隔离为零逃逸；结论只写“终局结构可继续 / 需修订”，不写“已上线”。

## 9. 明确的用户评审任务

主持人口述任务不得直接说出目标入口名称，不得用“点击旅程”“进入成长域”等方式泄露答案。

| ID | 用户任务 | 起点 | 成功终点 | 关键观察 |
|---|---|---|---|---|
| UR-01 | 用自己的话描述现在是谁、想成为谁，并建立第一阶段目标 | V0.2 价值预览 | 完成身份确认、域、目标和里程碑，到达长期总览 | 是否把域误当目标；是否理解草案需确认 |
| UR-02 | 在尚未确认身份时寻找可用主流程 | V0.2 身份表达中 | 在线性流程内继续或稍后恢复 | 是否被空壳 Tab 分流；预期结果应为看不到 Tab |
| UR-03 | 身份确认后查看长期方向，并找到画像历史与本机保存状态 | V0.2 两入口壳 | 分别在旅程与我的找到正确对象 | 是否理解两入口职责；是否误认为已云同步 |
| UR-04 | 在三个活跃域、每域多个目标中找到体能域指定目标，再查看专业计划并返回 | 终局旅程中段 | 旅程目标 → 成长域体能计划 → 返回原目标位置 | 多域密度、目标归属、返回迷失、V6 页面可发现性 |
| UR-05 | 不做启动仪式，直接开始已经安排的安全行动 | 终局今日，Pulse 未操作 | 直接进入行动 | 是否误认 Pulse 为解锁；直接入口是否被抢占 |
| UR-06 | 处理普通双域时间冲突，但保留两个目标 | 终局冲突状态 | 采纳建议或自己安排；两个目标都保留 | 是否区分普通建议与安全命令；同一时间槽是否仅一域 |
| UR-07 | 在虚构急性疼痛场景中理解暂停原因并选择下一步 | 终局执行中 | 只能进入安全替代、暂缓或复评；退出后危险方案仍不可启动 | 是否寻找覆盖安全的入口；能否说明目标仍保留 |
| UR-08 | 判断一条专业知识能否进入当前计划 | 成长域体能知识项 | 选择采纳、改造采纳、暂缓、不采纳之一；安全命中时走安全路径 | 是否以为知识会自动改计划；来源是否可回溯 |
| UR-09 | 判断系统当前是否有足够证据给出精确进度 | 数据不足与数据充分两个变体 | 不足时说出未知项；充分时找到公式/样本/置信说明 | 是否把不足当 0；是否盲信百分比或预计日期 |
| UR-10 | 使用键盘、读屏或 reduced motion 完成 Pulse 绕过、安全替代和两入口导航 | 对应可访问性状态 | 不依赖长按、滑动、颜色、动画或触觉完成 | 焦点顺序、控件名称、回焦、大字体与等效入口 |

记录至少包括：原型版本、起始状态、独立/提示/失败、路径、错误入口、返回迷失、关键原话与证据链接。没有真实样本时统一记为 `Insufficient evidence`，不得预填成功率。

## 10. “不推倒重做”验收

### 10.1 资产连续性验收

- Given 评审者同时查看 V6 和改造原型，When 比较代表性页面，Then 能识别相同的移动端画布、排版节奏、专业密度、状态表达与关键交互语言。
- Given V6 的动作执行、调整理由、来源回溯和 Pulse 资产，When 进入终局映射后的对应流程，Then 核心内容与交互被复用或增量修订，而不是被通用卡片占位替换。
- Given 六区到四入口映射，When 检查页面去向，Then 每个 V6 高频能力都有唯一主归属与必要交叉入口，不存在整区删除或平行重建。
- Given V6 知识能力迁移到终局四入口，When 用户从目标或成长域使用知识并从“我的”管理来源与数据权利，Then 内容消费/适用性评估/计划注入只在业务上下文完成，来源库/导入/授权与 Consent/保留删除只在全局管理入口完成，两处指向同一来源对象且无重复编辑真源；V0.2 不显示任一知识入口。

### 10.2 产品真源验收

- Given V6 表达与 Accepted Decision 冲突，When 执行评审，Then 以产品真源裁决并只纠正冲突语义，不以“保持旧原型”为由保留错误规则。
- Given 身份尚未确认，When 查看 V0.2，Then 不显示任何空壳 Tab；确认后只显示“旅程 / 我的”。
- Given V0.2 处于断网环境，When 完成并刷新身份—目标闭环，Then 本地状态可恢复且没有远程、HTTP、第三方模型或云同步入口。

### 10.3 安全与可信验收

- Given 安全/康复规则否决危险方案，When 用户返回、关闭提示或更换调性，Then 原危险方案仍不可启动，且只提供安全替代、暂缓或复评。
- Given 用户未操作 Pulse，When 选择主行动，Then 可直接开始，不需要长按、积分、准备度或仪式解锁。
- Given 公式、样本或置信任一不足，When 页面展示进展或预测，Then 不显示精确百分比或日期，而显示阶段、趋势、证据和数据充分度。

### 10.4 验收结论

只有同时满足以下条件，才可判定“增量改造通过”：

1. V6 视觉与专业体验主要资产可辨识且有追踪；
2. 所有产品真源冲突均已纠正；
3. V0.2 与终局结构在导航、数据、入口、措辞和证据上完全隔离；
4. P0 红线为 0；
5. 未修改正式工程或用评审原型冒充业务实现。

若视觉连续性失败，优先回退为复用 V6 资产；若产品规则失败，必须修正 V6 语义，不能以“不重做”为借口保留错误。

## 11. P0 红线

任一出现即退回，不进入下一阶段或用户评审：

1. 推倒 V6 主要视觉与体验语言，另起无关通用模板；
2. 以 V6 覆盖已接受产品真源，或把六 Tab 继续当终局导航；
3. 终局一级导航不是“今日 / 旅程 / 成长域 / 我的”，或把身体、训练、知识继续作为终局一级入口；
4. V0.2 身份确认前出现任何空壳 Tab，或确认后出现“旅程 / 我的”以外的一级入口；
5. V0.2 出现远程、HTTP、第三方模型、模型切换、云同步、日历、健康平台、通知或其他半入口；
6. V0.2 出现今日行动、训练、Pulse、知识注入、Eligibility 或未来功能锁定卡；
7. 安全/康复否决后仍能继续原危险方案，或能被用户、调性、时间编排、普通冲突或其他职责覆盖；
8. Prime Pulse 成为启动行动或训练的唯一入口，或长按成为唯一操作；
9. 公式、样本、窗口或置信不足时展示精确百分比、准备度分、预计完成日期或确定性结论；
10. 把最多三个活跃成长域误写成最多三个目标，或添加第四域时静默删除、暂停既有目标；
11. 同一时间段安排多个领域任务，或把休息日强制转换为另一域任务；
12. 普通冲突没有用户自决路径，或普通冲突处理会删除目标；
13. 草案、候选、模拟数据自动成为正式画像、计划或事实；
14. 使用 emoji 作为功能图标；图标必须采用统一 SVG 图标库，具体由架构师按项目选型锁定；
15. 使用紫色到粉色渐变、发光 AI 光球或空洞 AI 模板文案；
16. 用手势、颜色、动画、触觉作为关键流程唯一入口或唯一信息载体；
17. 终局知识只提供成长域业务入口而缺失“我的”管理入口，或把来源库、导入、授权/Consent、保留删除与全局治理塞入业务消费页；反向把内容消费、适用性评估或计划注入搬进“我的”也属于违规；
18. V0.2 展示任何知识业务入口或知识管理入口，或用禁用态/锁定态暗示未纳入能力；
19. 将 Review Prototype、内部走查或 Mock 数据表述为 V0.2 已实现、终局已上线或真实用户证据；
20. 为完成本策略而修改 HTML、Flutter 或其他代码。

## 12. RoleVerdict

**ACCEPT_INCREMENTAL_EVOLUTION**

- V6：保留为主要视觉与体验基线；
- 产品真源：拥有最终裁决权；
- V0.2：无 Tab 身份确认流程 + 确认后“旅程 / 我的”，纯本地闭环；
- 终局：固定“今日 / 旅程 / 成长域 / 我的”；
- 体能资产：身体、训练及知识业务能力下沉成长域中的体能空间；知识来源库、导入、授权/Consent、保留删除和全局治理进入“我的”，形成职责清晰的双入口；
- 评审方法：按三阶段增量改造，以明确用户任务验证，不以偏好投票或内部走查冒充冻结证据；
- 放行条件：不推倒重做验收通过且 P0 红线为 0。
