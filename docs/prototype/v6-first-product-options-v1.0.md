# PrimeAtlas V6-first 产品演进方案 v1.0

> 日期：2026-07-21  
> 角色：产品经理 / 许清楚  
> 状态：Decision Proposal  
> 决策对象：首次使用、方向建立、系统观察表达、V6 资产复用与导航渐进  
> 范围声明：本文提出产品演进与下一轮原型的可裁决方案，不扩大 V0.2 Release；不包含技术选型或实现方案。

---

## 0. RoleVerdict

### verdict

**RECOMMEND_OPTION_A — 采用“先进入今天，再把行动沉淀成方向”的 V6 原位渐进方案。**

首次打开直接进入 V6 的“今日”真实产品骨架，不再经过品牌入口页、身份问卷或研究模式选择。用户先在今日页完成一个有实际意义的动作；系统随后询问是否把这件事作为一段持续方向，由用户用自己的话确认。系统只陈述已发生的行为、数据和可撤销的建议，不宣布用户“是什么人”。

该推荐是终局产品体验方向与下一轮 V6-first 研究原型方向，**不是 V0.2 Release 范围变更**。V0.2 仍是纯本地身份/自我表达—目标闭环：确认前无 Tab，确认后仅“旅程 / 我的”，不出现今日、训练、身体、知识、Pulse、远程能力或未来空壳。

### blocking

1. R2 的独立入口页、逐步身份问卷、“我现在是 / 我想成为”二分填表和系统生成身份结论，永久退出候选集。
2. 未证明 V6 视觉资产连续性前，不得制作新的通用首页、卡片墙或另一套视觉语言。
3. 未经用户明确确认，任何行为、健康数据、内容浏览或系统推断都不得变成正式方向、身份事实或计划。
4. V0.2 与终局研究原型若在入口、导航、数据或措辞上混层，评审必须阻塞；原型展示不得成为 Release 承诺。
5. 若首次行动需要伪造个性化数据、伪造专业计划或放松安全门槛，方案 A 不得实施，应降级采用方案 B。

### advisory

- 下一轮原型只实现方案 A 的一条窄路径，并以方案 B 作为无合适安全起始行动时的降级路径；方案 C 仅用于专业体能用户定向研究。
- 首轮评审重点不问“喜欢哪个页面”，而验证三件事：用户能否不经解释开始；能否说清方向是自己选择的；能否区分系统观察与身份裁决。
- 产品叙事建议从“管理你成为谁”收敛为“帮助你持续接近自己选择的方向”。“身份”可作为用户自愿表达或内部可撤销候选，不再作为首次使用门槛。

### evidence

- V6 已有可复用的真实产品资产：今日安排与目标关联、身体基线和趋势、训练计划/动作教学/完成反馈、目标与里程碑、知识来源与注入评估、个人设置。
- V6 同时存在必须纠正的语义：登录后解锁、Pulse 解锁训练、六个一级入口、无依据的准备度/百分比，以及以功能 emoji 作为图标。
- `v6-evolution-strategy-v1.0.md` 已确认 V6 是体验基线、终局导航是“今日 / 旅程 / 成长域 / 我的”、V0.2 为纯本地且范围隔离。
- 用户对 R2 的直接反馈已证明：入口页不人类友好，逐步身份问卷机械，“我现在是 / 我想成为”擅自定义用户，主页面丑且偏离 V6。该反馈优先于 R2 原有设计假设。
- 行业参考采用专家包中最接近的 `ai-native.md`：能力需可感知、结果需可信、状态需透明、失败需可降级；其通用 AI 交互模式不得覆盖本产品的用户自主和健康安全合同。

## 1. 先纠正需求：用户要的不是“更好的身份问卷”

用户真正需要解决的问题不是“如何快速填完身份信息”，而是：

> 我能否先感受到 PrimeAtlas 确实帮助我做了一件有价值的事，再由我决定这件事是否值得成为持续方向；系统能否理解我的行动，却不替我定义我是谁。

因此，首次体验的核心对象应按以下顺序出现：

1. **一个真实产品场景**：今日、目标、身体基线、训练行动或知识应用中的一个；
2. **一个用户主动行为**：选择、记录、开始、完成、保存或拒绝；
3. **一个可核验观察**：系统说明“刚才发生了什么”，并给出来源；
4. **一次方向确认**：用户决定是否持续、如何命名、何时回看；
5. **逐步出现的产品结构**：只在用户已有对象可管理时出现对应导航。

首次使用不再要求用户先完成关于“自己是谁”的抽象作业。方向可以从具体行动中形成，也可以由用户直接表达；无论哪种来源，都必须经用户确认。

## 2. 共同产品合同

以下合同适用于全部方案，并可直接进入后续 Spec。

### 2.1 用户自主与观察表达

| ID | 合同 |
|---|---|
| PA-DIR-001 | **WHEN** 用户首次打开产品，**THEN** 系统 SHALL 将用户带入真实产品场景，不得先展示独立品牌入口页、产品模式选择或身份问卷。 |
| PA-DIR-002 | 系统 MUST 允许用户用自己的词命名、修改、暂停、归档或拒绝一个方向；系统 NEVER 把预设标签当作用户自我定义。 |
| PA-DIR-003 | **WHEN** 系统从行为、数据或内容中发现可能的持续方向，**THEN** 系统 SHALL 以“观察 + 证据 + 可选建议”表达，并提供“采用 / 修改 / 不采用”。 |
| PA-DIR-004 | **UNTIL** 用户显式确认方向，系统 SHALL 将其标记为“建议”或“草案”，不得写入正式旅程、生成长期计划或影响画像事实。 |
| PA-DIR-005 | 系统 NEVER 使用“你就是……”“系统确认你已成为……”或“从 X 人变成 Y 人”等裁决式句型，除非该文字是用户本人输入并被原样引用。 |
| PA-DIR-006 | 系统 MAY 说“过去 14 天记录了 4 次下肢训练”“你选择继续改善右膝稳定”“这项建议基于 3 次记录”；系统 MUST 同时显示来源、时间窗口和数据充分度。 |
| PA-DIR-007 | **WHEN** 证据不足，**THEN** 系统 SHALL 表达“目前只看到一次行为，尚不能判断趋势”，不得输出身份、稳定偏好、精确进度或预计日期。 |
| PA-DIR-008 | 用户拒绝系统建议后，系统 MUST 保存拒绝结果以避免重复打扰，但 NEVER 把拒绝解释为用户缺乏意志或目标。 |

## 3. 方案 A：先进入今天，再把行动沉淀成方向（推荐）

### 3.1 首次打开看到什么

用户直接看到 V6 的“今日”页面骨架，而不是入口 Hero 或身份表单。首次状态不是伪装成已个性化的满数据仪表盘，而是一个真实、诚实的空今日：

- 日期与“今天还没有安排”；
- 一个主区块“先做一件有用的事”；
- 2 至 3 条安全、低承诺、可独立完成的起始行动，以连续列表呈现而非等尺寸卡片墙，例如：
  - 记录一个你正在关注的身体或能力基线；
  - 浏览并选择一条可执行的入门行动；
  - 从一段已有资料中标记一条想实践的内容；
- 每条都说明完成后会得到什么，不声称是为用户定制；
- 次级入口“我已经知道想改善什么”，允许直接用一句话建立方向草案。

若当前产品版本没有通过专业与安全审核的可执行起始行动，则不伪造行动，自动使用方案 B 的“具体改变”入口。

### 3.2 用户第一个动作

用户点击一条具体行动，例如“记录当前助跑摸高”，或选择“我已经知道想改善什么”并输入“我想让右膝在训练后不再不舒服”。

第一个动作必须是选择、记录、浏览或开始真实对象，不是回答“你是谁”。

### 3.3 方向如何自然建立

行动完成后，在原页面原位置出现轻量回执：

> 已记录：助跑摸高 300 cm，来源为你刚才的手动记录。要围绕“提升起跳表现”持续观察吗？

提供三项操作：

- “作为方向继续”——进入可编辑草案；
- “换个说法”——由用户自己命名；
- “只保存这次记录”——不建立方向。

草案只收集完成持续安排所需的最小信息：用户命名、希望看到的变化、可接受的复查节奏、必要安全约束。所有字段可稍后补，确认前不产生长期计划。

### 3.4 系统如何表达观察而非身份裁决

系统采用固定三段式：

1. **观察**：“你刚刚记录了助跑摸高 300 cm”；
2. **证据**：“来源：手动记录；样本：1 次；目前不足以判断趋势”；
3. **询问**：“是否要把提升起跳表现作为一段持续方向？”

禁止表达：“你是扣篮训练者”“你想成为篮球运动员”“系统判断你缺乏爆发力”。

后续有数据时也只说：“近 4 周同条件记录上升 2 次”“这可能说明起跳表现正在改善”，并允许用户纠正记录或拒绝解释。

### 3.5 如何复用 V6 内容

| V6 资产 | 复用方式 |
|---|---|
| 今日 | 保留日期、时间线、主行动和紧凑移动端布局；首次空状态替换伪个性化数据与 Pulse 硬门槛。 |
| 目标 | 原目标卡改为方向确认后的“方向—目标—里程碑”连续对象；用户命名在上，系统建议在下。 |
| 训练 | 经安全审核的动作详情、组次、教学视频、文本步骤、完成反馈作为起始行动与后续专业执行页。 |
| 身体 | 基线记录、来源、趋势、更新时间、数据充分度作为行动证据；移除无依据准备度和百分比。 |
| 知识 | 阅读、来源回溯和“待实践”标记可作为起始行动；注入计划仍需适用性评估与用户确认。 |
| 我的 | 承载本机状态、草案/正式版本、数据权利、来源治理和调性；不承载方向主叙事。 |

### 3.6 导航如何渐进出现

- 第一次打开且无方向：产品画布内无底部 Tab，仅在 V6 今日骨架完成单一任务。
- 确认一个方向后：显示“旅程 / 我的”，落点为旅程中的刚确认方向。
- 后续 Release 真正具备每日安排后：加入“今日”，默认回到当日行动。
- 首个专业成长域内容通过范围门禁后：加入“成长域”；单域时隐藏域切换器。
- 终局顺序固定为“今日 / 旅程 / 成长域 / 我的”。

注意：以上是终局演进顺序。V0.2 正式 Release 仍执行确认前无 Tab、确认后仅“旅程 / 我的”。

### 3.7 优点

- 最符合“直接进入真实产品场景”，不要求用户先理解 PrimeAtlas 的概念体系。
- 最大化复用 V6 今日、训练、身体、知识的成熟资产，视觉与体验连续性最佳。
- 用户先获得小价值，再决定是否承诺持续方向，降低抽象表达成本。
- 系统观察建立在用户刚发生的行为上，较容易避免擅自身份裁决。
- 可自然演示“行动—证据—方向—旅程”的产品闭环。

### 3.8 缺点与风险

- 需要至少一条安全、诚实、无需既有画像也能完成的起始行动；体能内容尤其不能用通用计划冒充个性化计划。
- 若起始选项过多，会退化为工具菜单；必须限制为 2 至 3 条并按场景排序。
- 用户可能只完成一次行动、不建立方向；这应被视为合法退出，不得强制追问。
- 该完整体验超出 V0.2 Release，必须在研究原型中保持范围标识。

### 3.9 适用条件

- 下一轮目标是验证终局产品体验和 V6 资产承载力；
- 已有至少一个经过安全审查的低承诺起始动作；
- 团队能把 Mock/Research only 与正式本地数据严格隔离；
- 需要验证用户是否理解“方向来自我的选择，而不是系统定义”。

## 4. 方案 B：从一件具体想改变的事进入 V6（稳健降级）

### 4.1 首次打开看到什么

首次打开直接落在 V6“目标”内容骨架演进后的单对象页，不展示品牌 Hero，不问身份。页面标题为：

> 最近最想改变的一件具体事情是什么？

页面只有一个自由输入区和“先从最近一次经历说起”的辅助入口。下方可以展示 2 至 3 条真实输入示例，但不是可套用身份标签，例如“训练后右膝经常不舒服”“想在半年内完成一次公开演讲”“读完的专业书没有用到工作里”。

### 4.2 用户第一个动作

用户输入或口述一件具体变化；也可以选择“暂时说不清”，转而从 V6 的身体记录、训练历史或知识记录中选一个已有对象。

### 4.3 方向如何自然建立

系统只把输入结构化为可编辑方向草案：

- 用户原话；
- 想看到的变化；
- 可能的成长域，仅作为组织建议；
- 需要补充的最小约束；
- 第一次可验证证据。

系统一次性展示草案，不进行 3 至 6 轮机械追问。缺失信息标记为“稍后补充”；用户可以直接修改或确认。确认后进入 V6 旅程页，再从该方向直达身体、训练或知识的对应专业内容。

### 4.4 系统如何表达观察而非身份裁决

系统说：

> 你写的是“训练后右膝经常不舒服”。我先把方向记为“训练后保持右膝稳定”；这是草案，可以改名。为了判断变化，需要你确认疼痛相关安全信息和一次基线记录。

系统不说：“你是有膝伤的篮球爱好者”“你的目标身份是无伤运动员”。成长域归属只能表述为“建议放在体能域管理”，用户可改选。

### 4.5 如何复用 V6 内容

| V6 资产 | 复用方式 |
|---|---|
| 今日 | 方向确认前不出现；后续有真实安排时承载当日主行动与理由。 |
| 目标 | 最大程度复用目标、里程碑、资料关联结构；去掉三个目标上限、身份标签和伪精确进度。 |
| 训练 | 从确认后的体能方向直达相关训练或安全替代，不在首次输入前展示通用课表。 |
| 身体 | 用作方向的基线证据和安全约束采集，不先用仪表盘定义用户。 |
| 知识 | 当用户表达的是“把书用起来”时，直接进入来源—适用性—待实践链路。 |
| 我的 | 保存草案/正式版本、本机状态与数据权利。 |

### 4.6 导航如何渐进出现

- 输入与草案确认：无底部 Tab，保持线性单任务。
- 方向确认后：仅“旅程 / 我的”。
- 有真实今日安排和专业域内容后，再依次增加“今日 / 成长域”，终局顺序固定。
- V0.2 可采用该方案的范围内变体，因为它不要求展示今日、训练、身体或知识入口；这些资产只在终局研究层演示。

### 4.7 优点

- 与 V0.2 当前纯本地方向/目标闭环最兼容，范围风险最低。
- 不需要伪造首次行动或个性化计划，适合安全要求高的场景。
- 比身份问卷更直接：问变化，不问身份；一次草案替代多轮追问。
- 用户原话是正式方向的唯一上游，系统权力边界清晰。

### 4.8 缺点与风险

- 仍然要求用户先表达，获得产品价值较晚；若页面处理不好，可能仍被感知为“另一张表单”。
- 对说不清方向的用户帮助有限，需要提供“从已有经历/记录开始”的逃生路径。
- V6 今日、训练和身体的强资产在首次打开时不可见，V6-first 感知弱于方案 A。

### 4.9 适用条件

- 以 V0.2 Release 可交付闭环为首要目标；
- 暂无无需画像即可安全执行的起始行动；
- 用户群普遍已有明确问题或目标；
- 团队希望先解决“系统不擅自定义用户”，再验证行动闭环。

---

## 5. 方案 C：先记录一个基线，再决定是否持续（专业体能定向）

### 5.1 首次打开看到什么

直接进入 V6“身体”专业空间的轻量基线页。页面不展示完整健康仪表盘，也不预填准备度，而是说明：

> 先记录一个你在意的当前状态。记录本身不会生成身份或训练计划。

用户从经安全筛选的基线类型中选一项，例如助跑摸高、踝背屈、主观训练后不适；每项说明所需条件、风险提示、数据用途和“现在不记录”。

### 5.2 用户第一个动作

选择一个基线，阅读条件后手动记录，或因条件不合适而选择“稍后再测”。安全高风险项目不得以自测替代专业评估。

### 5.3 方向如何自然建立

记录后展示事实回执与数据充分度，并询问：

> 这次记录要服务于哪件你想改变的事？

用户可直接命名，也可选择“只留作记录”。系统可给出基于测量对象的方向建议，但必须说明这是组织建议而非结论。用户确认后，基线成为该方向的第一条证据。

### 5.4 系统如何表达观察而非身份裁决

允许：“已记录踝背屈 8 cm；测量条件由你确认；只有 1 次，尚无趋势。”

禁止：“你柔韧性差”“你属于高风险训练者”“你未来应成为运动达人”。即使数值落在参考区间外，也只陈述参考来源、测量误差与安全下一步。

### 5.5 如何复用 V6 内容

| V6 资产 | 复用方式 |
|---|---|
| 今日 | 方向和安全条件确认后，承载复测或安全行动；首次不出现。 |
| 目标 | 基线确认后形成方向与第一里程碑；不先给目标百分比。 |
| 训练 | 基线仅决定是否需要进一步评估，不能直接自动生成训练计划。 |
| 身体 | 最大化复用数字孪生、测量、趋势和来源结构，并补充分度与误差。 |
| 知识 | 测量方法、参考来源和安全说明可回溯；知识不自动注入计划。 |
| 我的 | 承载数据来源、更正、删除、本机状态与 Consent。 |

### 5.6 导航如何渐进出现

- 首次测量：无底部 Tab，处于成长域体能的深链单任务。
- 确认方向后：显示“旅程 / 我的”；旅程首屏展示方向与第一条证据。
- 后续具备真实体能专业空间时加入“成长域”；具备真实安排后加入“今日”。
- 终局仍固定“今日 / 旅程 / 成长域 / 我的”。

### 5.7 优点

- 价值与证据都非常具体，系统观察边界最容易被用户理解。
- 强复用 V6 身体、趋势、训练安全和专业知识资产。
- 天然建立“没有足够数据就不下结论”的可信习惯。
- 对体能首发锚点用户有较强专业感。

### 5.8 缺点与风险

- 容易让 PrimeAtlas 再次被理解为健身或身体数据 App，与多成长域终局定位不一致。
- 测量存在环境、方法和安全门槛，首次完成率可能低。
- 对表达、阅读、职业成长等非体能方向迁移性弱。
- 完整专业测量与健康安全能力不属于 V0.2，不可借原型提前承诺。

### 5.9 适用条件

- 只针对明确的体能专业用户做定向研究；
- 测量协议、安全提示、误差与数据用途已通过评审；
- 研究问题是“证据先行能否建立信任”，不是验证多域通用首次体验。

## 6. 决策矩阵

评分采用 1 至 5 分，5 分为最佳；范围风险一项 5 分代表风险最低。

| 维度 | 权重 | 方案 A 行动先行 | 方案 B 具体改变 | 方案 C 基线先行 |
|---|---:|---:|---:|---:|
| 直接进入真实产品 | 25% | 5 | 3 | 5 |
| V6 资产复用与连续性 | 20% | 5 | 3 | 4 |
| 尊重用户定义权 | 20% | 5 | 5 | 5 |
| 首次价值出现速度 | 15% | 5 | 3 | 4 |
| V0.2 范围风险低 | 10% | 2 | 5 | 1 |
| 多成长域可扩展 | 10% | 5 | 5 | 2 |
| 加权结果 | 100% | **4.65** | **3.80** | **3.95** |

### 推荐理由

方案 A 最能正面回应本轮失败：不再要求用户先读懂产品或完成身份作业，而是让 V6 本身承担价值说明。它也最符合长期北极星——用户通过自己选择和完成的行动，决定要持续接近的方向；系统的角色是保存证据、解释观察、提出可拒绝建议。

但推荐带一个硬条件：**方案 A 不能用伪个性化计划换取“真实感”。** 若安全、专业或范围条件不成立，必须使用方案 B 降级。方案 B 是 V0.2 最可交付、最诚实的路径，不是次等视觉方案；它承担正式 Release，而方案 A 主要承担终局产品方向验证。方案 C 不进入默认首次体验，只做体能专业用户研究。

## 7. R2 永久废弃模式

以下不是“本轮暂不做”，而是产品级永久反模式；后续不得换文案后重新引入。

| ID | 永久废弃模式 | 废弃原因 | 替代合同 |
|---|---|---|---|
| DEPR-01 | 产品前独立价值入口页 / 模式选择页 | 把用户挡在产品外，要求先理解概念 | 首次打开进入真实产品场景 |
| DEPR-02 | 多步身份问卷、逐轮追问后才能进入产品 | 机械、前置成本高、像审讯或填表 | 先行动，或一次表达具体改变；其余按需渐进补充 |

| DEPR-05 | 用抽象身份承诺作为解锁计划的门槛 | 用户必须先表态才能获得价值 | 低承诺行动可先完成；正式持续安排才需确认方向与约束 |

## 8. 推荐方案 A 的最小评审切片

下一轮不需要做完整四入口，只需做以下 6 个连续状态：

1. **首次今日空状态**：无独立入口页，无伪个性化数据；2 至 3 条安全起始行动；
2. **真实动作**：完成一次记录、浏览或低承诺行动；
3. **事实回执**：观察、来源、样本、充分度；
4. **方向邀请**：采用、改名或只保存本次记录；
5. **方向草案**：用户原话、期望变化、首条证据、必要约束；
6. **确认后的旅程**：方向与证据可见；仅在研究层演示其如何连接 V6 专业页面。

### 必测任务

- 用户不阅读说明，能否在 30 秒内开始一个动作；
- 用户完成后，能否说出“系统记录了什么”以及“系统没有替我决定什么”；
- 用户能否拒绝方向建议并保留单次记录；
- 用户改名后，系统是否始终使用用户的新名称；
- 用户能否从旅程进入 V6 专业页再返回原方向；
- 安全否决时是否不存在继续危险动作；
- 辅助技术用户能否不依赖长按、手势、颜色、动效或触觉完成同一路径；
- 评审者能否明确区分 Research only 与 V0.2 Release。

### 放行判断

无真实用户证据时结论 MUST 写 `Insufficient evidence`。下一轮只允许输出：

- Continue：继续方案 A；
- Revise：保留行动先行但修订起始动作或方向邀请；
- Fall back：因安全、专业或范围条件不足，降级方案 B；
- Block：出现身份裁决、范围逃逸或 V6 视觉断裂。

不得用内部评审偏好冻结 IA，不得将点击可用等同于产品方向已验证。

## 9. 与既有蓝图的裁决关系

`PrimeAtlas_完整需求蓝图_Final.md` 中仍有“定义我是谁”“身份进度”“系统确认身份迁移”等旧表述。结合本轮用户明确反馈，后续产品真源应按下列方式解释或修订：

1. “身份”不再是系统要求用户填写的必选数据结构，而是用户可选的自我表达；
2. 产品北极星从“系统管理你成为谁”修订为“帮助用户持续接近自己选择的方向”；
3. 系统可维护方向、目标、里程碑和证据，不拥有对用户身份的裁决权；
4. 用户原话可被引用，但系统生成文字必须标为草案/建议并可拒绝；
5. 原“身份进度指数”在公式、证据和用户授权未成立前不得作为对外北极星指标；优先验证“方向确认后持续产生有效证据的用户比例”；
6. 原“身份确认”相关用例应改为“方向确认 / 自我表达确认”，不得恢复二分身份问卷。

该修订不影响“目标驱动、成果驱动、自适应、安全专业、知识流动”等仍有效原则；它只重新界定系统与用户之间的解释权。
