# PrimeAtlas 完整原型方案评审稿 v1.0

> 日期：2026-07-20  
> 阶段：编码前原型评审  
> 状态：Candidate / 待用户评审  
> 范围：完整终局 IA 结构验证 + V0.2 身份—目标闭环验证

## 0. 本次评审目标

本轮评审原型是否正确表达 PrimeAtlas，而不是评审真实算法、真实模型质量或上线完成度。

请重点判断：

1. **整体导航**：主候选“今日 / 旅程 / 成长域 / 我的”是否能承载身份迁移主循环；
2. **V0.2 闭环**：是否完整覆盖“身份表达 → 画像确认 → 成长域 → 目标 → 里程碑 → 旅程总览”；
3. **终局承载力**：今日执行、专业体能、安全、融合、知识、成果、Consent、离线与同步是否各归其位；
4. **页面层级**：是否先呈现决策所需信息，再渐进展开证据、来源、版本和高级信息；
5. **范围诚实性**：V0.2、终局结构壳、A/B/C 导航对照是否明确分开。

### 本轮不决定

- 真实 API、数据库物理 Schema、真实模型接入；
- Eligibility 最终有效期、地区政策和转介内容；
- 指标公式、精确百分比和预测日期；
- 云同步、第三方模型和健康平台接入；
- 方案 A 是否正式 Frozen。最终冻结仍需真实 D3/D4 可用性证据。

## 2. 整体页面布局与导航

### 2.1 主候选 A

底部四 Tab：

1. **今日**：把身份目标带入今天的现实行动；
2. **旅程**：当前身份 → 目标身份、成长域、目标、里程碑、证据和重校准；
3. **成长域**：最多三个活跃域的专业工作空间，早期重点承载体能深扎；
4. **我的**：数据、Consent、调性、同步、账号和数据权利治理。

统一页面骨架：

```text
持续状态区：离线 / 本机已保存 / 待同步 / 安全 / 数据过期
页面标题 + 当前对象状态
主叙事或主决策
唯一主操作
次级选项
证据 / 来源 / 版本 / 审计的渐进展开
底部导航或上下文返回
```

### 2.2 A/B/C 对照

| 方案 | 一级导航 | 对照目的 | 主要风险 |
|---|---|---|---|
| A | 今日 / 旅程 / 成长域 / 我的 | 生命周期与专业空间平衡 | “旅程”“成长域”心智需验证 |
| B | 今日 / 身份 / 计划 / 我的 | 身份显著度上限 | 身份像静态档案，计划像任务工具 |
| C | 今日 / 身体 / 训练 / 成长 / 我的 | v6 迁移成本下限 | 继续像健身 App，多域扩展弱 |

主推荐仍为 A，但合法状态只能是 `Primary validation candidate`，不是 `Frozen`。

### 2.3 V0.2 导航边界

V0.2 原型不展示空壳 Today、训练、Pulse 或知识入口。身份确认完成后的主落点是旅程总览：

```text
当前身份 → 目标身份
├─ 1–3 个已激活成长域
├─ 每域多个已确认目标
├─ 里程碑与证据规则
└─ 画像版本 / 本地保存状态
```

### A0 核心点击流程

```text
A0-01 价值预览
→ A0-02 自由表达
→ A0-03 渐进追问（可随时够了）
→ A0-04 草案编辑
→ A0-05 显式确认
→ A0-06 选择 1–3 个成长域
→ A0-07 编辑/确认域内多个目标
→ A0-08 确认里程碑
→ A0-09 旅程总览
→ A0-10 目标详情
→ A0-11 画像版本与本地状态
```

### A0 必须可验证的异常流程

- 模型/第一方辅助超时、限流、非法结构：原输入和本地草案不丢失；
- 画像版本冲突：并列 base/current 版本、来源和影响，不自动合并；
- 第四成长域：只给聚焦建议，现有域和目标不变；
- 数据不足：只能建立探索目标或保守草案，不显示精确百分比和预计日期；
- 离线创建和编辑：显示“已保存本机 / 待同步”，刷新后可恢复；
- 账号不匹配：拒绝跨主体写入，不串号；
- 来源失效/更正：候选撤回或重算，正式历史不被静默改写；
- 键盘、读屏、200% 字体和 reduced motion 可完成全部核心流程。

### A0 绝对禁止出现

Today、PlanAction、执行/完成、RPE、训练、Pulse、准备度、知识注入、PR、成果分享、Eligibility、云端正式同步、第三方模型发送/切换、健康平台/日历/位置/通知半入口。

## 4. P-A1：终局 IA 结构壳

### 4.1 今日

**A1-01 今日根页**的层级：

1. 安全否决或 Eligibility 必处理状态；
2. 目标身份方向 + 今天这一步的作用；
3. 为什么今天这样安排：目标关联、调整来源、数据充分度；
4. 可执行主行动或安全替代；
5. 能量、准备度和数据充分度摘要；
6. 可选 Prime Pulse；
7. 其他行动；
8. 有真实证据时的身份/融合回流。

变体：正常、休息日、安全日、离线、无计划、数据过期。

**A1-02 Pulse**：长按 500ms 是可选仪式；必须有“开始启动仪式”按钮和读屏自定义动作；不改变准备度、不解锁训练，用户可直接开始行动。

**A1-03 行动详情/执行/反馈**：目标来源 → 安排理由 → 安全信息 → 开始/暂停/恢复/完成/跳过 → RPE/疼痛/质量。跳过 RPE 保存 `missing`，绝不复用历史值。

### 4.2 旅程

- **A1-JR-01 旅程总览**：当前→目标身份、身份差距、1–3 active 域、每域多个目标、里程碑、证据和重校准候选；
- **A1-JR-02 目标/里程碑详情**：目标与里程碑唯一正式编辑真源；
- **A1-JR-03 周期回顾/画像候选**：事实和证据→候选变化→确认/拒绝；确认后才形成新版本；
- **A1-JR-04 证据与来源回溯**：来源芯片→训练/计划/目标/笔记/书摘/回顾父节点→返回原上下文。

### 4.3 成长域

- **A1-06 成长域根页**：只展示 active 域；单域隐藏切换器；
- **A1-07 体能二级 IA**：状态 / 计划 / 历史 / 知识 / 安全；
- **A1-08 来源回溯**：父节点失效、撤回、无权限或离线未缓存均有解释；
- **A1-09 知识评估**：采纳 / 改造采纳 / 暂缓 / 不采纳；安全否决独立，未确认不改计划。

### 4.4 我的与跨页面状态

- **A1-ME-01 我的根页**：设备、本地状态和治理入口；
- **A1-ME-02 画像版本**：查看差异与恢复；恢复创建新的 restored active 版本；
- **A1-11 Eligibility**：仅未来训练 Release；未完成/拒绝/无法/不支持/过期/被替代/离线；
- **A1-12 权限与 Consent**：目的、范围、接收方、字段、有效期、拒绝/撤回与传播状态；
- **A1-13 离线/同步中心**：本机已保存、待同步、重试、冲突、需处理、账号不匹配；
- **A1-14 数据不足/内容失效**：学习态、过期、来源冲突、失效、撤回。

### 4.5 普通冲突与安全否决必须分离

**A1-04 普通冲突**：受影响目标、代价和依据；同时提供“一键采纳 / 我自己来”，不删除目标、不强制。

**A1-05 安全决定**：原因、证据、规则版本、安全替代、暂缓和复评条件；没有“继续原危险方案”；关闭页面后危险行动仍不可启动。

## 6. 关键页面视觉层次

### 6.1 身份自由表达

```text
页眉：身份定义 / 已保存本机
说明：无需选择标签，用自己的话表达
主输入：我现在是谁 / 我想成为谁
辅助：三个低干扰示例，点击只填入编辑区
主按钮：继续梳理
次按钮：我自己编辑
数据说明：本地优先；远程辅助按目的单独说明
```

### 6.2 身份草案确认

```text
状态标签：草案，尚未生效
当前身份 → 目标身份（页面唯一高层叙事）
结构化字段：动机 / 约束 / 优势 / 证据充分度
来源说明：用户原话 / 本地模板 / 第一方候选
主按钮：确认并创建画像版本
次按钮：返回编辑 / 暂不确认
```

### 6.3 旅程总览

避免“3 域 × 多目标卡片墙”，使用连续纵向结构：

```text
身份迁移头部
身份差距摘要
成长域分组标题
  目标列表行：名称 / 阶段 / 趋势 / 证据充分度 / 状态
  里程碑摘要
证据与回顾
画像候选（存在时）
```

每个目标只显示摘要；点击进入唯一正式详情页。

### 6.4 今日根页

不使用完成百分比仪表盘。首屏只回答：

- 今天为什么这样安排；
- 与哪个身份/目标相关；
- 现在最合适的行动是什么；
- 有哪些已知、未知或安全限制。

Pulse 降为可选次级入口，主行动始终直接可达。

### 6.5 安全决定页

```text
标题：当前方案已暂停
状态说明：发生了什么
命中依据：最小必要证据 + 规则版本
明确禁止：当前危险具体方案不可继续
主要操作：采用安全替代
次要操作：暂缓 / 查看复评条件
保留说明：目标仍保留，可继续使用身份和目标能力
```

风险表达直接但不诊断、不恐吓；安全色不受调性影响。

### 6.6 Consent 页面

```text
目的
接收方
字段清单
保留与撤回影响
默认未选选择项
同意 / 拒绝并继续本地能力
```

拒绝和撤回不制造红点、降权或无关功能阻断。

## 7. 设计系统

### 7.1 视觉方向

- 基调：专业、可信、克制，但不冷漠；
- 参考：Linear 的信息克制、Notion 的对象层级、Apple HIG 的移动可达性、Stripe 的异常状态质量；
- 差异化记忆点：**持续可见的“当前身份 → 目标身份”迁移线索**，不是 AI 光球、聊天头像或完成圆环。

### 7.2 Token

- 浅色优先；背景 `neutral-50`，表面 `neutral-0`；
- 主行动色 `atlas-blue-700`；融合关系 `teal-600`；成功/警告/危险分别独立；
- 字体：Inter + Noto Sans SC；版本、规则、时间和标识用 JetBrains Mono；
- 间距：4px 网格；
- 圆角：4 / 8 / 12 / 16；
- 动效：0 / 150 / 250 / 400ms；不使用弹跳和持续呼吸吸引操作；
- 图标：仅 Lucide，16 / 20 / 24px；
- 正文对比度 ≥ 4.5:1；iOS 触控 ≥ 44pt，Android ≥ 48dp。

### 7.3 禁止模式

紫色渐变、发光描边、玻璃拟态、AI 光球、emoji 功能图标、抽象口号 Hero、卡片套卡片、三列同构功能卡、streak/排行榜/红点、完成百分比主视觉、12 个 Agent 头像聊天入口、无依据精确指标。

### 7.4 手机与平板自适应合同

原型必须首先像一款真实移动端应用，而不是桌面网页缩进手机边框。Review Console 只属于研究工作台，不进入产品画布。

| 形态 | 参考宽度 | 布局合同 | 导航与交互 |
|---|---:|---|---|
| 紧凑手机 | 320–374dp | 单列；16dp 页边距；主叙事、表单和主操作纵向排列 | 主操作位于拇指可达区；触控目标 Android≥48dp、iOS≥44pt |
| 标准手机 | 375–479dp | 单列；内容最大阅读宽度；底部操作可安全吸附但不得遮挡内容 | 底部导航 3–5 项；系统返回/边缘返回不被覆盖 |
| 大屏手机/折叠窄态 | 480–599dp | 仍以单列为主，可增加摘要并列但不改变任务顺序 | 触摸优先，同时允许键盘和指针 |
| 小平板 | 600–839dp | portrait 使用宽单列或主内容+窄辅助栏；表单不机械拉满 | 可使用 NavigationRail 或保留底部导航，依任务密度选择 |
| 标准平板 | 840–1199dp | 优先列表—详情、编辑—证据、旅程—目标双栏；单栏内容设置最大宽度 | 支持触摸+指针+键盘；详情变化不得丢失左侧选择 |
| 横屏手机 | 高度 320–479dp | 压缩顶部说明；内容可滚动；主操作不能被底栏和键盘遮挡 | 禁止固定高画布导致流程不可完成 |

Flutter 映射使用 `LayoutBuilder + MediaQuery`，并以内容驱动断点验证；不得仅用设备名称判断布局。推荐对齐 Material 3 Window Size Class：compact `<600dp`、medium `600–839dp`、expanded `≥840dp`。

**布局重排原则：**

- 手机：身份输入、渐进追问、域选择和目标编辑保持单列，次级证据渐进披露；
- 平板 portrait：主任务仍居中，不为了填满屏幕而制造卡片墙；
- 平板 landscape：A0-04 可为“草案编辑 / 来源与充分度”，A0-07 可为“目标列表 / 目标编辑”，A0-09/A0-10 可为“旅程或目标列表 / 正式详情”，A0-11 可为“版本列表 / 差异与本地状态”；
- Safety、Eligibility、Consent 在任何尺寸都保持独立高关注流程，不塞进侧栏角落；
- 键盘弹出、200% 字体和横屏时，页面必须滚动到当前字段与主操作；
- 手机和平板保持同一信息架构和状态合同，只改变编排，不隐藏核心能力。

**必须测试的设备矩阵：**

- 手机：320×568、360×800、390×844、430×932；
- 手机横屏：568×320、800×360；
- 平板 portrait：600×960、768×1024、834×1194；
- 平板 landscape：960×600、1024×768、1194×834；
- 所有关键页面至少验证 100% 与 200% 字体、触摸和键盘路径、reduced motion。

### 7.5 视觉质量方向

视觉不是“通用浅色后台”。PrimeAtlas 应形成**专业成长手册 × 精密决策工具**的移动视觉语言：

- 采用 Notion 式温暖纸张中性色，而不是冰冷蓝灰背景；
- 吸收 Linear 的精确层级、细边界和紧凑状态表达，但不继承深色紫调与发光；
- 吸收 Apple HIG 的安全区、拇指可达、动态字体和克制动效；
- 吸收 Material 3 的手机/平板自适应、NavigationRail 与列表—详情模式；
- Atlas Blue 只用于主要交互，Teal 只用于跨域关系，安全色拥有最高语义优先级；
- 通过“当前身份 → 目标身份”的持续迁移线索建立品牌记忆，而不是依赖渐变、光球、头像或装饰性图表；
- 用空间、排版、分隔线和少量表面层级组织信息，避免所有内容都成为圆角卡片；
- 手机正文不低于 16sp/px 等效，标题层级明显；平板增加信息并置，不盲目放大字号；
- 动效只解释状态变化，150–300ms 为主，禁用弹跳、持续呼吸和无意义庆祝。

## 8. Flutter 可实施架构

正式实现保持现有技术基线：

```text
lib/core     纯 Dart 领域规则、事件、仓储和引擎
lib/app      Flutter 页面、组件、Riverpod、GoRouter、Theme
SQLite       本地对象、版本、事件、outbox
SyncAdapter  Noop 起步，云端后续替换
```

### 页面到实现模式

| 原型对象 | Flutter 页面/组件建议 | 状态管理 | 数据/核心映射 |
|---|---|---|---|
| A0 身份发现 | discovery flow + draft editor | AsyncNotifier/Notifier | 新 IdentityDraft Repository；事件总线记录候选事件 |
| 身份确认 | confirmation page | command state + optimistic lock | PortraitVersioner + PortraitRepository + Audit/Outbox 事务 |
| 成长域/目标/里程碑 | list/detail/editor | family providers | 新 domain/goal/milestone repository；snake_case 契约 |
| 旅程 | aggregate screen | derived providers | Portrait + Goal + Milestone + Evidence 聚合 |
| 普通冲突 | decision page | command provider | BasicConflictDetector，`blocked_user=false` 不变量 |
| 调性 | settings selector + ThemeExtension | tone provider | ToneEngine/ToneStateMachine |
| 离线/同步 | status center | stream/async provider | EventLogRepository + NoopSyncAdapter；后续 outbox |
| 版本历史 | list/diff/restore | family providers | PortraitVersioner.diff/rollback 语义需升级为 restored version |

### 现有 S0 的复用边界

当前仓库只有 `lib/core` 公共面，没有可直接复用的 `lib/app` 页面或组件。因此原型不是对既有 Flutter UI 的换皮，而是对未来 UI 层的实施契约验证。

**可直接复用或适配：**

- `lib/core/events/identity_event_bus.dart`：候选、确认、版本和本地保存操作的领域事件入口；
- `lib/core/portrait/portrait_engine.dart` 与 `portrait_types.dart`：画像版本创建、差异和活跃维度语义；
- `lib/core/storage/portrait_repository.dart`、`sqlite/sqlite_portrait.dart`：画像版本本地持久化；
- `lib/core/storage/event_log_repository.dart`、`sqlite/sqlite_event_log.dart`：事件与审计线索本地持久化；
- `lib/core/sync/sync_adapter.dart`：V0.2 使用 `NoopSyncAdapter`，只允许表达“本机保存/同步能力未启用”；
- `lib/core/conflict/*`：仅普通冲突；必须维持 `blocked_user=false`，不得复用为 Safety；
- `lib/core/tone/*`：未来调性设置与 ThemeExtension 的表达适配，不能改变事实和安全结论。

**需要新增而非假装已有：**

- `IdentityDraft`、Assisted Discovery 状态机与 Repository；
- `ActiveDomain`、`Goal`、`Milestone`、`EvidenceLink` 的实体、Repository 与迁移；
- 身份确认原子事务：画像版本 + 审计 + 本地 outbox 同成同败；
- 安全、Eligibility、Consent 的独立领域对象和状态页；
- `lib/app` 的 Theme、Router、Provider、Screen 和共享组件层。

**实现前必须纠正的现有语义：**

- `PortraitVersioner.rollback()` 当前原地切换 current version；正式实现必须改为创建新的 restored active version，不改写历史；
- `NoopSyncAdapter` 不产生真实云状态，原型中“待同步/冲突”仅属于 Future IA 测试夹具，不能声称 V0.2 已上线云同步；
- `BasicConflictDetector` 不能承载安全否决；Safety 必须是不可覆盖的独立合同。

### 路由建议

- 用 `GoRouter` 的 `StatefulShellRoute` 承载四个 Tab 独立返回栈；
- V0.2 身份发现使用 shell 外的线性路由；
- 安全、Eligibility、Consent、普通冲突是可深链业务状态页，不只用 Toast；
- 来源跳转携带 return context，返回原证据位置；
- Tab 再次点击行为保留为原型实验，不在编码前擅自定死。

HTML 原型应使用可追踪的稳定 hash route name，而不是把一个大弹窗当作全部页面：

```text
#a0/welcome
#a0/identity/expression
#a0/identity/:draftId/questions
#a0/identity/:draftId/draft
#a0/identity/:draftId/confirm
#a0/domains
#a0/domains/:domainId/goals
#a0/goals/:goalId/milestone/:milestoneId
#journey
#journey/goals/:goalId
#me/portrait/versions
```

最终 path 和 Shell 仍须等待 IA 证据冻结；当前先保持 route name 与页面对象稳定，便于映射未来 GoRouter、Provider、Use-case、Repository、SQLite 表、事件和 Test ID。

### 原型状态与命令合同

- HTML 使用单一 `prototypeState`，至少包含 `subject / identityDraft / domains / goals / milestones / portraitVersions / localSave / sync / dependency / route`；
- 主命令使用未来可映射的业务名称：`createIdentityDraft / submitIdentityDraft / confirmIdentityDraft / activateDomain / confirmGoal / upsertMilestone / restorePortraitVersion`；
- fixture 与 Repository 明确分离，所有演示数据永久标记 `Mock / Research only`；
- A0-05 原子事务故障必须全部回滚，不能出现画像已激活但审计或 local outbox 缺失；
- A0-11 恢复历史版本创建新的 `restored(active)` 版本，禁止原地修改历史记录；
- 评审事件日志只记录模式、任务、页面、动作和状态，不记录身份自由文本、健康原文或其他敏感内容。

## 10. 原型验收门禁

### P0：任一出现即退回

- 安全否决可继续原危险方案；
- Eligibility 拒绝/无法/过期/被替代后仍进入普通精细训练；
- Pulse 成为唯一训练入口；
- 手势、颜色或动画是关键流程唯一信息；
- 普通冲突没有“我自己来”或删除目标；
- 草案/候选自动成为正式事实；
- 未激活域出现、占位或进入指标权重；
- 离线/同步覆盖已发生事实、新安全状态或 Consent 撤回；
- P-A0 出现 Today、训练、Pulse、知识或 Eligibility 半入口；
- 把内部走查冒充真实 D3/D4 证据。

### 必须验证

- 手机窄屏、平板/桌面评审器、横屏和 200% 字体；
- 全键盘操作、焦点顺序、焦点返回和 `aria-live`；
- `prefers-reduced-motion`；
- 所有任务可重置到明确起点；
- 状态切换后有持久文本，不只 Toast；
- A/B/C 使用相同内容和视觉完成度；
- 原型事件日志记录模式、任务、页面、操作和状态，但不预填“用户通过”。

### D0–D2 交付准入

提交本轮用户评审前，至少必须满足：

1. HTML/CSS/JS 解析成功，浏览器 Console 0 error，资源 0 个 404；
2. A0 主闭环从干净状态一次完成，且 11 页对象与版本状态一致；
3. A0 的 Won't Have 入口与数据对象扫描为 0；
4. Safety、Eligibility 非通过态、Consent 拒绝/撤回、普通冲突、Pulse 绕过、RPE `missing` 均无逃逸路径；
5. 键盘、语义结构、对比度、触控目标、200% 字体、窄屏、横屏和 reduced motion 完成 D2 走查；
6. 16 类 P0 禁止项为 0；任何一项存在都退回修复并重测。

本轮通过只能表述为“可进入用户评审”，不能表述为 IA Frozen。最终冻结仍要求 LF-IA-01～18 的真实 D3 裁决卡，以及核心无障碍旅程的 D4 或 QA 明确接受的等价真实证据。

## 11. 评审与推进规则

本稿用于定义编码前的原型实施规格；独立可点击 HTML 原型将按本稿直接制作，但**原型完成不等于方案已通过用户评审**。

推进顺序：

1. 按本稿制作独立可点击 HTML 原型，不修改正式 Flutter 工程；
2. 执行内部 D0/D1/D2 走查，P0 缺陷归零后提交胜总评审；
3. 胜总评审页面、流程、视觉和范围；评审通过前不进入正式 Flutter 功能编码；
4. 组织真实 D3/D4 验证，证据达标后才建议冻结 IA；
5. 回填并冻结 Spec，再进入正式 Flutter 编码。

若评审要求的是小改，如页面排序、文案或状态补充：更新本稿、原型和变更记录；若改变一级导航、主循环或 V0.2 范围，则回到产品/Spec 门禁重新校准。
