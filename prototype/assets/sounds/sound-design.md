# PrimeAtlas 音效系统设计规范

## 概述

PrimeAtlas 音效系统旨在提供沉浸式的运动训练反馈体验。所有音效默认静音，用户通过设置手动开启。

---

## 音效架构

### 分级体系

| 级别 | 名称 | 说明 | 包含音效 |
|------|------|------|----------|
| L1 | 核心音效 | 关键训练节点，不可错过 | sfx_go, sfx_pr_break, sfx_countdown, sfx_last_set |
| L2 | 反馈音效 | 训练完成确认 | sfx_set_complete, sfx_rest_end, sfx_training_done |
| L3 | 氛围音效 | 操作反馈和交互 | sfx_tap, sfx_partner_msg |

### 音量设置

| 模式 | 说明 | 行为 |
|------|------|------|
| 完全静音 | 默认模式 | 不播放任何音效 |
| 仅核心 | 只播放L1音效 | 倒计时/PR/关键节点 |
| 标准 | L1 + L2 | 训练全流程反馈 |
| 沉浸 | L1 + L2 + L3 | 包含所有操作反馈 |

### 系统适配

- 检测系统静音模式（iOS 静音开关 / Android 勿扰模式），自动设为"完全静音"
- 蓝牙耳机连接时自动降低音量至70%
- 训练计时器运行时，其他应用音频自动降低（AudioFocus）

## 音效详细规范

### L1 - 核心音效

#### sfx_countdown — 倒计时鼓点
- **场景**: 3-2-1-GO 倒计时
- **描述**: 低沉有力的鼓点，每数一个数敲击一次，力度逐次增强
- **触发时机**: 倒计时每个数字出现时
- **优先级**: 高
- **时长**: ~200ms/次
- **音色**: 低频打击乐（类似太鼓/定音鼓），60-120Hz范围
- **节奏**: 3(较弱) → 2(中等) → 1(强力) → GO!(爆发)
- **文件名**: `sfx_countdown_3.mp3`, `sfx_countdown_2.mp3`, `sfx_countdown_1.mp3`

#### sfx_go — GO! 爆发
- **场景**: 倒计时结束，训练开始
- **描述**: 重击鼓面 + 低频共鸣，充满力量感
- **触发时机**: GO! 文字出现瞬间
- **优先级**: 高
- **时长**: ~500ms
- **音色**: 大鼓重击(80Hz) + 铜钹轻击(8kHz) + 低频混响衰减
- **文件名**: `sfx_go.mp3`

#### sfx_pr_break — PR突破
- **场景**: 用户确认打破个人记录
- **描述**: 史诗级铜管齐鸣 + 鼓阵，胜利感拉满
- **触发时机**: PR确认弹窗出现时
- **优先级**: 最高
- **时长**: ~2.5s
- **音色**: 
  - 0-0.3s: 上升音阶弦乐(快速渐强)
  - 0.3-1.0s: 铜管齐鸣(法国号+小号)主旋律
  - 1.0-2.0s: 定音鼓滚奏 + 钹击
  - 2.0-2.5s: 混响自然衰减
- **文件名**: `sfx_pr_break.mp3`

#### sfx_last_set — 最后一组警告
- **场景**: 进入最后一组训练
- **描述**: 警报式低频脉冲 + 渐强，提示这是最后冲刺
- **触发时机**: 最后一组开始时
- **优先级**: 高
- **时长**: ~800ms
- **音色**: 低频合成器脉冲(40-60Hz)，3次快速脉冲，每次渐强
- **文件名**: `sfx_last_set.mp3`

---

### L2 - 反馈音效

#### sfx_set_complete — 每组完成
- **场景**: 组计时器归零，一组动作完成
- **描述**: 清脆金属敲击声 + 短回声
- **触发时机**: 组计时器归零
- **优先级**: 中
- **时长**: ~300ms
- **音色**: 三角铁/音叉清脆敲击(3-5kHz) + 0.3s混响衰减
- **文件名**: `sfx_set_complete.mp3`

#### sfx_rest_end — 休息结束
- **场景**: 组间休息倒计时最后3秒
- **描述**: 上升音阶"嘀嘀嘀"提示音
- **触发时机**: 休息倒计时 3, 2, 1 秒
- **优先级**: 中
- **时长**: ~150ms/次
- **音色**: 电子音色(1-2kHz)，音高逐次上升：C5→E5→G5
- **文件名**: `sfx_rest_end_3.mp3`, `sfx_rest_end_2.mp3`, `sfx_rest_end_1.mp3`

#### sfx_training_done — 训练完成
- **场景**: 全部动作完成
- **描述**: 胜利铜管短句 + 渐弱
- **触发时机**: 最后一个动作最后一组完成
- **优先级**: 高
- **时长**: ~1.5s
- **音色**: 铜管上行音阶(1s) + 0.5s混响衰减
- **文件名**: `sfx_training_done.mp3`

---

### L3 - 氛围音效

#### sfx_tap — 按钮点击
- **场景**: 所有按钮点击反馈
- **描述**: 轻微触感反馈音，类似机械键盘轻触
- **触发时机**: 按钮按下
- **优先级**: 低
- **时长**: ~50ms
- **音色**: 短促"嗒"声(800Hz-2kHz)，极短混响
- **文件名**: `sfx_tap.mp3`

#### sfx_partner_msg — 伙伴消息
- **场景**: 伙伴发送新消息
- **描述**: 轻柔通知音，不打扰但能注意到
- **触发时机**: 伙伴新消息气泡出现
- **优先级**: 低
- **时长**: ~200ms
- **音色**: 柔和"叮"声(2-3kHz)，类似消息提示但更轻柔
- **文件名**: `sfx_partner_msg.mp3`

## 技术规范

### 文件格式

| 格式 | 用途 | 编码 |
|------|------|------|
| MP3 | 主力格式 | 128kbps CBR, 44.1kHz |
| OGG | 兼容格式 | 96kbps VBR, 44.1kHz |

### 预加载策略

```
页面加载 → 预加载L1核心音效（立即）
训练页面进入 → 预加载L2反馈音效
设置开启音效 → 预加载L3氛围音效
```

### 播放控制 API 设计

```javascript
// 音效管理器接口
const SoundManager = {
  // 设置音量模式: 'mute' | 'core' | 'standard' | 'immersive'
  setMode(mode),
  
  // 播放指定音效
  play(soundId),
  
  // 预加载音效组
  preload(soundIds[]),
  
  // 停止所有音效
  stopAll(),
  
  // 检查系统静音状态
  checkSystemMute(),
  
  // 获取当前音量
  getVolume(),
  
  // 设置自定义音量 0.0-1.0
  setVolume(level),
};
```

### 音效叠加规则

- 同一音效最小间隔: 200ms（防抖）
- L1音效可打断L2/L3（优先级抢占）
- L2音效排队播放，不叠加
- L3音效不排队，丢弃超时的

### 性能要求

- 音效文件总大小 < 2MB
- 单个音效文件 < 200KB
- 预加载不阻塞页面渲染
- 播放延迟 < 50ms（Web Audio API）

## 音效状态指示器 UI

在训练页面和设置页面显示音效状态：

```
[🔇 静音] → [🔈 核心] → [🔉 标准] → [🔊 沉浸]
```

- 快速切换：点击图标循环切换
- 长按图标：弹出音量滑块
- 训练中自动显示在计时器旁边
